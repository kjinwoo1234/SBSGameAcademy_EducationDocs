# CurriculumReviewSkill

커리큘럼(복수 문서/트랙) 리뷰와 개선 업무를 수행할 때 사용하는 AI 가이드라인입니다.

## 1) 목적
- 학원 강의 운영용 커리큘럼으로서 구조/순서/난이도/완성도를 정량 평가한다.
- 시중 학습 커리큘럼(입문서/온라인 코스) 대비 부족한 지점을 식별한다.
- 즉시 반영 가능한 개편안(재배치/신설/통합/분리)을 제안한다.

## 2) 리뷰 범위
- 대상 트랙의 `README` + 하위 학습 문서 전체
- 필요 시 인접 트랙(C, C++, C#, Unity, Unreal)과 연계성까지 점검

## 3) 평가 항목 (100점)
아래 10개 항목을 합산해 100점으로 평가한다.

1. 학습 목표/수강 대상 명확성 (10점)
2. 커리큘럼 순서의 논리성(선수지식 -> 심화) (12점)
3. 난이도 상승 곡선의 안정성 (10점)
4. 핵심 주제 커버리지 완성도 (12점)
5. 실습/프로젝트 연계성 (12점)
6. 평가/복습 체계(퀴즈, 체크리스트, 과제) (10점)
7. 문서 간 연결성(이전/다음 학습, 참조 링크) (8점)
8. 초보자 친화성(용어, 설명 방식) (8점)
9. 시중 커리큘럼 대비 경쟁력 (10점)
10. 확장성(중급/고급 트랙 연결 가능성) (8점)

## 4) 비교 기준 (시중 커리큘럼)
- 최소 2개 이상의 일반적 레퍼런스 구조와 비교한다.
  - 예: 입문 -> 실습 -> 미니프로젝트 -> 심화 파이프라인
  - 예: 개념 40% + 실습 40% + 검증/회고 20% 비율
- 비교 시 "장점/열세"를 항목별로 명시한다.

## 5) 리뷰 작성 규칙
- 요약보다 문제점/리스크를 우선 제시한다.
- 각 문제는 반드시 원인과 개선 방법을 함께 작성한다.
- 개선안은 즉시 실행 가능한 작업 단위로 작성한다.
- 점수는 항목별 점수 + 총점 + 감점 근거를 숫자로 제시한다.
- 실제 문서를 직접 읽은 근거를 최소 2개 이상(문장/섹션 단위) 명시한다.

## 6) 개선안 작성 형식
각 이슈는 아래 형식을 따른다.

- 문제: 현재 커리큘럼의 결함
- 영향: 학습자/강의 운영 측면 손실
- 수정 제안: 문서 재배치/신설/통합/분리 등 구체 작업
- 예상 효과: 학습 효율 또는 운영 효율 변화

## 7) 확장 판단 규칙 (신규 커리큘럼 신설/확장)
다음 조건 중 2개 이상이면 확장을 권고한다.
- 핵심 선수지식 문서가 부재
- 중급/고급으로 이어지는 다리 문서가 부재
- 실습 프로젝트가 단절되어 성취 경험이 약함
- 산업 현장 요구 역량(협업, 배포, 디버깅) 반영 부족

확장 시 반드시 아래를 포함한다.
- 신규 문서 목록(파일명 단위)
- 권장 학습 순서 재정의
- 기존 대비 예상 점수 변화

## 8) 개선 후 재평가
- 제안 반영을 가정한 "개선 후 예상 점수"를 제시한다.
- 상승 폭(예: +18점)과 핵심 상승 요인을 2~3개로 요약한다.

## 9) 결과물 저장 위치
- 기본 결과 문서는 `./.result/ReviewResult-curriculum.md`로 출력한다.
- 통합 리뷰 수행 시 `./.result/SkillIntegratedReviewResult.md`를 사용한다.

## 10) 출력 템플릿 (필수)
아래 구조를 반드시 지킨다.

1. 대상 커리큘럼(폴더/문서 범위)
2. 총평 (적합/보완 필요/부적합)
3. 항목별 점수표 (10개 항목 + 총점)
4. 시중 커리큘럼 비교 (장점/열세)
5. 주요 문제점 TOP 3~5 (심각도 순)
6. 상세 개선안 (즉시 반영 가능한 작업 단위)
7. 확장 필요 여부 및 신규 문서 제안
8. 개선 후 예상 점수 및 근거
9. 다음 액션 (우선 수정 순서 1~3)
