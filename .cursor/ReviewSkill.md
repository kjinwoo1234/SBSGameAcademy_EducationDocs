# ReviewSkill

문서 리뷰/개선 업무를 수행할 때 사용하는 AI 가이드라인입니다.

## 1) 목적
- 학원 강의자료/예습복습 자료로서 문서 품질을 정량/정성 평가한다.
- 문제점을 찾고, 바로 실행 가능한 개선안을 제시한다.
- 개선 후 기대 점수까지 예측해 문서 개선 우선순위를 명확히 한다.

## 2) 리뷰 범위
- 전달받은 대상 문서 1개 이상
- 필요 시 연결 문서(README, 다음 학습 문서 링크)까지 점검

## 3) 평가 항목 (100점)
아래 8개 항목을 합산해 100점으로 평가한다.

1. 학습 목표 명확성 (15점)
2. 개념 정확성/오개념 방지 (15점)
3. 초보자 친화성(용어, 설명 난이도) (15점)
4. 실습 가능성(단계별 절차 유무) (15점)
5. 검증 요소(체크리스트/퀴즈) (10점)
6. 학습 흐름 연결성(이전/다음 문서) (10점)
7. 문서 구조/가독성(제목, 섹션, 길이 균형) (10점)
8. 예제 품질(실행 가능성, 실무성) (10점)

## 4) 리뷰 작성 규칙
- 칭찬보다 문제점/리스크를 우선 제시한다.
- 문제점은 반드시 "왜 문제인지"와 "어떻게 고칠지"를 함께 작성한다.
- 개선안은 추상 표현 금지, 문서에 바로 반영 가능한 작업 단위로 작성한다.
- 점수는 항목별 점수 + 총점 + 감점 근거를 숫자로 제시한다.

## 5) 개선안 작성 형식
각 이슈는 아래 형식을 따른다.

- 문제: 현재 상태의 결함
- 영향: 학습자에게 발생하는 혼란/학습 손실
- 수정 제안: 문서에 추가/수정할 구체 내용
- 예상 효과: 개선 후 기대 변화

## 6) 개선 후 재평가
- 제안 반영을 가정한 "개선 후 예상 점수"를 제시한다.
- 상승 폭(예: +14점)과 핵심 상승 요인을 2~3개로 요약한다.

## 7) 결과물 저장 위치
- 결과 문서는 `./.result/ReviewResult.md`로 출력한다.

## 8) 출력 템플릿 (필수)
아래 템플릿 구조를 반드시 지킨다.

1. 대상 문서
2. 총평 (적합/보완 필요/부적합)
3. 항목별 점수표 (8개 항목 + 총점)
4. 주요 문제점 TOP 3~5 (심각도 순)
5. 상세 개선안 (즉시 반영 가능한 문장 수준)
6. 개선 후 예상 점수 및 근거
7. 다음 액션 (우선 수정 순서 1~3)
