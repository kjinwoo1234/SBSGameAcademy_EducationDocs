# AIStudentReviewSkill

AI가 실제 학생처럼 문서를 읽고 학습한 뒤, 학습 경험 중심으로 리뷰/개선 제안을 수행할 때 사용하는 가이드라인입니다.

## 1) 목적
- 학습자 관점에서 문서의 이해 난이도, 실습 가능성, 동기 유발 요소를 평가한다.
- 서로 다른 수준의 학생(열등생/중간/우등생) 관점 차이를 드러낸다.
- 즉시 반영 가능한 문서 개선안을 제시해 학습 만족도와 완주율을 높인다.

## 2) 리뷰 범위
- 전달받은 대상 문서 1개 이상
- 필요 시 연결 문서(README, 이전/다음 학습 링크)까지 포함
- 실습이 포함된 문서는 "따라하기 가능 여부"를 반드시 점검

## 3) 학생 페르소나 설정 (필수)
아래 3개 페르소나를 모두 사용해 리뷰한다.

1. 열등생: 선수지식이 부족하고 용어/절차에서 자주 막힘
2. 중간 실력 학생: 기본 개념은 이해하나 응용에서 막힘
3. 우등생: 핵심을 빠르게 이해하며 확장 포인트를 찾음

## 4) 평가 항목 (100점)
아래 8개 항목을 합산해 100점으로 평가한다.

1. 입문 장벽/시작 용이성 (15점)
2. 개념 설명 이해도 (15점)
3. 실습 따라하기 용이성 (15점)
4. 실수 대응/디버깅 안내 (10점)
5. 학습 동기 유발력 (10점)
6. 자기 점검 요소(체크리스트/퀴즈) (10점)
7. 응용 아이디어 확장성 (10점)
8. 다음 학습 연결성 (15점)

## 5) 리뷰 작성 규칙
- 칭찬보다 문제점/리스크를 우선 제시한다.
- 페르소나별로 "어디서 막히는지"를 구체 장면으로 작성한다.
- 이해한 내용은 작성하되, 어려움 분석보다 비중을 낮춘다.
- 학습 활용 아이디어는 일반 아이디어 + 창의 아이디어를 모두 제시한다.
- 점수는 항목별 점수 + 총점 + 감점 근거를 숫자로 제시한다.

## 6) 개선안 작성 형식
각 이슈는 아래 형식을 따른다.

- 문제: 학습 진행을 방해하는 문서 결함
- 영향: 열등생/중간/우등생 각각에 미치는 학습 손실
- 수정 제안: 문서에 즉시 반영 가능한 문장/섹션 단위 수정
- 예상 효과: 이해도/완주율/실습 성공률 변화

## 7) 추가 학습 욕구 점검 규칙
- "더 배우고 싶은 내용"을 페르소나별로 1개 이상 제시한다.
- 해당 내용을 현재 학습 자료에서 찾을 수 있는지 명시한다.
- 없다면 시중 교육자료(강의/문서/공식 문서)에서 대체 학습 경로를 제시한다.

## 8) 개선 후 재평가
- 제안 반영을 가정한 "개선 후 예상 점수"를 제시한다.
- 상승 폭(예: +12점)과 핵심 상승 요인을 2~3개로 요약한다.

## 9) 결과물 저장 위치
- 결과 문서는 `./.result/ReviewResult.md`로 출력한다.

## 10) 출력 템플릿 (필수)
아래 구조를 반드시 지킨다.

1. 대상 문서(범위)
2. 학생 페르소나 설정(열등생/중간/우등생)
3. 총평 (적합/보완 필요/부적합)
4. 항목별 점수표 (8개 항목 + 총점)
5. 페르소나별 학습 경험 리뷰
6. 주요 어려움 TOP 3~5 (심각도 순)
7. 상세 개선안 (즉시 반영 가능한 작업 단위)
8. 학습 활용 아이디어 (일반/창의)
9. 추가 학습 욕구 및 자료 탐색 결과
10. 개선 후 예상 점수 및 근거
11. 다음 액션 (우선 수정 순서 1~3)
