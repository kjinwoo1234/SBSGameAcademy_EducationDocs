# ApplyReviewSkill

`AIStudentReviewSkill`, `CurriculumReviewSkill`, `DocumentReviewSkill`의 리뷰 결과를 실제 문서 개선 작업으로 전환할 때 사용하는 통합 실행 가이드입니다.

## 1) 목적
- 리뷰 문서를 "분석 보고서"에서 "실제 문서 수정"으로 연결한다.
- 학생 관점/문서 품질/커리큘럼 구조 관점을 함께 반영해 개선 우선순위를 정한다.
- 변경 후 재검토까지 포함한 개선 루프를 표준화한다.

## 2) 입력 문서
- 학생 관점 리뷰: `./.result/ReviewResult-student.md`
- 문서 품질 리뷰: `./.result/ReviewResult-document.md`
- 커리큘럼 리뷰: `./.result/ReviewResult-curriculum.md`
- 통합 리뷰(있는 경우): `./.result/SkillIntegratedReviewResult.md`

리뷰 문서가 일부만 있어도 실행 가능하지만, 가능한 3종 모두를 먼저 반영한다.

## 3) 적용 원칙
1. **타당성 검토 우선**
   - 각 리뷰의 이슈가 실제 문서에서 확인되는지 원문으로 재확인한다.
2. **근거 기반 수정**
   - 모든 수정 항목은 "문제-영향-수정-예상 효과"를 1줄 이상 기록한다.
3. **작은 배치 적용**
   - 대규모 일괄 수정 대신 트랙/섹션 단위로 나눠 반영한다.
4. **학생 경험 우선**
   - 점수 개선보다 "학습자가 실제로 덜 막히는가"를 우선 판단 기준으로 둔다.

## 4) 리뷰 통합 방식
중복/충돌 이슈는 아래 규칙으로 병합한다.

- 공통으로 반복된 이슈: **즉시 반영 1순위**
- 학생 관점에서 심각(이탈 위험) + 문서 품질도 낮음: **최우선**
- 커리큘럼 구조 이슈(문서 신설/순서 개편): **2순위**
- 표현 다듬기/문체 개선: **후순위**

충돌 시 우선순위:
1) 학습 실패를 줄이는 수정  
2) 검증 요소(체크리스트/퀴즈) 보강  
3) 구조/표현 정리

## 5) 실행 절차 (필수)
### Step 1. 이슈 수집
- 각 리뷰의 `주요 문제점`, `상세 개선안`, `다음 액션`을 모아 통합 리스트를 만든다.
- 동일 이슈를 묶어 중복 항목을 제거한다.

### Step 2. 작업 분류
- 분류 A: 즉시 수정 가능한 문장/섹션 추가
- 분류 B: 다수 문서 템플릿 표준화
- 분류 C: 신규 문서 신설(브릿지/미니프로젝트)

### Step 3. 우선순위 부여
각 작업에 아래 태그를 붙인다.
- 영향도: `상/중/하`
- 작업비용: `상/중/하`
- 범위: `단일 문서/트랙/전체`

권장 실행 순서:
1. 영향도 상 + 작업비용 하  
2. 영향도 상 + 작업비용 중  
3. 나머지

### Step 4. 문서 수정
- 실제 파일을 수정하고, 링크/섹션/용어 일관성을 함께 점검한다.
- 템플릿 확장은 대표 문서(입문/중간/마무리)부터 적용 후 전체 확장한다.

### Step 5. 검증
- 수정 파일 기준으로 오탈자/링크/섹션 누락을 확인한다.
- 기존 리뷰 이슈가 실제로 해소되었는지 재확인한다.

### Step 6. 재평가
- 동일 스킬로 재리뷰해 점수 변화를 기록한다.
- 필요하면 `ReviewResult-*` 파일을 최신화한다.

## 6) 문서 개선 체크리스트
- [ ] README에 시작 가이드/진행 체크가 있는가?
- [ ] 각 트랙 README에 권장 대상/선수지식/완주 후 역량이 있는가?
- [ ] 본문 문서에 학습 목표/실습/검증 요소가 있는가?
- [ ] 문서 간 이전/다음/상위 링크가 자연스러운가?
- [ ] 공통 브릿지(디버깅/협업)가 트랙과 연결되어 있는가?
- [ ] 학생 관점 이상점(최소 3개)이 실제로 해소되었는가?

## 7) 결과물 작성 규칙
- 개선 실행 로그는 `./.result/SkillIntegratedReviewResult.md`에 누적 정리한다.
- 스킬별 재리뷰 결과는 기존 파일명 규칙을 유지한다.
  - `ReviewResult-student.md`
  - `ReviewResult-document.md`
  - `ReviewResult-curriculum.md`

## 8) 출력 템플릿 (실행 결과 보고)
아래 구조를 따른다.

1. 적용한 리뷰 문서 목록
2. 통합 이슈 목록(중복 제거)
3. 우선순위 매트릭스(영향도/작업비용)
4. 실제 반영한 파일 목록
5. 반영 내용 요약(문제-수정-효과)
6. 미반영 항목 및 사유
7. 재평가 점수 변화
8. 다음 액션(1~3)
