# SVN 기본과 작업본 이해

SVN은 중앙 저장소 기반 버전 관리 시스템입니다. 로컬 작업본은 중앙 저장소와의 동기화를 기준으로 관리됩니다.

## 학습 목표
- 저장소 체크아웃과 상태 확인을 수행할 수 있다.
- 작업본과 저장소의 관계를 설명할 수 있다.
- 기본 커밋 흐름을 이해할 수 있다.

## 핵심 명령어
```bash
svn checkout <repository-url>
svn status
svn update
svn commit -m "입력 예외 처리 보강"
```

## 단계별 실습
1. 저장소를 체크아웃한다.
2. 파일을 수정한 뒤 `svn status`로 변경을 확인한다.
3. `svn update`로 최신 변경을 반영한다.
4. 충돌이 없으면 `svn commit`으로 반영한다.

## 실수 포인트
- 업데이트 없이 바로 커밋해 충돌 위험을 높이는 문제
- 커밋 메시지를 모호하게 작성해 변경 목적이 불분명해지는 문제

## 셀프 퀴즈
1. SVN에서 작업 시작 전에 `svn update`를 권장하는 이유는?
2. `svn status`로 먼저 확인해야 하는 이유는?

## 정답
1. 중앙 저장소 최신 변경을 반영해 충돌 가능성을 줄이기 위해서
2. 자신의 변경 범위를 정확히 파악하고 누락을 방지하기 위해서
