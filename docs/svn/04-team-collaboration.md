# 협업 운영 실전

SVN 협업에서는 공통 규칙을 명확히 정하는 것이 생산성을 좌우합니다.

## 학습 목표
- 팀 단위 커밋 규칙과 리뷰 기준을 설정할 수 있다.
- 릴리스 전 점검 루틴을 구성할 수 있다.
- 장애 발생 시 이전 리비전으로 복구 절차를 이해할 수 있다.

## 운영 규칙 예시
- 커밋 전 `svn update` 필수
- 커밋 메시지에 변경 목적과 영향 범위 명시
- 릴리스 전 `tags` 생성과 검증 보고 기록

## 단계별 실습
1. 팀 규칙 문서를 만들고 커밋 메시지 템플릿을 정의한다.
2. 기능 작업 1건을 선택해 `update -> 수정 -> commit` 흐름을 수행한다.
3. 리뷰어가 변경 목적/영향 범위를 이해할 수 있는지 상호 점검한다.
4. 릴리스 후보 리비전에 태그를 생성하고 테스트 결과를 기록한다.

## 실전 체크리스트
- 변경 파일 목록과 목적이 리뷰어에게 명확한가?
- 충돌 해결 이력이 문서화되어 있는가?
- 릴리스 후보 리비전에 대한 테스트 결과가 있는가?

## 이해 점검 체크리스트
- 팀 공통 커밋 규칙이 문서화되어 있는가?
- 릴리스 전 태그 생성과 검증 기록을 남겼는가?
- 장애 발생 시 복구 리비전을 빠르게 찾을 수 있는가?
