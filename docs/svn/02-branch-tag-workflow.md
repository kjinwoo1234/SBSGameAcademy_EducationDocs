# 브랜치와 태그 운영

SVN에서는 보통 `trunk/branches/tags` 구조를 명확히 분리해 운영합니다.

## 학습 목표
- `trunk`, `branches`, `tags`의 역할을 설명할 수 있다.
- 기능 작업용 브랜치 생성/병합 흐름을 이해할 수 있다.
- 릴리스 태그의 의미를 설명할 수 있다.

## 표준 구조
- `trunk`: 메인 개발 라인
- `branches`: 기능/유지보수 분기
- `tags`: 릴리스 스냅샷(읽기 전용 운영 권장)

## 기본 명령어
```bash
svn copy ^/trunk ^/branches/feature-input-check -m "기능 브랜치 생성"
svn merge ^/branches/feature-input-check
svn copy ^/trunk ^/tags/release-1.0.0 -m "릴리스 태그 생성"
```

## 단계별 실습
1. 현재 저장소 구조에서 `trunk/branches/tags` 위치를 확인한다.
2. `svn copy`로 기능 브랜치를 생성한다.
3. 브랜치에서 작업 후 `trunk`로 병합 시뮬레이션을 수행한다.
4. 릴리스 시점으로 가정해 `tags/release-1.0.0` 태그를 생성한다.
5. 태그는 수정 대상이 아님을 팀 규칙으로 명시한다.

## 실수 포인트
- 태그를 일반 작업 브랜치처럼 수정하는 문제
- 브랜치 목적/기간 없이 무분별하게 생성하는 문제
