# 업데이트와 충돌 해결

SVN은 중앙 저장소 기준으로 동작하므로, 업데이트 타이밍과 충돌 처리 절차가 매우 중요합니다.

## 학습 목표
- 업데이트 전후 상태를 점검할 수 있다.
- 충돌 파일 표식을 이해하고 수동 병합할 수 있다.
- 해결 후 커밋 전 검증 절차를 수행할 수 있다.

## 기본 절차
1. `svn update` 실행
2. 충돌 파일 확인(`C` 표시)
3. 수동 병합
4. `svn resolve --accept working <file>`
5. 빌드/테스트 후 커밋

## 핵심 명령어
```bash
svn update
svn status
svn resolve --accept working src/player_move.cpp
svn commit -m "충돌 해결 후 이동 로직 통합"
```

## 실수 포인트
- 충돌 원인 분석 없이 한쪽 변경만 덮어쓰는 문제
- 해결 후 테스트 없이 즉시 커밋하는 문제
