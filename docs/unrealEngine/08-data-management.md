# 데이터 관리 (DataTable, SaveGame)

게임 밸런스용 정적 데이터와 플레이 진행 저장 데이터는 성격이 달라 분리 설계가 필요합니다.

## 학습 목표
- DataTable과 SaveGame의 용도를 구분할 수 있다.
- Struct 기반 DataTable을 구성할 수 있다.
- 저장/불러오기 흐름을 구현하고 버전 이슈를 점검할 수 있다.

## 핵심
- `DataTable`: 아이템/몬스터 스펙 같은 정적 데이터 관리
- `Struct`: 데이터 스키마 정의
- `SaveGame`: 플레이어 진행 상태를 슬롯 단위로 저장/로드

## 단계별 실습
1. 아이템 정보를 담는 Struct를 만든다.
2. CSV 또는 에디터 입력으로 DataTable을 생성한다.
3. SaveGame 클래스를 만들고 플레이어 레벨/골드 값을 저장한다.
4. 시작 시 Load, 종료 시 Save 흐름을 테스트한다.

## 실수 포인트
- SaveGame 구조를 바꿨는데 버전 처리 없이 로드해 데이터가 깨진다.
- DataTable 키 이름과 조회 키가 달라 데이터를 찾지 못한다.
