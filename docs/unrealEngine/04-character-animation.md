# Chapter 04 캐릭터 애니메이션을 통한 복잡한 상호작용

## 학습 목표
- 애니메이션 블루프린트에서 상태 머신과 변수 연동을 설명할 수 있다.
- 애니메이션 몽타주와 노티파이로 타이밍 기반 공격을 설계할 수 있다.

## 세부 주제
- 애니메이션 블루프린트, 몽타주, 콤보 실습, 데미지 프레임워크 개념

## 실습 체크리스트
- Third Person 템플릿의 애니메이션 블루프린트에서 속도 변수로 Locomotion 전환을 확인한다.
- 몽타주에 **Anim Notify**를 넣고 공격 판정 함수를 호출한다.

## 본문

### 4-1 애니메이션 블루프린트

- **Anim Blueprint**는 스켈레탈 메시의 **포즈를 매 틱 계산**하는 그래프입니다.
- **Event Graph**에서 속도·입력 값을 읽고, **Anim Graph**의 **State Machine**으로 걷기/달리기/점프를 전환합니다.

용어 설명:
| 용어 | 의미 |
|------|------|
| State Machine | 애니메이션 상태(Idle, Run)와 전환 조건을 관리하는 구조 |

---

### 4-2 애니메이션 몽타주

- **Anim Montage**는 여러 애니 클립을 섹션으로 묶고 **재생 구간·루프·슬롯**을 제어합니다.
- 공격·스킬처럼 **기본 로코모션과 섞어 재생**할 때 자주 씁니다.
- **Default Slot**에 몽타주를 끼우면 State Machine의 기본 포즈 위에 상체 공격 등을 겹칠 수 있습니다.

오류 방지(몽타주·슬롯):
1. **언제**: 몽타주 재생 노드를 호출했는데 캐릭터 포즈가 전혀 바뀌지 않을 때.
2. **왜**: **Anim Graph**의 **Slot** 이름과 몽타주가 사용하는 **슬롯** 이름이 다르거나, 몽타주가 **다른 스켈레톤**용이면 재생이 무시됩니다.
3. **해결**: Anim Blueprint 그래프에서 **Slot** 노드 이름을 몽타주 에셋의 슬롯과 맞추고, 메시·스켈레톤·ABP가 한 캐릭터 계열로 연결됐는지 확인합니다.

---

### 4-3 (실습) 몽타주·노티파이로 콤보 공격

- **Anim Notify** 또는 **Notify State**의 시작·끝에 **공격 판정**(Sphere Trace, Hitbox)을 둡니다.
- 콤보는 **입력 버퍼**와 **몽타주 섹션 전환**으로 이어 붙입니다.

#### 데미지 프레임워크

- 단순히 `Apply Damage`만 쓰기보다 **인터페이스·태그·데미지 타입**으로 받는 쪽과 주는 쪽을 나누면 확장에 유리합니다.
- GAS를 쓰면 **Gameplay Effect**로 HP 변화를 일관되게 처리할 수 있습니다(9장과 연결).

예시: Anim Notify 추가(재현 절차)
1. **애니메이션 몽타주** 에셋을 연다.
2. 타임라인에서 손이 적중할 프레임에 **Notify**를 추가한다.
3. Notify에서 **블루프린트 함수**(예: `ApplyMeleeDamage`)를 호출하도록 설정한다.
4. PIE로 재생해 해당 프레임에 로그가 찍히는지 확인한다.

절차 해석:
- Notify는 **애니메이션 타임라인에 맞춘 콜백(callback, 정해진 프레임에서 엔진이 대신 호출해 주는 함수 연결)**이므로, Tick보다 **판정 시점이 디자이너에게 명확**합니다.

연습문제:
1. 문제: Anim Notify와 Tick에서 매 프레임 공격 판정을 하는 방식의 장단점을 각각 한 줄씩 적으세요.
   - 입력: 해당 없음
   - 출력: Notify 장단점, Tick 장단점 각 2줄 이내
   - 조건: 성능·타이밍 명확성 중 하나씩 포함
2. 문제: State Machine 전환 조건으로 "속도 크기" 대신 "가속도"를 쓰면 어떤 연출에 유리한지 예를 드세요.
   - 입력: 해당 없음
   - 출력: 연출 예 1개 + 이유 1문장
   - 조건: 급출발·급정지 중 하나 언급
3. 문제: Third Person 템플릿에서 **Speed** 변수가 0일 때와 600일 때 재생되는 애니 상태 이름을 각각 적으세요.
   - 입력: 실행 중인 템플릿 프로젝트
   - 출력: 상태 이름 또는 애니 시퀀스 이름
   - 조건: Anim Blueprint에서 변수 출처를 한 번 열어볼 것

정답 포인트:
- ABP = 포즈 계산 + 상태 전환; Montage = 섹션 단위 재생; 콤보 = 노티파이 타이밍 + 입력 버퍼.
