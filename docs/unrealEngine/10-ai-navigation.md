# Chapter 10 AI와 내비게이션

## 학습 목표
- NavMesh 기반 이동의 기본 개념을 이해한다.
- 간단한 추적 AI를 구현할 수 있다.

## 세부 주제
### 10-1 NavMesh
- 이동 가능한 영역 베이크
### 10-2 AI 이동 명령
- MoveTo
### 10-3 Behavior 확장
- 순찰/추적 분기

## 실습 체크리스트
- 적이 플레이어를 추적하는 기본 AI를 만든다.

## 본문
### 10-1 NavMesh
- 내비게이션 메시는 AI가 이동 가능한 공간 정보를 제공합니다.
### 10-2 AI 이동 명령
- 타깃 위치를 기준으로 경로를 계산해 이동합니다.
### 10-3 Behavior 확장
- 거리/시야 조건으로 순찰과 추적을 분기할 수 있습니다.

약어 설명(AI):
- **AI**는 **Artificial Intelligence**의 약자입니다.
- 게임에서는 NPC의 의사결정/행동 제어 로직을 뜻합니다.
- 초급 단계에서는 규칙 기반 AI부터 구현하는 것이 적합합니다.

예시 코드(AI 이동 명령):
```cpp
AAIController* AI = Cast<AAIController>(GetController());
if (AI) { AI->MoveToActor(TargetActor); }
```

예상 결과:
```text
플레이어 접근 시 적이 경로를 따라 추적
```

코드 해석:
- AI Controller가 내비게이션 경로를 계산해 목표까지 이동합니다.
- 타깃 유효성 검사를 함께 두면 런타임 오류를 줄일 수 있습니다.

연습문제:
1. 문제: 순찰 포인트 순환 이동
   - 입력: 포인트 배열
   - 출력: 순찰 경로 이동
   - 조건: 도착 후 다음 포인트
2. 문제: 추적 범위 제한
   - 입력: 플레이어 거리
   - 출력: 범위 내 추적, 범위 밖 복귀
   - 힌트: 거리 비교 조건

정답 포인트:
- NavMesh 설정이 AI 이동 성공의 전제
- 단순 규칙부터 확장하는 방식 권장
