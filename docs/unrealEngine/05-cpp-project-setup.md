# Chapter 05 C++ 프로젝트 설정

## 학습 목표
- Unreal C++ 프로젝트 구조를 이해한다.
- 클래스 추가/빌드/핫리로드 기본 흐름을 익힌다.

## 세부 주제
### 05-1 프로젝트 파일 구조
- Source, Config, Content
### 05-2 C++ 클래스 추가
- Actor 클래스 생성
### 05-3 빌드와 반영
- Compile 버튼/IDE 빌드

## 실습 체크리스트
- 새 C++ Actor 클래스를 생성해 레벨에 배치한다.

## 본문
### 05-1 프로젝트 파일 구조
- Source는 코드, Content는 에셋, Config는 설정을 담습니다.
### 05-2 C++ 클래스 추가
- 에디터 메뉴로 클래스를 만들면 기본 헤더/소스가 생성됩니다.
### 05-3 빌드와 반영
- 코드 수정 후 빌드하여 에디터에 반영합니다.

예시 코드:
```cpp
// MyActor.cpp
#include "MyActor.h"

AMyActor::AMyActor()
{
    PrimaryActorTick.bCanEverTick = true;
}
```

예상 결과:
```text
새 Actor 클래스를 월드에 배치 가능
```

코드 해석:
- 생성자에서 Tick 동작 여부를 초기 설정합니다.
- 클래스 생성 -> 빌드 -> 에디터 반영이 C++ 작업의 기본 루틴입니다.

연습문제:
1. 문제: 로그 출력 Actor 생성
   - 입력: 없음
   - 출력: BeginPlay 로그
   - 조건: C++ 클래스 생성 후 빌드
2. 문제: Tick 비활성화 테스트
   - 입력: Tick on/off
   - 출력: 성능/동작 차이
   - 힌트: `bCanEverTick` 값 변경

정답 포인트:
- 코드/에셋/설정 경계를 이해해야 유지보수 쉬움
- 빌드 반영 루틴을 안정적으로 익히는 것이 중요
