# Chapter 10 도전과제

## 학습 목표
- Lyra Starter Game의 모듈 구조와 학습 포인트를 스스로 목록화할 수 있다.
- 샘플 프로젝트에서 "읽을 것"과 "따라 칠 것"을 구분한다.

## 세부 주제
- LyraStarterGame 분석 관점

## 실습 체크리스트
- Epic Launcher 또는 소스에서 **Lyra**를 받아 프로젝트를 연다.
- **Plugins**, **GameFeature**, **UI**, **GAS** 관련 폴더를 각각 1회 이상 연다.

## 본문

### 10-1 모범 사례 프로젝트 분석 (LyraStarterGame)

- **Lyra**는 언리얼 팀이 제공하는 **멀티플레이·GAS·UI·콘텐츠 확장**을 보여 주는 레퍼런스입니다.
- 처음부터 전부 읽기보다 **플레이 모드 → 입력 → 캐릭터 → 무기 → UI** 순으로 추적합니다.
- **샘플 코드 전체 복사**보다 **폴더 구조·모듈 경계**를 배우는 것이 목표에 맞습니다.

분석 체크 표:
| 영역 | 무엇을 볼지 |
|------|-------------|
| Gameplay | Ability Set, 캐릭터 데이터 에셋 |
| 네트워크 | 복제 변수, RPC 사용 위치 |
| UI | CommonUI, 위젯 계층 |
| 확장 | Game Feature Plugin 로딩 흐름 |

용어 설명:
| 용어 | 의미 |
|------|------|
| Game Feature Plugin | 런타임에 기능 묶음을 켜고 끄는 확장 단위 |

예시: Lyra 프로젝트 열기(재현 절차)
1. Epic Games Launcher **Learn** 또는 마켓에서 **Lyra Starter Game**을 내려받는다.
2. 언리얼 버전을 프로젝트와 맞춘 뒤 `.uproject`를 연다.
3. 셰이더·에셋 컴파일이 끝날 때까지 기다린 뒤 PIE로 한 판 플레이한다.
4. **콘텐츠 브라우저**에서 `Plugins`, `Source`, `GameFeatures` 중 하나를 골라 폴더 트리만 10분간 훑는다.

절차 해석:
- Lyra는 **완성 게임**에 가까운 분량이라, 한 번에 소화하려 하기보다 **질문을 정하고 검색**하는 방식이 효율적입니다.

연습문제:
1. 문제: Lyra에서 "한 가지 스킬"이 ASC에서 어빌리티로 등록되는 경로를 파일/클래스 이름 수준으로 추적해 보고 메모하세요.
   - 입력: 로컬에 클론된 Lyra
   - 출력: 에셋 경로 또는 C++ 클래스 이름 2개 이상
   - 조건: 검색 키워드(`GameplayAbility`, `AbilitySet` 등)를 메모에 적을 것
2. 문제: 자신의 미니 프로젝트에 Lyra에서 베껴올 **한 가지** 패턴과, 당장 베끼면 안 되는 **한 가지**를 구분해 적으세요.
   - 입력: 해당 없음
   - 출력: 패턴 1 + 이유, 비권장 1 + 이유
   - 조건: 팀 규모·기한을 한 번 언급
3. 문제: Lyra의 **CommonUI** 사용 여부를 확인하고, 본인 프로젝트가 "메뉴 1개" 수준이면 CommonUI를 도입할 가치가 있는지 한 문장으로 판단하세요.
   - 입력: Lyra UI 폴더 탐색
   - 출력: 판단 + 이유 2문장
   - 조건: 학습 비용 vs 확장성 중 하나

정답 포인트:
- Lyra = 구조 참고용; 전부 복사보다 **원리 이해 후 선택적 도입**.
