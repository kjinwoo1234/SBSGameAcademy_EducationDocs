# Chapter 02 게임 꾸미기

## 학습 목표
- 액터 배치·라이팅·랜드스케이프로 기본 맵을 구성할 수 있다.
- 머티리얼과 랜드스케이프 머티리얼로 색·애니메이션·질감을 표현할 수 있다.

## 세부 주제
- 액터 배치로 게임 맵 꾸미기
- 라이팅으로 게임 조명 추가
- 랜드스케이프란? / 랜드스케이프로 지형 꾸미기
- 머티리얼이란? / 색·색 애니메이션·도형 표현
- 랜드스케이프 머티리얼로 바닥 질감 추가

## 실습 체크리스트
- 콘텐츠 브라우저에서 메시를 끌어 레벨에 배치한다.
- Directional Light + Sky Atmosphere(또는 프로젝트 템플릿 구성)로 낮 분위기를 만든다.
- 랜드스케이프에 레이어 머티리얼을 적용한다.

## 본문

### 2-1 액터 배치로 게임 맵 꾸미기

- **콘텐츠 브라우저**에서 Static Mesh 등을 **레벨 뷰포트**로 드래그해 배치합니다.
- **스냅(Snap)**과 **그리드**로 정렬을 맞추면 반복 배치가 쉬워집니다.
- **폴더**로 월드 아웃라이너를 정리하면 협업과 탐색이 편해집니다.

---

### 2-2 라이팅으로 게임 조명 추가

- **Directional Light**는 태양광에 가까운 평행광, **Point/Spot**은 지역 광원입니다.
- **Sky Light**, **Sky Atmosphere**, **Exponential Height Fog** 등과 조합해 야외 분위기를 만듭니다.
- 라이트의 **Movable / Stationary / Static** 이동성은 품질·빌드 시간·그림자 방식과 연결됩니다.

오류 방지(머티리얼·라이팅):
1. **언제**: 머티리얼을 저장했는데 메시가 검게 나오거나, 라이트를 움직였는데 그림자·조명이 기대와 다를 때.
2. **왜**: 컴파일 오류·미리보기 캐시 문제이거나, **Static** 라이트는 런타임에 위치가 바뀌어도 **라이트맵 빌드** 결과가 갱신되지 않아서일 수 있습니다.
3. **해결**: 머티리얼 에디터 **플랫폼 통계**에서 오류 메시지를 확인하고, 움직이는 태양·손전등에는 **Movable**(또는 Stationary)을 검토하며, 정적 조명이면 **Build → Build Lighting Only**로 맵을 다시 굽습니다.

---

### 2-3 랜드스케이프란?

- **Landscape**는 대규모 **지형**을 위한 전용 액터입니다. 높이맵·페인팅으로 디테일을 줍니다.
- 일반 Static Mesh 지형보다 **LOD**와 **편집 도구**에 최적화되어 있습니다.

약어 설명(LOD):
- **LOD**는 **Level Of Detail**의 약자입니다.
- 한국어로는 "세부 묘사 단계" 정도로 이해하면 되며, 카메라와의 거리 등에 따라 **덜 디테일하게 그릴지·더 디테일하게 그릴지**를 바꿔 **성능과 화질의 균형**을 맞출 때 씁니다.
- 랜드스케이프는 지형이 넓어 LOD·타일링 전략이 특히 중요한 경우가 많습니다.
- 지형이 넓을수록 **컴포넌트 단위 분할** 덕분에 편집과 렌더링이 효율적으로 동작하는 경우가 많습니다.

---

### 2-4 랜드스케이프로 지형 꾸미기

- **Landscape 모드**에서 브러시로 높이를 올리고 깎습니다.
- **레이어**에 풀·흙·돌 텍스처를 페인팅해 디테일을 추가합니다.
- **스무스(Smooth)** 브러시로 날카로운 경사를 완만하게 다듬으면 캐릭터 이동과 시각적 자연스러움에 도움이 됩니다.

---

### 2-5 머티리얼이란?

- **머티리얼(Material)**은 표면이 **어떻게 보일지**(색, 반사, 노멀, 투명도 등)를 정의하는 자산입니다.
- **머티리얼 에디터**에서 노드 그래프로 셰이더 로직을 만듭니다.

---

### 2-6 머티리얼로 색상 표현

- **Base Color** 입력에 상수 색 또는 텍스처 샘플을 연결합니다.
- **Constant3Vector**로 단색, **Texture Sample**로 이미지 기반 색을 씁니다.

---

### 2-7 머티리얼로 색상 애니메이션 표현

- **Time** 노드와 **Panner**, **Sine** 등을 조합해 UV나 색을 시간에 따라 변화시킬 수 있습니다.
- 인스턴스 머티리얼에서 **Scalar Parameter**로 속도를 노출하면 조정이 쉽습니다.

---

### 2-8 머티리얼로 도형 표현

- **World Position**, **Fresnel**, **Normal** 조합으로 간단한 기하 느낌·윤곽광을 만들 수 있습니다.
- **Procedural** 패턴은 UV가 단순한 프리미티브에도 디테일을 줄 때 유용합니다.

---

### 2-9 랜드스케이프 머티리얼로 바닥 질감 추가

- **Landscape Layer Blend**로 여러 레이어 텍스처를 높이·가중치에 따라 섞습니다.
- **Landscape Grass Type**으로 풀 메시를 자동 배치할 수 있습니다.

예시: 콘텐츠 브라우저에서 메시 배치(재현 절차)
1. **콘텐츠 브라우저**에서 `StarterContent` 등 폴더를 연다.
2. **Static Mesh** 자산을 레벨 **뷰포트**로 드래그한다.
3. 배치된 액터를 선택한 뒤 **디테일**에서 위치·회전을 숫자로 맞춘다.
4. **저장(Ctrl+S)**으로 레벨을 저장한다.

절차 해석:
- 드래그 앤 드롭은 내부적으로 해당 메시를 사용하는 **액터**를 레벨에 **스폰**하는 과정과 같습니다.

연습문제:
1. 문제: Static 라이트와 Movable 라이트를 각각 쓰기 좋은 상황을 한 가지씩 적으세요.
   - 입력: 해당 없음
   - 출력: 상황 2개 + 이유 한 줄씩
   - 조건: "움직이는 태양" vs "고정 조명" 중 하나를 예로 쓸 것
2. 문제: 머티리얼의 Base Color와 Roughness가 화면에 주는 차이를 비전공자에게 설명한다면?
   - 입력: 해당 없음
   - 출력: 비유 또는 일상 예 2문장 이상
   - 조건: Roughness는 "빛 반사가 흐릿한지"로 풀어쓸 것
3. 문제: 머티리얼 에디터에서 **Constant3Vector → Base Color**만 연결한 머티리얼을 만든 뒤, 큐브에 적용한 스크린샷 또는 적용 경로를 메모하세요.
   - 입력: 빈 머티리얼 자산
   - 출력: 적용된 메시 이름 + 색상 값(RGB)
   - 조건: 저장한 머티리얼 자산 이름 포함

정답 포인트:
- 맵 = 액터 배치 + 라이트 + 지형; 머티리얼 = 표면 외관; 랜드스케이프 머티리얼 = 지형용 멀티 레이어.
