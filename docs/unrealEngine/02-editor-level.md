# Chapter 02 에디터와 레벨 기본

## 학습 목표
- Unreal 에디터의 핵심 창 역할을 이해한다.
- 레벨 배치와 저장을 수행할 수 있다.

## 세부 주제
### 02-1 주요 창
- World Outliner, Details, Viewport
### 02-2 레벨 구성
- 액터 배치
### 02-3 저장/실행
- 레벨 저장과 PIE 테스트

## 실습 체크리스트
- 기본 액터를 배치하고 레벨 저장 후 실행한다.

## 본문
### 02-1 주요 창
- Outliner는 액터 목록, Details는 속성 편집 창입니다.
### 02-2 레벨 구성
- 레벨은 액터를 배치해 게임 공간을 구성하는 단위입니다.
### 02-3 저장/실행
- 저장 누락은 작업 유실 원인이므로 자주 저장해야 합니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 레벨(level) | 게임 월드의 한 공간/맵 단위 |
| 액터(actor) | 레벨에 배치되는 기본 엔티티 |

예시 코드(레벨 시작 로그):
```cpp
void ALevelHintActor::BeginPlay()
{
    Super::BeginPlay();
    UE_LOG(LogTemp, Warning, TEXT("Level Loaded"));
}
```

예상 결과:
```text
레벨에 배치된 액터가 PIE에서 정상 표시
```

코드 해석:
- 레벨 시작 시점 로그를 남겨 레벨 로드 확인에 사용합니다.
- 에디터 배치와 런타임 실행 연결을 검증할 수 있습니다.

연습문제:
1. 문제: 기본 라이트/카메라 배치
   - 입력: 없음
   - 출력: 기본 장면 구성
   - 조건: 액터 이름 정리
2. 문제: 레벨 파일 저장
   - 입력: 없음
   - 출력: 저장된 레벨 에셋
   - 힌트: 폴더 구조 통일

정답 포인트:
- 에디터 창 역할 구분이 작업 효율 핵심
- 레벨 저장 습관화 필요
