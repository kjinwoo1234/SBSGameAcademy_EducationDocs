# Chapter 04 Blueprint 기초

## 학습 목표
- Blueprint 노드 기반 개발 흐름을 이해한다.
- 이벤트 그래프로 기본 상호작용을 구현할 수 있다.

## 세부 주제
### 04-1 Blueprint 개념
- 시각 스크립팅
### 04-2 Event Graph
- BeginPlay, Tick
### 04-3 변수/함수
- 노드화된 로직

## 실습 체크리스트
- 키 입력으로 액터를 이동시키는 블루프린트를 만든다.

## 본문
### 04-1 Blueprint 개념
- Blueprint는 코딩 없이도 노드 연결로 게임 로직을 구성할 수 있는 시스템입니다.
### 04-2 Event Graph
- 이벤트 시작점에서 실행 흐름을 연결해 동작을 만듭니다.
### 04-3 변수/함수
- 변수로 상태를 저장하고 함수로 반복 로직을 재사용합니다.

약어 설명(BP):
- **BP**는 **Blueprint**의 약자입니다.
- Unreal에서 자주 쓰는 표현으로, 노드 기반 스크립팅 자산을 의미합니다.
- 초기 프로토타입 제작 속도를 높이는 데 강점이 있습니다.

예시 코드(C++ 이벤트 대응 예시):
```cpp
void AMyBPBridgeActor::BeginPlay()
{
    Super::BeginPlay();
    // BlueprintImplementableEvent를 호출해 BP 그래프와 연결 가능
    OnGameStarted();
}
```

예상 결과:
```text
입력 이벤트 연결 시 액터 이동 동작 확인
```

코드 해석:
- C++ 이벤트 지점과 Blueprint 그래프를 연결하면 하이브리드 개발이 가능합니다.
- 프로토타입은 BP, 성능 민감 로직은 C++로 분리하는 방식이 자주 사용됩니다.

연습문제:
1. 문제: 키 입력 점프 로직 연결
   - 입력: Space 키
   - 출력: 점프 동작
   - 조건: 이벤트 노드 사용
2. 문제: 체력 변수 감소 함수
   - 입력: 피해량
   - 출력: 체력 감소
   - 힌트: 함수 노드 분리

정답 포인트:
- 이벤트 시작점과 실행 흐름 연결을 명확히
- 변수/함수 분리로 그래프 복잡도 제어
