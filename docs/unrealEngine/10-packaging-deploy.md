# 패키징과 배포

패키징은 프로젝트를 실행 파일로 변환하는 마지막 단계이며, 사전 점검 품질이 결과물 안정성을 좌우합니다.

## 학습 목표
- 패키징 전 필수 점검 항목을 설명할 수 있다.
- 플랫폼별 패키징 흐름을 실행할 수 있다.
- 배포용 빌드와 개발용 빌드를 구분해 관리할 수 있다.

## 핵심
- 사전 점검: `Default Map`, 입력 설정, 프로젝트 아이콘
- 패키징: `Package Project`에서 타깃 플랫폼 선택
- 결과물 관리: 출력 폴더 구조와 빌드 타입 구분

## 단계별 실습
1. 프로젝트 설정에서 시작 맵(Default Map)을 지정한다.
2. 불필요 에셋을 정리하고 참조 오류가 없는지 확인한다.
3. Windows 플랫폼으로 패키징을 실행한다.
4. 생성된 실행 파일을 별도 환경에서 테스트한다.

## 실수 포인트
- 시작 맵 미지정으로 실행 시 검은 화면 또는 빈 레벨이 나온다.
- 개발 빌드를 그대로 배포해 디버그 정보가 노출된다.
