# Chapter 12 미니 프로젝트

## 학습 목표
- Unreal 기본 시스템을 하나의 플레이 루프로 통합한다.
- 입력/AI/UI/저장을 결합한 프로토타입을 완성한다.

## 세부 주제
### 12-1 요구사항 정의
- 핵심 플레이 루프
### 12-2 시스템 통합
- 캐릭터, 적 AI, UI
### 12-3 빌드 전 점검
- 성능/버그 체크

## 실습 체크리스트
- 3분 플레이 가능한 프로토타입을 완성한다.

## 본문
### 12-1 요구사항 정의
- 기능을 많이 넣기보다 핵심 루프를 먼저 완성하는 것이 중요합니다.
### 12-2 시스템 통합
- 캐릭터 조작, 적 반응, 점수 UI를 하나의 흐름으로 연결합니다.
### 12-3 빌드 전 점검
- 충돌 누락, UI 갱신 지연, 저장 실패 등 주요 리스크를 점검합니다.

프로젝트 체크 표:
| 항목 | 최소 완료 기준 |
|---|---|
| 플레이어 | 이동/공격 1개 이상 |
| AI | 순찰 또는 추적 1개 이상 |
| UI | 점수/체력 표시 |
| 데이터 | 저장/불러오기 1개 이상 |

예시 코드(게임 상태 스케치):
```cpp
enum class EGameState : uint8 { Ready, Playing, GameOver };
EGameState CurrentState = EGameState::Ready;
```

예상 결과:
```text
핵심 루프(입력->상호작용->결과 표시->저장)가 동작
```

코드 해석:
- 상태 열거형으로 게임 흐름을 분리하면 시스템 통합이 쉬워집니다.
- 상태 전환 규칙을 명확히 두면 버그 재현/수정이 빨라집니다.

연습문제:
1. 문제: 웨이브 생존 프로토타입 제작
   - 입력: 이동/공격 입력
   - 출력: 시간 또는 점수
   - 조건: 게임오버 처리 포함
2. 문제: QA 체크리스트 작성
   - 입력: 테스트 항목 5개
   - 출력: 통과/실패 기록
   - 힌트: 입력/AI/UI/저장 항목 포함

정답 포인트:
- 프로젝트 품질은 기능 수보다 안정성/일관성
- 최종 단계에서는 재현 가능한 테스트 기준이 중요
