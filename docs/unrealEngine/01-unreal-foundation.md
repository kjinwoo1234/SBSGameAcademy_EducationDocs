# Chapter 01 언리얼 기초 이해

## 학습 목표
- Epic Games Launcher를 통해 Unreal Engine을 설치하고 프로젝트를 생성할 수 있다.
- Blueprint와 레벨·액터의 역할을 구분해 설명할 수 있다.
- 블루프린트 **이벤트 그래프**에서 실행 흐름과 데이터 흐름을 구분하고, 자주 쓰는 **노드** 범주를 설명할 수 있다.

## 세부 주제
- 언리얼 설치 방법
- 블루프린트의 이해
- 블루프린트 그래프와 노드(실행 핀·데이터 핀, 이벤트·흐름 제어·변수·함수, 컴파일·오류)
- 레벨이란?
- 액터란?

## 실습 체크리스트
- 템플릿(예: Third Person)으로 새 프로젝트를 만들고 에디터를 실행한다.
- **Actor Blueprint**를 만들고 **Event BeginPlay** → **Print String**을 연결한 뒤 **Compile**·**PIE**로 로그 또는 화면에 문자열이 나오는지 확인한다.
- 의도적으로 **서로 다른 타입**의 핀을 연결해 컴파일 오류를 본 뒤, 올바른 타입으로 고쳐 본다.
- 레벨에 액터를 하나 배치한 뒤 PIE로 플레이한다.

## 본문

### 1-1 언리얼 설치 방법

- **Epic Games Launcher**에서 **Unreal Engine** 탭을 연 뒤 원하는 버전을 설치합니다.
- 디스크 여유 공간과 GPU 드라이버를 미리 확인합니다. 공식 문서의 **권장 사양**을 참고합니다.
- 첫 실행 시 **콘텐츠 예제**나 **Starter Content** 포함 여부를 프로젝트 생성 화면에서 선택할 수 있습니다.

오류 방지(설치·런처):
1. **언제**: 디스크 여유가 부족하거나 GPU 드라이버가 오래됐을 때 설치가 중단되거나 에디터가 실행 직후 종료될 수 있습니다.
2. **왜**: 엔진 본체·콘텐츠 용량이 크고, 뷰포트는 GPU에 의존하기 때문입니다.
3. **해결**: Epic Launcher의 **라이브러리**에서 해당 버전 옆 **옵션**으로 검증(Verify)을 돌리고, 공식 문서의 **하드웨어·소프트웨어 요구 사항**을 확인한 뒤 여유 공간을 확보합니다.

용어 설명:
| 용어 | 의미 |
|------|------|
| Epic Games Launcher | 언리얼 엔진과 마켓 콘텐츠를 받는 공식 실행기 |

---

### 1-2 블루프린트의 이해

- **Blueprint**는 노드를 연결해 로직을 만드는 **비주얼 스크립팅** 방식입니다. C++ 없이도 프로토타입과 게임플레이를 빠르게 구성할 수 있습니다.
- **Blueprint 클래스**는 부모 클래스(`Actor`, `Pawn` 등)를 상속해 이벤트·함수·변수를 정의합니다.
- C++와 **혼용**할 수 있어, 성능이 중요한 부분은 C++로 두고 기획·연출은 Blueprint로 두는 식이 흔합니다.

---

### 1-3 블루프린트 그래프와 노드

- **이벤트 그래프(Event Graph)** 는 노드를 배선(wiring, 핀과 핀을 선으로 연결하는 것)해 **언제 무엇을 할지**를 적는 작업 공간입니다. **디테일** 패널이 값을 고정해 두는 편이라면, 그래프는 **시간에 따라 바뀌는 동작**을 담당합니다.
- **노드(Node)** 는 하나의 처리 덩어리입니다. **이벤트 노드**(예: **Event BeginPlay**)는 “그 시점에 실행을 시작한다”는 **출발점**이 되는 경우가 많습니다.
- **실행 핀**(보통 **흰색 삼각형** 핀)은 **실행 흐름**이 지나가는 길입니다. 실행 핀끼리 연결된 **한 줄기**를 따라 엔진이 위에서 아래(또는 분기)로 노드를 순서대로 실행합니다.
- **데이터 핀**(색이 있는 원형 핀)은 **값**(숫자, 벡터, 객체 참조 등)을 옮깁니다. 데이터 핀은 **타입**(형식)이 맞아야 연결되며, 타입이 맞지 않으면 연결이 안 되거나 컴파일 시 **붉게** 표시되는 경우가 많습니다.

| 구분 | 실행 핀 | 데이터 핀 |
|------|---------|-----------|
| 역할 | “지금 이 노드를 실행할 차례” 신호 전달 | 숫자·참조·구조체 등 **값** 전달 |
| 연결감 | 흰색 선으로 이어짐 | 색 있는 선(타입별 색)으로 이어짐 |
| 입문 팁 | **시작 이벤트**에서 첫 **실행 핀**을 꼭 연결 | **변환 노드**(Cast 등)로 타입을 맞추는 경우가 많음 |

- **흐름 제어** 노드로 **Branch**(조건에 따라 갈림), **Sequence**(한 실행을 여러 갈래로 순서대로 실행) 등을 써서 “만약 ~면 / 먼저 A 다음 B”를 표현합니다.
- **변수 Get/Set**과 **함수 호출(Call Function)** 노드로 상태를 읽고 바꾸며, 같은 그래프·다른 액터의 함수를 부를 수 있습니다.
- 툴바 **Compile**로 그래프를 검증합니다. 오류가 있으면 **붉은 노드·메시지**로 알려 주므로, 그 핀부터 타입과 실행 선이 끊기지 않았는지 확인합니다.

오류 방지(블루프린트 컴파일):
1. **언제**: **Compile** 후에도 실행이 안 되거나, 노드·핀이 빨갛게 남을 때.
2. **왜**: **실행 핀**이 시작 이벤트에서 이어지지 않았거나, **데이터 핀** 타입이 맞지 않거나, 참조할 액터·컴포넌트가 **유효하지 않을** 수 있기 때문입니다.
3. **해결**: **Event BeginPlay**(또는 해당 이벤트)에서 **실행 선**이 끊기지 않았는지 따라가고, 붉은 핀의 **툴팁·컴파일러 결과**를 읽은 뒤 Cast·형 변환·노드 교체로 타입을 맞춥니다.

예시: 첫 블루프린트 메시지(재현 절차)
1. **콘텐츠 브라우저**에서 **우클릭 → Blueprint Class**로 **Actor**를 부모로 한 블루프린트를 만듭니다.
2. 더블클릭해 **Event Graph**를 열고, 빈 곳을 **우클릭**해 **Event BeginPlay**를 둡니다.
3. **Event BeginPlay**의 **실행 핀**에서 선을 끌어 **Print String** 노드의 실행 입력에 연결합니다.
4. **Print String**의 **In String**에 원하는 문구를 적고 **Compile** → 레벨에 이 액터를 배치한 뒤 **PIE**로 실행해 화면 왼쪽 위(기본 설정)에 글자가 뜨는지 봅니다.

절차 해석:
- **BeginPlay**는 액터가 레벨에 나타난 뒤 한 번(또는 그에 준하는 타이밍) 호출되는 **진입점**이라, 테스트 출력을 찍기에 적합합니다.

---

### 1-4 레벨이란?

- **레벨(Level)**은 게임 월드의 한 **장면**입니다. 배치된 액터, 라이팅, 네비메시, 스카이 등이 저장됩니다.
- **월드 아웃라이너(World Outliner)**에서 현재 레벨에 있는 객체 목록을 볼 수 있습니다.
- 레벨 간 **이동(Open Level, 스트리밍)**으로 타이틀 → 인게임 → 결과 화면을 나눌 수 있습니다.

---

### 1-5 액터란?

- **액터(Actor)**는 월드에 **배치 가능한 기본 단위**입니다. 스스로 기능이 없을 수 있고, **컴포넌트**를 붙여 움직임·렌더·충돌 등을 만듭니다.
- **루트 컴포넌트**가 있는 액터는 그 컴포넌트의 변환(위치·회전·스케일)을 기준으로 움직입니다.
- **월드 아웃라이너**에서 액터를 선택하면 **디테일** 패널로 해당 액터·컴포넌트 속성을 한곳에서 조정할 수 있습니다.

약어 설명(PIE):
- **PIE**는 **Play In Editor**의 약자입니다.
- 한국어로는 보통 "에디터 내 플레이"라고 부르며, 빌드 없이 현재 레벨을 바로 실행해 동작을 확인하는 모드입니다.
- 레벨을 자주 저장하지 않은 채 실험할 때 빠른 피드백을 받기 위해 쓰며, **Stop**으로 즉시 에디터로 돌아올 수 있습니다.

예시: PIE 재현 절차(에디터)
1. 프로젝트를 연 뒤 기본 레벨이 열려 있는지 확인합니다.
2. 툴바에서 **Play** 버튼(삼각형)을 누릅니다.
3. 뷰포트가 게임 뷰로 바뀌면 WASD 등으로 플레이어를 움직여 봅니다.
4. **Esc** 또는 툴바 **Stop**으로 실행을 종료합니다.

절차 해석:
- PIE는 패키징된 빌드가 아니라 **에디터가 시뮬레이션**하는 것이므로, 일부 설정(네트워크·성능)은 최종 빌드와 다를 수 있습니다.

연습문제:
1. 문제: Blueprint만으로 가능한 작업과 C++가 유리한 작업을 각각 한 가지씩 적어 보세요.
   - 입력: 해당 없음
   - 출력: 각 한 줄 이상 설명
   - 조건: 실제 본인이 해본 경험을 한 문장 포함
2. 문제: 레벨과 액터의 관계를 "집과 가구"에 비유해 한 문장으로 설명해 보세요.
   - 입력: 해당 없음
   - 출력: 비유 문장 1개
   - 조건: 레벨=공간, 액터=배치 객체가 드러나야 함
3. 문제: **실행 핀**과 **데이터 핀**의 차이를 각각 한 문장으로 쓰고, **Branch** 노드가 실행 핀을 두 개(True/False) 갖는 이유를 한 문장으로 설명해 보세요.
   - 입력: 해당 없음
   - 출력: 실행 핀 설명 1문장 + 데이터 핀 설명 1문장 + Branch 이유 1문장
   - 조건: 데이터 핀에 "값·타입" 중 하나 포함
4. 문제: PIE와 **Standalone Game**(에디터 메뉴에서 실행)의 차이를 공식 문서 또는 에디터 도움말로 검색·정리해 적어 보세요.
   - 입력: 공식 문서 또는 에디터 도움말
   - 출력: 차이 요약 2~3문장
   - 조건: "에디터 붙어 있음/분리 실행" 중심으로 쓸 것

정답 포인트:
- 설치 = Launcher + 엔진 버전; Blueprint = 시각적 스크립트; **노드 = 실행·데이터 핀으로 배선**; 레벨 = 장면; 액터 = 월드에 놓이는 객체.
