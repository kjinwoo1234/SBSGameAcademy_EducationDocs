# 트랙 종합 미니프로젝트

앞선 1~10 문서에서 배운 내용을 하나로 묶어 작은 완성형 게임을 만드는 단계입니다.

## 학습 목표
- 입력, 충돌, UI, 애니메이션, 데이터 저장을 하나의 흐름으로 통합할 수 있다.
- 블루프린트 중심 게임 루프(시작 -> 플레이 -> 결과 -> 재시작)를 구현할 수 있다.
- 빌드 가능한 프로젝트 단위로 결과물을 정리할 수 있다.

## 프로젝트 주제 예시
- **주제:** 코인 수집 타임어택
- **핵심 규칙:** 제한 시간 안에 맵의 코인을 최대한 많이 획득한다.

## 최소 구현 요구사항 (MVP)
- 플레이어 이동: 키 입력으로 3D 이동 가능
- 상호작용: 코인과 Overlap 시 점수 증가
- UI: 남은 시간/점수 표시, 게임 종료 메시지 출력
- 애니메이션: Idle/Run 상태 전이 적용
- 데이터: 최고 점수 SaveGame 저장/불러오기

## 단계별 실습
1. 기본 레벨과 플레이어 캐릭터를 배치한다.
2. 코인 Actor Blueprint를 만들고 Overlap 시 파괴/점수 증가 로직을 연결한다.
3. UMG 위젯으로 타이머와 점수 UI를 구성한다.
4. Animation Blueprint에 이동 상태 전이를 연결한다.
5. 게임 종료 조건(시간 0초)을 만들고 결과 UI를 표시한다.
6. 최고 점수를 SaveGame으로 저장/로드한다.
7. Windows 패키징으로 실행 파일을 생성한다.

## 제출 체크리스트
- [ ] Play 시 기본 이동과 카메라 조작이 정상 동작하는가?
- [ ] 코인 획득 시 점수 UI가 즉시 갱신되는가?
- [ ] 시간이 끝나면 게임 종료 상태로 전환되는가?
- [ ] 재실행 후 최고 점수가 유지되는가?
- [ ] 패키징된 실행 파일에서 동일하게 동작하는가?

## 확장 과제 (선택)
- 난이도 단계(쉬움/보통/어려움) 추가
- 스폰 시스템으로 코인 위치 랜덤화
- 결과 화면에 플레이 통계(획득 코인 수, 평균 획득 간격) 출력

## 셀프 퀴즈

2. 최고 점수처럼 실행 후에도 유지해야 하는 데이터는 어디에 저장하는가?
