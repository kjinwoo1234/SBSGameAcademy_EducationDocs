# 빌드와 배포

프로젝트를 실행 파일로 만드는 마지막 단계입니다.

## 학습 목표
- Build Settings에서 플랫폼별 빌드를 수행할 수 있다.
- 빌드 전 필수 점검 항목을 적용할 수 있다.
- 배포용 빌드에서 발생하는 대표 이슈를 사전 예방할 수 있다.

## 기본 절차
1. `File > Build Settings` 열기
2. 플랫폼 선택 (PC, Android 등)
3. `Add Open Scenes`로 씬 등록
4. `Build` 또는 `Build And Run`

## 체크리스트
- 첫 씬(메인 메뉴/로비) 인덱스 확인
- 필요한 리소스 누락 여부 확인
- 플랫폼별 입력/해상도 테스트
- Player Settings의 회사명/제품명/아이콘 확인
- 개발용 로그/디버그 UI 비활성화 확인

## 미니 실습 과제
1. PC 플랫폼으로 `Build`를 수행합니다.
2. 빌드 결과물 실행 후 씬 전환/입력/UI를 테스트합니다.
3. 버전 번호를 변경하고 재빌드해 갱신 여부를 확인합니다.

## 실수 포인트
- Build Settings에 씬이 빠져 게임이 시작되지 않는 문제
- 플랫폼 전환 후 입력 설정 누락
- 배포 빌드에 테스트용 오브젝트/로그를 남겨두는 문제

## 이해 점검 체크리스트
- [ ] 시작 씬이 Build Index 0으로 설정되었는가?
- [ ] 타겟 플랫폼에서 입력/해상도를 검증했는가?
- [ ] 배포 전 불필요 디버그 요소를 제거했는가?
