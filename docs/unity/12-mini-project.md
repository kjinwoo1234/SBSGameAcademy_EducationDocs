# Chapter 12 미니 프로젝트

## 학습 목표
- Unity 기초 기능을 하나의 게임 루프로 통합한다.
- 입력/물리/UI/저장 시스템을 함께 적용한다.

## 세부 주제
### 12-1 요구사항 정의
- 핵심 루프 선정
### 12-2 시스템 통합
- 플레이어/적/UI/점수
### 12-3 최종 점검
- 빌드 전 체크

## 실습 체크리스트
- 1분 플레이 가능한 프로토타입을 완성한다.

## 본문
### 12-1 요구사항 정의
- "이동-회피-점수" 같은 단순 루프부터 완성하는 것이 중요합니다.
### 12-2 시스템 통합
- 입력, 충돌, UI, 저장 기능을 느슨하게 연결해 확장성을 확보합니다.
### 12-3 최종 점검
- 플레이 테스트 체크리스트로 버그를 줄이고 조작감을 보정합니다.

프로젝트 체크 표:
| 항목 | 최소 완료 기준 |
|---|---|
| 플레이어 제어 | 이동/충돌 반응 동작 |
| UI | 점수/게임오버 표시 |
| 데이터 | 최고 점수 저장/복원 |
| 피드백 | 효과음 또는 파티클 1개 이상 |

예시 코드(게임 상태 전환 스케치):
```csharp
public enum GameState { Ready, Playing, GameOver }
public class GameFlow : MonoBehaviour
{
    public GameState State { get; private set; } = GameState.Ready;
    public void StartGame() => State = GameState.Playing;
    public void EndGame() => State = GameState.GameOver;
}
```

예상 결과:
```text
입력 -> 상호작용 -> 점수 증가 -> 저장까지 한 흐름이 동작
```

코드 해석:
- 챕터 목표는 "새 문법"보다 "시스템 연결 능력"입니다.
- 기능을 작게 완성하고 반복 개선하는 방식이 가장 안정적입니다.
- 상태 enum으로 게임 흐름을 명확히 분리하면 UI/입력/저장 연계가 쉬워집니다.

연습문제:
1. 문제: 장애물 회피 게임 프로토타입 제작
   - 입력: WASD/방향키
   - 출력: 생존 시간/점수
   - 조건: 게임오버 처리 포함
2. 문제: 빌드 전 점검표 작성
   - 입력: 테스트 항목 5개
   - 출력: 점검 결과 체크리스트
   - 힌트: 입력/충돌/UI/저장/사운드 항목 포함

정답 포인트:
- 완성도는 기능 수보다 안정성/일관성이 좌우
- 체크리스트 기반 검증이 최종 품질 핵심
