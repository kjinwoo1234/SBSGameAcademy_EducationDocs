# 포트폴리오 커리큘럼

포트폴리오 제작을 목표로, 학생별 수준 진단부터 게임 개발·정리까지 진행하는 실전형 커리큘럼입니다.

---

## 1) 수업 진행 방식

| 단계 | 내용 | 산출물 |
|---|---|---|
| 1 | 학생별 수준 진단 및 목표 설계 | 개인 목표 시트, 기술 우선순위 |
| 2 | 포트폴리오용 게임 개발 | 플레이 가능한 게임 빌드 |
| 2-1 (선택) | 포트폴리오용 Unity 게임 개발 | Unity 프로젝트/시연 영상 |
| 2-2 (선택) | 포트폴리오용 Unreal 게임 개발 | Unreal 프로젝트/시연 영상 |
| 3 | Notion 정리 및 아카이빙 | 포트폴리오 문서 페이지 |

> 목표 포트폴리오 예시: [YouTube 참고 영상](https://www.youtube.com/watch?v=6INjf4HjQSA&t=133s)

## 2) 게임 개발 주제

아래 항목을 자신의 게임 기획에 맞게 조합하여 포트폴리오 완성도를 높입니다.

### 2-1. 캐릭터 애니메이션

| 대분류 | 세부 항목 | 구현 포인트 |
|---|---|---|
| Locomotion | 대기 애니메이션 | 기본 상태 전환의 기준 포즈 구축 |
| Locomotion | 보행 및 주행 | 속도 기반 블렌딩 |
| Locomotion | 스프린트 | 가속/감속 리듬과 카메라 연출 |
| Locomotion | 앉기/웅크리기 | 자세 전환 시 콜리전/속도 제어 |
| Locomotion | 점프 및 착지 | 점프 시작-공중-착지 상태 분리 |
| Locomotion | Strafe 이동 | 정면 주시 상태에서 좌우/대각 이동 |
| Locomotion | Turn In Place | 시야 방향 기준 제자리 회전 |
| Locomotion | Pivot Turn | 급격한 방향 전환 시 자연스러운 전환 |
| 액션/전투 | 근접 공격/투사체/스킬 시전 | 공격 타입별 애니메이션 분기 |
| 액션/전투 | 총기 재장전 | 상태 제한(사격 불가)과 연동 |
| 액션/전투 | Hit 리액션/Flinch | 피격 방향/강도별 반응 분리 |
| 액션/전투 | Dodge/Roll | 무적 프레임/거리 조절 |
| 액션/전투 | Guard/Block | 방어 판정과 스태미나 연동 |
| 액션/전투 | 히트 스톱/카메라 쉐이크 | 타격감 강화 연출 |
| 상호작용 | 아이템 줍기 | 상호작용 트리거와 동기화 |
| 상호작용 | 기믹 애니메이션 | 문/레버 등 월드 오브젝트 연동 |
| 상호작용 | 파쿠르 애니메이션 | 오르기/내리기 등 고난도 이동 |
| 상호작용 | 탈것 승하차 | 상태 전환 및 카메라 전환 |
| 협동/동기화 | 처형 애니메이션 | 상대 위치 정렬 및 타이밍 동기화 |
| 협동/동기화 | Revive 애니메이션 | 다운 상태 복구 흐름 |
| 협동/동기화 | Co-op Traversal | 2인 상호작용 이벤트 처리 |
| 협동/동기화 | Takedown | 잠입 제압 연쇄 동작 |
| 소셜/감정 | Emote | 감정 표현 및 입력 매핑 |
| 소셜/감정 | 입술 동기화 | 대사 오디오와 립싱크 동기화 |
| 절차적 애니메이션 | 시선 추적(고개/눈) | 타겟 기반 본 제어 |
| 절차적 애니메이션 | Foot IK | 경사/계단 지면 보정 |
| 절차적 애니메이션 | Ragdoll | 사망/충돌 시 물리 전환 |
| 애니메이션 레이어링 | 상하체 분리 동작 | 이동 중 상체 액션 유지 |
| 물리 기반 애니메이션 | 애니메이션+물리 혼합 | 비틀거림 등 하이브리드 연출 |
| 물리 기반 애니메이션 | 모션 워핑 | 타겟 거리/각도 보정으로 정확도 향상 |
