# Chapter 06 반복문

## 학습 목표
- `while`로 조건에 쓸 변수를 만들고, 조건을 검사하고, 본문에서 그 조건을 바꾸는 흐름을 실행으로 확인한다.
- `do-while`과 `while`이 조건을 언제 검사하는지 구분한다.
- `for`가 위 세 단계를 한 줄에 모아 둔 형태임을 이해하고, `break`와 `continue`를 구분해 쓴다.

## 본문

### 06-1 `while`

**반복문**은 같은 작업을 조건이 참인 동안 되풀이하는 구조입니다. **`while`**은 그 조건을 앞에 두고, 참일 때만 중괄호 안을 실행합니다.

`while`을 쓰려면 보통 세 가지를 따로 적습니다. 먼저 조건에 쓸 변수를 만들고, 그다음 `while`로 조건을 검사하고, 마지막으로 본문 안에서 그 변수를 바꿔 언젠가 조건이 거짓이 되게 합니다. 본문에서 변수를 바꾸지 않으면 조건이 영원히 참이라 **무한 루프**에 빠집니다.

끝이 “몇 번 돌릴지”보다 입력이나 상태에 달려 있을 때 `while`이 잘 맞습니다. 작성하기 전에 언제 끝나는지를 한 줄로 먼저 적어 두면 실수를 줄일 수 있습니다.

반복에서 자주 쓰는 **증감**은 `n--`와 `i++`입니다. `n--`는 지금 값을 쓴 뒤 1을 감소하고 `n = n - 1`과 같고, `i++`는 1을 증가하며 `i = i + 1`과 같습니다. 앞 장에서 본 `+=`처럼 값을 조금씩 바꾸는 계열입니다.

아래를 `Main`에 넣고 실행해 보세요. `n--`를 주석 처리하면 끝나지 않는지도 확인해 보면 좋습니다.

```csharp
using System;

class Program
{
    static void Main()
    {
        int n = 5; // 조건에 쓸 변수
        Console.Write("while 카운트다운: ");
        while (n > 0) // 조건 검사
        {
            Console.Write($"{n} ");
            n--; // 본문에서 조건 변화
        }
        Console.WriteLine();
    }
}
```

**예상 출력**

```text
while 카운트다운: 5 4 3 2 1
```

**코드 해석**
- `n`을 만든 뒤, `n > 0`이 참인 동안만 출력합니다.
- 매 회차 `n--`로 `n`을 줄여, 언젠가 조건을 거짓으로 만듭니다.
- `n--`가 없으면 `n`이 5로 남아 끝나지 않습니다.

### 06-2 `do-while` — `while`과의 차이

**`do-while`**도 조건이 참인 동안 반복하지만, 조건을 검사하는 시점이 `while`과 다릅니다. `while`은 조건을 먼저 보고 참일 때만 본문을 실행합니다. `do-while`은 본문을 먼저 한 번 실행한 뒤 조건을 봅니다.

| | `while` | `do-while` |
|---|---|---|
| 순서 | 조건 먼저, 참이면 본문 | 본문 먼저, 그다음 조건 |
| 최소 실행 | 조건이 처음부터 거짓이면 0회 | 조건과 관계없이 최소 1회 |
| 문법 | `while (조건) { }` | `do { } while (조건);` — 끝에 세미콜론 |

메뉴를 한 번 보여 주고 입력을 받는 것처럼, 본문이 최소 한 번은 실행되어야 할 때 `do-while`을 자주 씁니다.

아래를 실행해 보세요. 종료 번호인 `2`를 입력할 때까지 메뉴가 반복되는지 확인합니다. 연습할 때는 숫자만 입력하세요.

```csharp
using System;

class Program
{
    static void Main()
    {
        int menu = 0;

        do
        {
            Console.WriteLine("메뉴: 1. 시작  2. 종료");
            Console.Write("입력: ");
            menu = int.Parse(Console.ReadLine());
        } while (menu != 2);

        Console.WriteLine("프로그램 종료");
    }
}
```

**예상 출력** — 입력으로 `1`을 넣고 이어서 `2`를 넣은 경우

```text
메뉴: 1. 시작  2. 종료
입력: 1
메뉴: 1. 시작  2. 종료
입력: 2
프로그램 종료
```

**코드 해석**
- `do` 블록이 먼저 한 번 실행된 뒤, `while (menu != 2)`로 계속할지 판단합니다.
- `while`만 쓰고 `menu`가 처음부터 `2`라면 메뉴를 한 번도 안 보여줄 수 있습니다. `do-while`은 그 경우에도 최소 한 번은 보여 줍니다.
- `while (조건);` 끝의 세미콜론을 빠뜨리지 마세요.

### 06-3 `for` — 세 조각을 한곳에

`while`에서 흩어 두었던 세 단계를 다시 보면 아래와 같습니다.

```csharp
int i = 1;        // 조건에 쓸 변수 만들기
while (i <= 5)    // 조건 검사
{
    // ... 할 일 ...
    i++;          // 본문에서 조건 변화
}
```

**`for`**는 이 세 단계를 괄호 안 한곳에 모아 둡니다. “1부터 N까지”처럼 횟수나 범위가 분명할 때 읽기 쉽습니다.

```csharp
for (int i = 1; i <= 5; i++)
//    변수 만들기   검사     변화
{
    // ... 할 일 ...
}
```

| `for`의 칸 | `while`에서 하던 일 |
|---|---|
| 첫 칸 `int i = 1` | 루프 앞에서 변수 만들기 |
| 둘째 칸 `i <= 5` | `while`의 조건 검사 |
| 셋째 칸 `i++` | 본문 끝에서 조건 바꾸기 |

칸과 칸은 세미콜론으로 나눕니다. 형태는 `for (초기; 조건; 증감)`입니다.

아래를 `Main`에 넣고 실행해 보세요. `while`로 같은 합을 만든 결과와 출력이 같은지 비교합니다.

```csharp
using System;

class Program
{
    static void Main()
    {
        int sumWhile = 0;
        int i = 1;
        while (i <= 5)
        {
            sumWhile += i;
            i++;
        }
        Console.WriteLine($"1~5 합(while): {sumWhile}");

        int sumFor = 0;
        for (int j = 1; j <= 5; j++)
        {
            sumFor += j;
        }
        Console.WriteLine($"1~5 합(for): {sumFor}");
    }
}
```

**예상 출력**

```text
1~5 합(while): 15
1~5 합(for): 15
```

**코드 해석**
- `while` 쪽은 변수, 검사, `i++`가 여러 줄에 흩어져 있습니다.
- `for` 쪽은 `for (int j = 1; j <= 5; j++)` 한 줄에 같은 세 단계를 모았습니다. 결과는 같습니다.
- 증감이나 조건을 잘못 쓰면 `while`과 같이 무한 루프가 납니다.

세 가지를 한 표로 정리하면 아래와 같습니다.

| 방식 | 실행 시점 | 적합한 상황 | 주의점 |
|---|---|---|---|
| `while` | 조건 확인 후 실행 | 끝이 입력·상태에 달려 있을 때 | 본문에서 조건 갱신 누락 |
| `do-while` | 본문 1회 실행 후 조건 확인 | 최소 1회가 필요한 메뉴·입력 | 끝 `while (조건);` |
| `for` | 초기 → 조건 → 본문 → 증감 | 횟수·범위가 분명할 때 | 세 칸·세미콜론 실수 |

### 06-4 흐름 제어

반복 안에서 이번 회차만 건너뛰고 다음으로 가려면 **`continue`**를 쓰고, 반복 자체를 즉시 끝내려면 **`break`**를 씁니다. `continue`는 `for`에서 남아 있던 본문을 건너뛰고 증감과 조건 검사로 돌아가게 합니다. `break`는 조건이 아직 참이어도 루프를 벗어납니다.

둘 다 보통 앞 장에서 배운 `if`와 함께 씁니다. 앞 장 `switch`의 `break`는 반복을 끝내는 것이 아니라, 해당 `case` 처리를 마치고 `switch`를 빠져나가는 뜻입니다. 루프인지 `switch`인지 문맥을 먼저 보세요.

`continue` 위치를 잘못 잡으면 합계나 출력이 빠질 수 있습니다. 건너뛸 조건을 검사한 뒤에는, 스킵해도 되는 코드만 `continue` 아래에 두세요.

아래 루프를 실행해, `continue`로 건너뛴 수와 `break`로 끝난 지점을 출력에서 확인해 보세요.

```csharp
using System;

class Program
{
    static void Main()
    {
        for (int i = 1; i <= 10; i++)
        {
            if (i % 3 == 0)
            {
                continue; // 3의 배수는 출력 생략
            }
            if (i >= 8)
            {
                break; // 8 이상이면 반복 종료
            }
            Console.Write($"{i} ");
        }
        Console.WriteLine();
    }
}
```

**예상 출력**

```text
1 2 4 5 7
```

**코드 해석**
- `3`과 `6`은 `continue`로 출력하지 않습니다.
- `8`에서 `break`로 끝나므로 `8`, `9`, `10`은 나오지 않습니다.

배열 순회와 `foreach`는 다음 장에서 이어갑니다.

### 연습문제

문제 1·2·3은 필요하면 앞 장처럼 `int.Parse(Console.ReadLine())`을 쓰고, 이 장에서 배운 순서대로 풀면 됩니다.

**문제 1**
- 문제: 정수 `n`을 입력받아, `while`로 `n`부터 1까지 카운트다운을 한 줄에 출력하세요.
- 입력: 정수 1개. 예: `4`
- 출력: `4 3 2 1` 형태. 끝 공백은 허용합니다.
- 조건: `while` 사용. 변수 만들기, 조건 검사, 본문에서 감소를 모두 둘 것

**문제 2**
- 문제: `do-while`로 메뉴를 한 번 이상 보여 주고, `3`을 입력하면 종료하세요.
- 입력: 메뉴 번호를 여러 번. 예: `1` 다음에 `3`
- 출력: 매 회차 `메뉴: 1. 시작  2. 설정  3. 종료`와 `입력: ` 안내 후, 끝에 `프로그램 종료`
- 조건: `do-while` 사용. `while (조건);` 끝 세미콜론 포함. 본문 예제는 `2`에서 끝나므로, 종료 번호와 메뉴 문구를 다르게 둘 것

**문제 3**
- 문제: 단을 2에서 9 사이로 입력받아 해당 구구단을 `for`로 출력하세요.
- 입력: 정수 1개. 예: `3`
- 출력: 곱셈식 9줄. 예: `3 x 1 = 3`부터 `3 x 9 = 27`까지
- 조건: `for` 사용. 초기, 조건, 증감을 한곳에 둘 것

### 정답 포인트

- 문제 1: `int n = int.Parse(...);` 후 `while (n > 0) { Console.Write($"{n} "); n--; }`
- 문제 2: `do { 안내와 입력 } while (menu != 3);` — 본문이 조건보다 먼저, 끝의 `);` 유지
- 문제 3: `for (int i = 1; i <= 9; i++)` 안에서 `$"{dan} x {i} = {dan * i}"`
- 공통: `while`의 세 단계가 `for` 한 줄과 같은 역할인지 먼저 짚기
