# C# 상속

상속은 기존 클래스의 속성과 기능을 재사용하는 메커니즘입니다.  
중복 코드를 줄이고 타입 계층을 설계할 수 있습니다.

## 학습 목표
- 상속의 기본 목적(재사용/확장)을 설명할 수 있다.
- 부모/자식 클래스 관계를 코드로 구현할 수 있다.
- `base` 사용 시점을 이해하고 적용할 수 있다.

## 핵심
- 부모 클래스: 공통 속성/동작 제공
- 자식 클래스: 기존 기능 재사용 + 확장
- `base`: 부모 생성자/멤버에 명시 접근

## 1) 기본 상속
```csharp
class Person
{
    public string Name { get; set; }
    public void Eat() => Console.WriteLine("식사");
}

class Student : Person
{
    public void Study() => Console.WriteLine("공부");
}
```

## 2) `base` 키워드
부모 생성자/멤버를 명시적으로 호출할 때 사용합니다.

```csharp
class Student : Person
{
    public Student(string name) { Name = name; }
}
```

## 실수 포인트
- 상속이 필요한 상황이 아닌데도 계층을 과도하게 늘린다.
- 부모 공통 기능을 중복 구현해 상속 장점을 놓친다.
- `base` 호출 시점/목적을 혼동한다.

## 단계별 실습
1. `Person`에 공통 속성/메서드를 선언한다.
2. `Student`가 `Person`을 상속받도록 구성한다.
3. 자식 클래스 고유 메서드(`Study`)를 추가한다.
4. 부모 기능 호출과 자식 기능 호출을 각각 확인한다.
