# Chapter 09 클래스와 객체

## 학습 목표
- 클래스와 객체의 관계를 이해한다.
- 필드/메서드로 상태와 동작을 설계할 수 있다.
- **생성자**로 초기 상태를 설정하고, `public`/`private`로 접근 범위를 구분한다.

## 본문

### 09-1 클래스 기본

**클래스(class)**는 데이터(필드)와 기능(메서드)을 하나로 묶는 **설계도**입니다. **객체(object)**는 그 설계도로 만든 **실제 인스턴스**입니다. 예: `Player` 클래스가 있고, `new Player(100)`으로 체력 100인 플레이어 객체를 만듭니다. 필드는 상태(체력 등), 메서드는 동작(`Damage` 등)을 담습니다.

### 09-2 생성자

**생성자(constructor)**는 클래스와 같은 이름의 특수 메서드로, `new`로 객체를 만들 때 **자동 호출**됩니다. `public Player(int hp)`처럼 초기값을 받아 필드를 안전하게 설정합니다. 생성자가 있으면 “빈 객체 뒤에 필드를 하나씩 대입”하는 실수를 줄일 수 있습니다.

### 09-3 접근 제한자

**접근 제한자**는 멤버를 어디서 쓸 수 있는지 정합니다. **`public`**은 클래스 밖에서도 호출·접근할 수 있고, **`private`**는 **같은 클래스 안**에서만 씁니다. 필드는 `private`로 감추고, 필요한 읽기·변경만 `public` 메서드로 열어 두는 방식이 **캡슐화**의 기본입니다.

| 제한자 | 접근 가능 범위 | 이럴 때 |
|---|---|---|
| `public` | 클래스 밖에서도 사용 | 외부에 공개할 메서드·생성자 |
| `private` | 같은 클래스 내부만 | 직접 만지면 위험한 필드 |

| 용어 | 의미 |
|---|---|
| 인스턴스(instance) | 클래스로부터 생성된 실제 객체 |
| 캡슐화(encapsulation) | 데이터와 동작을 묶고 내부 상태를 보호하는 설계 방식 |

아래 코드를 실행해 `new`·생성자·`Damage` 후 체력 출력을 확인해 보세요.

```csharp
using System;

class Player
{
    private int hp;

    public Player(int hp)
    {
        this.hp = hp;
    }

    public void Damage(int d)
    {
        hp -= d;
    }

    public int GetHp()
    {
        return hp;
    }
}

class Program
{
    static void Main()
    {
        Player p = new Player(100);
        p.Damage(15);
        Console.WriteLine(p.GetHp());
    }
}
```

**예상 출력**

```text
85
```

**코드 해석**
- `new Player(100)`이 생성자를 호출해 `hp`를 `100`으로 둡니다.
- `Damage(15)`가 `private` 필드 `hp`를 `85`로 바꿉니다.
- 밖에서는 `hp`에 직접 손대지 못하고 `GetHp()`로만 읽습니다.

### 연습문제

**문제 1**
- 문제: 이름과 체력을 가지는 `Enemy` 클래스를 작성하세요.
- 입력: 없음(코드에서 `new Enemy("슬라임", 30)` 등)
- 출력: 예: `슬라임 HP:30` (이름·체력을 한 줄로 출력)
- 조건: 생성자로 이름·체력 초기화

**문제 2**
- 문제: 체력을 감소시키는 `Hit` 메서드를 구현하세요.
- 입력: 없음(예: 체력 `30`인 적에 `Hit(10)`)
- 출력: `20`
- 조건: 체력 필드는 `private` 유지, `Hit`는 `public`

### 정답 포인트

- 문제 1: 필드 + `public Enemy(string name, int hp)` 생성자 + 출력 메서드
- 문제 2: `public void Hit(int dmg) { hp -= dmg; }` 후 체력 출력
- 공통: 객체 생성은 `new`, 필드는 `private` + 메서드로 제어
