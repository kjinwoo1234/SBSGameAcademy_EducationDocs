# Chapter 12 예외 처리

## 학습 목표
- `try-catch-finally` 흐름을 이해한다.
- 예외 메시지와 오류 복구 전략의 기초를 익힌다.

## 세부 주제
### 12-1 예외 기본
- 예외 발생과 전파

### 12-2 `try-catch-finally`
- 처리/정리 분리

### 12-3 사용자 정의 예외
- `throw` 활용

## 실습 체크리스트
- 잘못된 입력 상황을 예외 처리로 안전하게 처리한다.

## 본문

### 12-1 예외 기본
- 예외는 실행 중 발생하는 오류를 구조적으로 처리하는 메커니즘입니다.
- 예외를 무시하면 프로그램이 비정상 종료될 수 있습니다.

### 12-2 `try-catch-finally`
- `try`에서 실행, `catch`에서 처리, `finally`에서 정리 코드를 실행합니다.
- 파일/네트워크 자원은 `finally` 또는 `using`으로 정리합니다.

### 12-3 사용자 정의 예외
- 정책 위반 상황을 `throw new Exception(...)`으로 명확히 알릴 수 있습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 예외 전파 | 처리되지 않은 예외가 호출 스택 상위로 전달되는 동작 |
| `finally` | 예외 발생 여부와 관계없이 마지막에 실행되는 정리 구문 |

예시 코드:
```csharp
using System;

class Program
{
    static void Main()
    {
        try
        {
            int n = int.Parse("abc");
            Console.WriteLine(n);
        }
        catch (FormatException ex)
        {
            Console.WriteLine($"입력 형식 오류: {ex.Message}");
        }
    }
}
```

예상 출력:
```text
입력 형식 오류: ...
```

코드 해석:
- `"abc"`를 정수로 바꾸는 과정에서 `FormatException`이 발생합니다.
- `catch`에서 예외를 잡아 프로그램 종료 대신 사용자 메시지를 출력합니다.

연습문제:
1. 문제 1 - 0 나누기 처리
   - 문제: 나눗셈 프로그램에서 0으로 나눌 때 예외를 처리하세요.
   - 입력: 피제수, 제수
   - 출력: 결과 또는 오류 메시지
   - 조건: `try-catch` 사용
2. 문제 2 - 사용자 정의 검사
   - 문제: 점수가 0~100 범위를 벗어나면 예외를 던지세요.
   - 입력: 점수 1개
   - 출력: 정상/오류 메시지
   - 힌트: 조건 불일치 시 `throw`

정답 포인트:
- 예외는 "문제 숨기기"가 아니라 "문제 명확화" 도구
- 예상 가능한 오류는 사용자 친화 메시지 제공
