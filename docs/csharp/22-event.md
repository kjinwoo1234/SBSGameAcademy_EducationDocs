# Chapter 22 event

## 학습 목표
- 이벤트와 델리게이트의 관계를 이해한다.
- 발행자-구독자 패턴을 C# 이벤트로 구현할 수 있다.

## 본문

### 22-1 이벤트 기본

**이벤트(event)**는 델리게이트를 감싸 **알림 전용**으로 쓰는 문법입니다. 밖에서는 **`+=`/`-=`로 구독·해지**만 하고, **발생(`Invoke`)은 선언한 클래스 안**에서만 합니다. 공개 델리게이트 필드처럼 외부에서 마음대로 호출하는 일을 막습니다.

| 구분 | 외부에서 | 이럴 때 |
|---|---|---|
| 일반 델리게이트 필드 | 할당·호출 가능(위험) | 내부 전략 교체 |
| `event` | 구독/해지만 | 상태 변화 알림 |

### 22-2 구독/해지

구독은 **`+=`**, 해지는 **`-=`**입니다. 구독을 남기면 객체 수명이 늘어나 **메모리·중복 호출** 문제가 날 수 있으므로, 더 이상 필요 없으면 해지합니다.

### 22-3 발행자-구독자 패턴

**발행자**는 “일이 생겼다”만 알리고, **구독자**는 각자 반응(로그·UI)을 구현합니다. 두 쪽을 직접 메서드 호출로 묶지 않아 **결합이 느슨**합니다.

| 용어 | 의미 |
|---|---|
| 발행자(publisher) | 이벤트를 발생시키는 객체 |
| 구독자(subscriber) | 이벤트를 받아 처리하는 객체 |

아래 코드를 실행해 `Damage(20)` 후 `HP: 80`이 출력되는지 확인해 보세요. (`= delegate { }`는 구독자가 없을 때도 호출이 안전하도록 **빈 구독**을 미리 두는 형태입니다.)

```csharp
using System;

class Player
{
    public event Action<int> OnHpChanged = delegate { };
    private int hp = 100;

    public void Damage(int d)
    {
        hp -= d;
        OnHpChanged.Invoke(hp);
    }
}

class Program
{
    static void Main()
    {
        Player p = new Player();
        p.OnHpChanged += hp => Console.WriteLine($"HP: {hp}");
        p.Damage(20);
    }
}
```

**예상 출력**

```text
HP: 80
```

**코드 해석**
- `event` 덕분에 외부는 `+=`로만 연결합니다.
- `Damage` 안에서만 `Invoke`로 알립니다.
- 구독 람다가 새 체력을 출력합니다.

### 연습문제

**문제 1**
- 문제: 레벨이 오를 때 이벤트를 발생시키는 클래스를 작성하세요.
- 입력: 없음(예: 경험치 추가 후 레벨업 조건 충족)
- 출력: `Level Up!` (또는 `Level: 2`)
- 조건: `event` + 구독자 1개 이상

**문제 2**
- 문제: 구독 후 `-=`로 해지하면 더 이상 호출되지 않는지 확인하세요.
- 입력: 없음(같은 이벤트 트리거 2회, 중간에 해지)
- 출력: 첫 트리거만 로그 1줄, 두 번째는 로그 없음
- 조건: `+=` 후 `-=`

### 정답 포인트

- 문제 1: 레벨 증가 메서드 끝에서 `OnLevelUp.Invoke(...)` (빈 구독 `= delegate { }`를 두면 안전)
- 문제 2: 핸들러를 변수에 담아 `+=`/`-=` (람다만 쓰면 해지가 어려울 수 있음)
- 공통: 이벤트는 알림, 발행과 처리를 분리
