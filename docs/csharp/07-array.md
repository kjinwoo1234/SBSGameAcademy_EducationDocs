# Chapter 07 배열

## 학습 목표
- 1차원 배열 선언/초기화/순회를 익힌다.
- 인덱스 범위와 예외 위험을 이해한다.

## 본문

### 07-1 배열 선언

**배열(array)**은 같은 타입 값을 **연속된 칸**에 모아 두는 자료 구조입니다. 크기를 정해 만들려면 `int[] scores = new int[5];`처럼 쓰고, 칸 개수는 `5`입니다. 처음부터 값을 넣으려면 `int[] scores = { 80, 95, 70, 100 };`처럼 **초기화 목록**을 쓸 수 있습니다. 배열 이름 뒤의 `[]`는 “원소에 접근할 때 인덱스를 쓴다”는 뜻입니다.

### 07-2 인덱스 접근

**인덱스(index)**는 각 칸의 위치 번호이며 **`0`부터** 시작합니다. 네 칸짜리 배열이면 유효 인덱스는 `0`~`3`입니다. `scores[0]`은 첫 값, `scores[scores.Length - 1]`은 마지막 값입니다. 범위를 벗어나 `scores[4]`처럼 접근하면 실행 중 **`IndexOutOfRangeException`**이 납니다. 길이(`Length`)와 “마지막 인덱스 = 길이 - 1”을 항상 함께 떠올리세요.

| 용어 | 의미 |
|---|---|
| 인덱스(index) | 배열에서 각 원소의 위치 번호 (`0`부터) |
| 예외(exception) | 프로그램 실행 중 발생한 오류 상황 |

### 07-3 순회

배열의 모든 칸을 살펴보는 것을 **순회**라고 합니다. **`for`**는 인덱스를 직접 다루므로 “몇 번째 칸인지”가 필요할 때 유리합니다. **`foreach`**는 값을 순서대로 꺼내기만 할 때 문장이 짧습니다. 평균·합처럼 읽기만 하면 `foreach`로 충분하고, 특정 인덱스를 바꾸거나 건너뛸 때는 `for`가 맞습니다.

| 방식 | 적합한 상황 | 장점 | 주의점 |
|---|---|---|---|
| `for` | 인덱스가 필요할 때 | `i`로 위치·조건 제어 | 범위(`0`~`Length-1`) 실수 |
| `foreach` | 값만 순서대로 읽을 때 | 문장이 간결 | 루프 변수로 원본 칸을 바꾸기 어려움 |

아래 코드를 `Main`에 넣고 실행해, `foreach` 합·평균과 `scores[0]` 출력을 확인해 보세요. (`scores[4]`처럼 범위를 벗어나면 예외가 납니다. 지금은 주석으로만 알아 두세요.)

```csharp
using System;

class Program
{
    static void Main()
    {
        int[] scores = { 80, 95, 70, 100 };
        int sum = 0;
        foreach (int s in scores)
        {
            sum += s;
        }
        Console.WriteLine((double)sum / scores.Length);
        Console.WriteLine(scores[0]); // 첫 원소
        // Console.WriteLine(scores[4]); // Length는 4 → 인덱스 4는 범위 밖(예외)
    }
}
```

**예상 출력**

```text
86.25
80
```

**코드 해석**
- `foreach`로 모든 점수를 더해 합을 만듭니다.
- `scores.Length`로 개수를 나눠 평균을 구하고, `(double)`으로 실수 나눗셈을 유지합니다.
- `scores[0]`은 첫 값 `80`입니다. 주석의 `scores[4]`는 유효 인덱스가 아니라 예외입니다.

### 연습문제

**문제 1**
- 문제: 정수 배열에서 최대값을 출력하세요.
- 입력: 없음(배열을 코드에 `{ 3, 9, 2, 7 }`처럼 두고 시작해도 됨)
- 출력: `9`
- 조건: 반복문으로 직접 비교 (`Max` 메서드 사용 금지)

**문제 2**
- 문제: 배열에서 짝수 원소 개수를 구하세요.
- 입력: 없음(예: `{ 1, 2, 3, 4, 6 }`)
- 출력: `3`
- 조건: `% 2 == 0`으로 짝수 판정, 반복문으로 개수 누적

- 문제 2: 짝수일 때마다 카운터 `++`
