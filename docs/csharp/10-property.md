# Chapter 10 프로퍼티

## 학습 목표
- 프로퍼티의 목적과 문법을 이해한다.
- 필드 직접 노출 대신 프로퍼티를 사용하는 이유를 설명할 수 있다.
- 자동 구현 프로퍼티와 `private set`으로 읽기/쓰기 범위를 구분한다.

## 본문

### 10-1 프로퍼티 기본

**프로퍼티(property)**는 필드처럼 `p.Hp`로 쓰되, 내부에서는 **get(읽기)·set(쓰기)** 규칙으로 접근을 통제하는 창구입니다. 공개 `public int hp` 필드에 아무 값이나 대입하는 대신, 프로퍼티로 “음수 체력 금지” 같은 검사를 넣을 수 있습니다. 겉모습은 변수, 속은 메서드에 가깝습니다.

| 방식 | 겉으로 보이는 사용 | 장점 | 주의점 |
|---|---|---|---|
| 필드 직접 공개 | `p.hp = -1;` | 단순 | 규칙·검사 넣기 어려움 |
| 프로퍼티 | `p.Hp = 10;` | get/set에 규칙 가능 | 문법을 익혀야 함 |

### 10-2 자동 구현 프로퍼티

값이 단순 보관만 필요하면 **자동 구현 프로퍼티** `public int Level { get; set; }`를 씁니다. 컴파일러가 **백킹 필드**를 대신 만들므로 필드를 직접 적지 않아도 됩니다. 초기값은 `= 0`처럼 붙일 수 있습니다.

### 10-3 접근 제어

`public int Hp { get; private set; }`처럼 **setter만** `private`로 두면, 밖에서는 읽고(`p.Hp`) 쓰기는 클래스 안(또는 같은 클래스 메서드)만 가능합니다. “조회는 공개, 변경은 메서드 경유” 설계에 맞습니다.

| 형태 | 외부 읽기 | 외부 쓰기 | 이럴 때 |
|---|---|---|---|
| `{ get; set; }` | 가능 | 가능 | 자유롭게 읽고 쓸 값 |
| `{ get; private set; }` | 가능 | 불가 | 변경은 메서드로만 |
| `{ get; }` (생성 시 설정 등) | 가능 | 불가 | 불변에 가까운 Id |

| 용어 | 의미 |
|---|---|
| 프로퍼티(property) | 필드 접근을 제어하는 C# 문법 |
| 백킹 필드(backing field) | 프로퍼티가 내부에서 값을 저장하는 실제 필드 |

아래 코드를 실행해 `Name` 쓰기와 `Hp`의 `private set`(외부 직접 대입 불가, `Damage`로만 감소)을 확인해 보세요. `new Player { Name = "Kim" }`은 객체 만들 때 프로퍼티에 값을 넣는 **객체 초기화**입니다.

```csharp
using System;

class Player
{
    public string Name { get; set; } = "";
    public int Hp { get; private set; } = 100;

    public void Damage(int d)
    {
        Hp -= d;
    }
}

class Program
{
    static void Main()
    {
        Player p = new Player { Name = "Kim" };
        p.Damage(20);
        Console.WriteLine($"{p.Name} {p.Hp}");
    }
}
```

**예상 출력**

```text
Kim 80
```

**코드 해석**
- `Name`은 `{ get; set; }`라 읽고 쓸 수 있습니다.
- `Hp`는 `private set`이라 `p.Hp = 50` 같은 외부 대입은 컴파일 오류이고, `Damage`만 체력을 바꿉니다.
- `Damage(20)` 후 `Hp`는 `80`입니다.

### 연습문제

**문제 1**
- 문제: 생성 시만 설정 가능한 `Id` 프로퍼티를 설계하세요.
- 입력: 없음(예: `new User(10)` 후 `Id` 출력)
- 출력: `10`
- 조건: 외부에서 `Id` 수정 불가 (`private set` 또는 생성자만 설정)

**문제 2**
- 문제: `Level` 프로퍼티를 1 이상만 허용하도록 구현하세요.
- 입력: 없음(예: `Level = 0` 시도 후 유효 값 `3` 저장)
- 출력: `3` (0 이하는 저장하지 않거나 무시)
- 조건: 수동 `get`/`set` 블록에서 조건 검사

### 정답 포인트

- 문제 1: 생성자에서만 대입 + `{ get; private set; }` 또는 `{ get; }`
- 문제 2: `set { if (value >= 1) _level = value; }` 형태
- 공통: 필드 공개보다 프로퍼티 공개, 쓰기는 필요할 때만 열기
