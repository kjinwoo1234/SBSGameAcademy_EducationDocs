# Chapter 04 연산자

## 학습 목표
- **산술·비교·논리 연산자**를 구분해 조건식과 계산에 쓸 수 있다.
- 정수 나눗셈과 실수 나눗셈 결과 차이를 설명할 수 있다.
- **형 변환(casting / conversion)**이 필요한 상황과 `Parse`/`Convert`의 쓰임새를 이해한다.

## 본문

### 04-1 산술/비교 연산자

**연산자(operator)**는 값과 값을 연결해 새 결과를 만드는 기호입니다. **산술 연산자**는 `+ - * / %`처럼 수치를 계산할 때, **비교 연산자**는 `== != > < >= <=`처럼 “같다/크다”를 물어 **`bool` 결과**를 만들 때 씁니다. 조건문(`if`)에 들어가기 전에, 비교식 자체가 `true`/`false`를 만든다는 점을 먼저 익히면 이후 챕터가 수월합니다.

특히 `/`(나누기)는 **피연산자 타입**에 따라 결과가 달라집니다. `int`끼리 나누면 **소수점이 버려진 정수**가 나오고, 한쪽이라도 `double`이면 실수 나눗셈이 됩니다. `%`는 나머지로, “짝수/홀수”, “N의 배수” 판정에 자주 씁니다.

| 분류 | 연산자 | 용도 | 입문 예 |
|---|---|---|---|
| 산술 | `+ - * / %` | 계산 | `7 / 2`, `7 % 2` |
| 비교 | `== != > < >= <=` | 조건식 | `score >= 60` |
| 대입 | `= += -= *= /=` | 값 저장·갱신 | `score += 10` |

**자주 하는 실수:** 같음 비교는 `==`이고, `=`는 대입입니다. `score = 60`을 “같은지 비교”인 줄 알고 쓰면, 비교가 아니라 대입이 되어 예상과 다른 동작·컴파일 오류로 이어질 수 있습니다.

아래 코드를 `Main`에 넣고 실행해 예상 출력과 비교해 보세요.

```csharp
using System;

class Program
{
    static void Main()
    {
        int a = 7, b = 2;
        Console.WriteLine($"합: {a + b}");
        Console.WriteLine($"나머지: {a % b}");
        Console.WriteLine($"a > b ? {a > b}");
        Console.WriteLine($"정수 나눗셈: {a / b}");
        Console.WriteLine($"실수 나눗셈: {(double)a / b}");
    }
}
```

**예상 출력**

```text
합: 9
나머지: 1
a > b ? True
정수 나눗셈: 3
실수 나눗셈: 3.5
```

**코드 해석**
- `%`로 7을 2로 나눈 나머지 `1`을 구합니다.
- `a > b`는 `bool`이라 `True`로 출력됩니다.
- `a / b`는 둘 다 `int`라 `3`(소수 버림)입니다.
- `(double)a / b`는 실수 나눗셈이라 `3.5`입니다.

### 04-2 논리 연산자

**논리 연산자**는 `bool` 값을 결합·반전할 때 씁니다. `&&`(AND)는 **양쪽이 모두 참**일 때만 참, `||`(OR)는 **하나라도 참**이면 참, `!`(NOT)는 참/거짓을 **뒤집습니다.** 게임에서는 “체력이 0보다 크고 **그리고** 무적 상태가 아니다”, “키 A **또는** 키 B를 눌렀다”처럼 조건을 묶을 때 필요합니다.

C#의 `&&`와 `||`는 **단락 평가(short-circuit evaluation)**를 합니다. 왼쪽만으로 결과가 확정되면 오른쪽을 **평가하지 않습니다.** 예를 들어 `left && right`에서 `left`가 `false`면 `right`는 실행되지 않습니다. 덕분에 `text != null && text.Length > 0`처럼, 앞 조건으로 뒤를 안전하게 가드하는 패턴이 가능합니다.

조건을 여러 개 묶을 때는 괄호로 의도를 드러내는 편이 안전합니다. `score >= 60 && attend >= 0.8`처럼 읽히게 쓰면, 나중에 조건을 고칠 때도 실수가 줄어듭니다.

아래 코드를 `Main`에 넣고 실행해 예상 출력과 비교해 보세요.

```csharp
using System;

class Program
{
    static void Main()
    {
        int score = 85;
        double attend = 0.9;

        bool pass = (score >= 60) && (attend >= 0.8);
        bool warn = (score < 60) || (attend < 0.8);

        Console.WriteLine($"합격: {pass}");
        Console.WriteLine($"경고 대상: {warn}");
        Console.WriteLine($"합격 아님: {!pass}");
    }
}
```

**예상 출력**

```text
합격: True
경고 대상: False
합격 아님: False
```

**코드 해석**
- 점수·출석률 **둘 다** 기준을 넘어야 `pass`가 `true`입니다.
- `!pass`는 합격을 뒤집어 “불합격에 해당하는가”를 봅니다.

### 04-3 형 변환

**형 변환**은 값을 다른 타입으로 해석·복사하는 일입니다. 정수끼리 나눗셈에서 소수 결과가 필요하면 `(double)a / b`처럼 **캐스트(cast)**로 한쪽을 실수로 올립니다. `(double)(a / b)`는 **이미 정수로 나눈 뒤** 실수로 바꾸는 것이라 `3.0`이 되어, 원하는 `3.5`와 다를 수 있습니다. **괄호 위치**가 결과에 직결됩니다.

문자열을 숫자로 바꿀 때는 앞 장에서 배운 `int.Parse`를 쓰고, `Convert.ToInt32`처럼 `Convert` 클래스도 자주 등장합니다. `Parse`는 “이 타입으로 해석”에 초점이 있고, `Convert`는 여러 타입 간 변환을 한곳에서 다루는 편의 API에 가깝습니다. 입문에서는 **입력 문자열 → 숫자**는 `Parse`, **이미 숫자인 값의 계산용 승격**은 `(double)` 캐스트를 먼저 익히면 충분합니다.

| 방식 | 예 | 적합한 상황 |
|---|---|---|
| 캐스트 | `(double)a` | 숫자 ↔ 숫자(의도적 승격/축소) |
| `Parse` | `int.Parse(s)` | 문자열 → 숫자(입력 처리) |
| `Convert` | `Convert.ToInt32(x)` | 다양한 타입 변환을 한 스타일로 쓸 때 |

**오류 방지 3요소 (정수 나눗셈으로 평균이 깨짐)**
- **발생 조건**: `(a + b + c) / 3`처럼 합과 개수가 모두 `int`일 때
- **영향**: 소수 평균이 필요한데도 `80`처럼 정수만 나와 UI·판정이 어긋남
- **해결 원리**: `(a + b + c) / 3.0` 또는 `(double)(a + b + c) / 3`처럼 **나누기 전에** 실수 쪽으로 올림

아래 코드를 `Main`에 넣고 실행해, 세 나눗셈 결과가 어떻게 다른지 확인해 보세요.

```csharp
using System;

class Program
{
    static void Main()
    {
        int a = 7, b = 2;
        Console.WriteLine(a / b);           // 3
        Console.WriteLine((double)a / b);   // 3.5
        Console.WriteLine((double)(a / b)); // 3 (주의)

        string text = "42";
        int n = int.Parse(text);
        Console.WriteLine(n + 8);           // 50
    }
}
```

**예상 출력**

```text
3
3.5
3
50
```

**코드 해석**
- 두 번째 줄만 실수 나눗셈입니다.
- 세 번째 줄은 정수 나눗셈 결과를 실수로 올린 것이라 `3`입니다.
- `"42"`를 파싱한 뒤 `+ 8`은 숫자 덧셈입니다. 파싱 없이 `text + 8`이면 문자열 연결이 됩니다.

### 연습문제

**문제 1**
- 문제: 정수 3개의 평균을 **소수점**으로 출력하세요.
- 입력: 예) `70`, `80`, `90`
- 출력: 예) `80` 또는 `80.0` (실수 나눗셈 결과)
- 조건: 형 변환 또는 `3.0` 나눗셈 사용

**문제 2**
- 문제: 점수와 출석률로 합격 조건식(`bool`)을 만들어 출력하세요. (점수 60 이상 **그리고** 출석률 0.8 이상)
- 입력: 없음(변수에 값 대입해도 됨). 예) 점수 `85`, 출석률 `0.9`
- 출력: 예) `True`
- 조건: 논리 연산자 `&&` 사용. `bool` 결과를 그대로 출력

**문제 3**
- 문제: 정수 `n`이 짝수인지 여부를 `bool`로 출력하세요. (`n % 2 == 0`)
- 입력: 없음(변수에 값 대입해도 됨). 예) `n = 4`
- 출력: 예) `True`
- 조건: `%` 연산자 사용. `even`/`odd` 문자열 분기는 조건문 장에서 이어감

### 정답 포인트

- 문제 1: 합을 `double`로 나눔. `(a + b + c) / 3`만 쓰면 정수 평균이 됨
- 문제 2: `bool pass = (score >= 60) && (attend >= 0.8);` → `Console.WriteLine(pass);`
- 문제 3: `Console.WriteLine(n % 2 == 0);`
- 공통: 정수 나눗셈은 소수점 손실, 조건 결합은 괄호로 의도 명확화
