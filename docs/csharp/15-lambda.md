# Chapter 15 람다

## 학습 목표
- 람다식의 문법과 사용 목적을 이해한다.
- 메서드 대신 짧은 동작을 인라인으로 전달할 수 있다.

## 세부 주제
### 15-1 람다 기본
- `(x) => x * 2`

### 15-2 Func/Action
- 반환값 유무

### 15-3 컬렉션 처리
- `FindAll`, `Sort`

## 실습 체크리스트
- 숫자 리스트를 람다로 필터링하고 정렬한다.

## 본문

### 15-1 람다 기본
- 람다는 이름 없는 짧은 함수 표현식입니다.
- 간단한 로직을 전달할 때 메서드 선언 없이 바로 작성할 수 있습니다.

### 15-2 Func/Action
- `Func<T, TResult>`는 반환값이 있는 델리게이트 타입입니다.
- `Action<T>`는 반환값이 없는 작업을 표현합니다.

### 15-3 컬렉션 처리
- 리스트 필터링/정렬에서 람다를 자주 사용합니다.
- 코드 길이를 줄이고 의도를 더 명확히 표현할 수 있습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 람다(lambda) | 이름 없이 즉시 전달해 사용하는 함수 표현식 |
| 델리게이트(delegate) | 메서드를 참조하는 타입 |

예시 코드:
```csharp
using System;
using System.Collections.Generic;

class Program
{
    static void Main()
    {
        List<int> nums = new List<int> { 3, 1, 8, 5 };
        nums.Sort((a, b) => a.CompareTo(b));
        nums.ForEach(x => Console.Write(x + " "));
    }
}
```

예상 출력:
```text
1 3 5 8
```

코드 해석:
- `Sort((a,b) => ...)`에 람다를 전달해 정렬 기준을 지정합니다.
- `ForEach(x => ...)`로 각 원소 출력 로직도 짧게 표현합니다.

연습문제:
1. 문제 1 - 짝수 필터
   - 문제: 리스트에서 짝수만 골라 출력하세요.
   - 입력: 정수 리스트
   - 출력: 짝수 목록
   - 조건: 람다식 사용
2. 문제 2 - 문자열 길이 정렬
   - 문제: 문자열 리스트를 길이순으로 정렬하세요.
   - 입력: 문자열 리스트
   - 출력: 정렬 결과
   - 힌트: `Sort((a,b) => a.Length.CompareTo(b.Length))`

정답 포인트:
- 람다는 "짧고 목적이 분명한 동작"에 적합
- 복잡 로직은 별도 메서드로 분리하는 것이 가독성에 유리
