## 실습 체크리스트
- 지연 작업을 비동기로 처리하고 UI/메인 흐름 블로킹을 줄인다.

## 본문

### 17-1 비동기 개념
- 비동기는 오래 걸리는 작업을 기다리는 동안 다른 작업을 진행할 수 있게 합니다.
- 네트워크/파일/DB 작업에서 특히 중요합니다.

### 17-2 `async`와 `await`
- 비동기 메서드는 보통 `Task` 또는 `Task<T>`를 반환합니다.
- `await`는 작업 완료를 기다리되, 블로킹 코드를 줄여 흐름을 단순화합니다.

### 17-3 예외 처리와 주의점
- 비동기 메서드 안에서 예외가 발생하면 `await` 지점에서 전달됩니다.
- `async void`는 이벤트 핸들러 외에는 지양하는 것이 좋습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| Task | 비동기 작업 단위를 나타내는 타입 |
| 블로킹(blocking) | 현재 스레드가 작업 완료까지 멈춰 있는 상태 |

예시 코드:
```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main()
    {
        Console.WriteLine("start");
        await Task.Delay(500);
        Console.WriteLine("done");
    }
}
```

예상 출력:
```text
start
done
```

코드 해석:
- `async Task Main()`으로 비동기 진입점을 구성합니다.
- `await Task.Delay(500)`에서 지연 작업을 기다린 뒤 다음 출력이 실행됩니다.

연습문제:

   - 입력: 없음
   - 출력: 오류 메시지
   - 힌트: `await` 호출부에서 처리
