# Chapter 26 테스트/디버깅 기초

## 학습 목표
- 재현 가능한 디버깅 절차를 따라 문제를 추적할 수 있다.
- 최소 단위 테스트(`Assert`·예외 케이스)로 핵심 로직을 검증할 수 있다.

## 본문

### 26-1 디버깅 기본 흐름

디버깅은 감으로 고치는 일이 아니라 **절차**입니다. 같은 입력으로 **재현**하고, 변수·흐름을 **관찰**하고, 원인 **가설**을 하나 고른 뒤, 수정 후 같은 케이스로 **검증**합니다.

1. 재현: 같은 입력에서 같은 오류가 나는지 확인
2. 관찰: 변수 값/흐름을 수집
3. 가설: 원인 후보를 하나로 좁힘
4. 검증: 수정 후 재현 케이스 재실행

### 26-2 중단점과 로그

**중단점(breakpoint)**은 특정 줄에서 실행을 멈춰 상태를 봅니다. **watch**는 관심 변수·식을 계속 보여 줍니다. **로그**(`Console.WriteLine` 등)는 빠르지만 너무 많으면 오히려 길을 잃습니다. “입력 / 분기 결과 / 반환값”처럼 **핵심 몇 줄**만 남기는 편이 낫습니다.

### 26-3 단위 테스트 기초

**단위 테스트**는 작은 기능을 자동으로 확인해 **회귀**(고친 뒤 예전 기능이 다시 깨짐)를 줄입니다. 최소로 **정상 입력**, **경계값**, **예외 상황**을 하나씩 둡니다. 입문에서는 `Debug.Assert`와 `try-catch`로도 연습할 수 있습니다. (`Assert`는 보통 Debug 구성에서 실패를 드러냅니다.)

| 용어 | 의미 |
|---|---|
| 회귀(regression) | 수정 이후 기존 기능이 다시 깨지는 문제 |
| 단위 테스트 | 함수/메서드 단위의 자동 검증 |

아래 코드를 실행해 Assert가 통과하고 `예외 테스트 통과`가 출력되는지 확인해 보세요.

```csharp
using System;
using System.Diagnostics;

class Calculator
{
    public static int DivideSafe(int a, int b)
    {
        if (b == 0) throw new ArgumentException("b는 0일 수 없습니다.");
        return a / b;
    }
}

class Program
{
    static void Main()
    {
        Debug.Assert(Calculator.DivideSafe(10, 2) == 5);
        Debug.Assert(Calculator.DivideSafe(-9, 3) == -3);

        try
        {
            Calculator.DivideSafe(1, 0);
        }
        catch (ArgumentException)
        {
            Console.WriteLine("예외 테스트 통과");
        }
    }
}
```

**예상 출력**

```text
예외 테스트 통과
```

**코드 해석**
- `Debug.Assert`로 정상·음수 나눗셈 결과를 확인합니다.
- `b == 0`일 때 예외가 나는지도 `try-catch`로 검증합니다.

### 연습문제

**문제 1**
- 문제: 점수→등급 함수를 만들고 경계 테스트를 작성하세요.
- 입력: 점수 `59`, `60`, `89`, `90`, `100` (등급 규칙은 본인이 문서에 한 줄로 고정)
- 출력: 예: `F`, `D`, `B`, `A`, `A` (규칙과 일치하면 OK)
- 조건: 경계값 포함 Assert 또는 출력 비교 5개

**문제 2**
- 문제: 로그가 많은 함수에서 **핵심 로그 3개만** 남기세요.
- 입력: 없음(직접 만든 함수 또는 가상 코드)
- 출력: 남긴 로그 3줄의 역할 — `입력` / `분기 결과` / `반환값` (문구 자유, 역할만 맞으면 됨)
- 조건: 역할 이유 1줄씩

**문제 3**
- 문제: 버그 1개의 재현·검증 체크리스트를 작성하세요.
- 입력: 없음(예: `DivideSafe`에 `b=0`을 안 막은 버전)
- 출력: 아래 형식으로 채운 기록
  - 재현: 입력 / 기대 / 실제
  - 수정: 한 줄
  - 검증: 다시 실행한 결과
- 조건: 재현 가능하게 숫자·메시지 명시

### 정답 포인트

- 문제 1: 경계(59/60 등)마다 기대 등급을 Assert
- 문제 2: 입력·분기·반환만 남기고 나머지 삭제
- 문제 3: 같은 입력으로 수정 전후를 비교
- 공통: 증거(재현) 중심 디버깅, 테스트는 최소 안전망
