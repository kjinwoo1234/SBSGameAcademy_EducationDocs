# Chapter 09 static

## 학습 목표
- `static`의 의미와 사용 시점을 이해한다.
- 인스턴스 멤버와 정적 멤버를 구분할 수 있다.

## 세부 주제
### 09-1 정적 필드/메서드
- 클래스 공용 데이터

### 09-2 인스턴스 vs static
- 메모리/접근 차이

### 09-3 유틸 클래스 설계
- 상태 없는 기능 모음

## 실습 체크리스트
- 카운터 클래스에 정적 멤버를 적용해 생성 개수를 추적한다.

## 본문

### 09-1 정적 필드/메서드
- `static` 멤버는 객체가 아니라 클래스 자체에 소속됩니다.
- 모든 인스턴스가 같은 정적 필드를 공유합니다.

### 09-2 인스턴스 vs static
- 인스턴스 멤버는 객체마다 값이 다릅니다.
- 정적 멤버는 프로그램 전체에서 공용으로 사용됩니다.

### 09-3 유틸 클래스 설계
- 상태가 필요 없는 계산 기능은 `static` 메서드로 분리하기 좋습니다.
- 예: `Math`, `Console` 클래스.

비교 표:
| 항목 | 인스턴스 멤버 | 정적(static) 멤버 |
|---|---|---|
| 소속 | 객체 | 클래스 |
| 접근 | 객체를 통해 접근 | 클래스명으로 접근 |
| 값 공유 | 객체마다 다름 | 전체 공유 |

용어 설명:
| 용어 | 의미 |
|---|---|
| 인스턴스 멤버 | 각 객체가 개별적으로 가지는 필드/메서드 |
| 정적 멤버 | 클래스 전체가 공유하는 필드/메서드 |

예시 코드:
```csharp
using System;

class Enemy
{
    public static int Count = 0;
    public Enemy() { Count++; }
}

class Program
{
    static void Main()
    {
        _ = new Enemy();
        _ = new Enemy();
        Console.WriteLine(Enemy.Count);
    }
}
```

예상 출력:
```text
2
```

코드 해석:
- `Enemy.Count`는 모든 `Enemy` 객체가 공유하는 정적 필드입니다.
- 생성자에서 `Count++`를 수행해 생성 횟수를 누적합니다.

연습문제:

   - 입력: 없음
   - 출력: 생성 개수
   - 조건: `static int` 사용
2. 문제 2 - 정적 유틸 메서드

   - 입력: 정수 2개
   - 출력: 큰 값 1개
   - 힌트: 클래스명으로 호출

정답 포인트:
- 공유 상태가 필요할 때만 `static` 사용
- 남용하면 테스트/확장성이 떨어질 수 있음
