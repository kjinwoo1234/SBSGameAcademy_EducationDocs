## 본문

### 14-1 제네릭 메서드
- 제네릭은 타입만 다른 중복 코드를 줄이는 방법입니다.
- `T`는 실제 사용 시점에 구체 타입으로 결정됩니다.

### 14-2 제네릭 클래스
- `List<int>`, `List<string>`처럼 같은 구조를 타입만 바꿔 재사용할 수 있습니다.
- 캐스팅 오류를 줄이고 컴파일 시점 타입 검사가 강화됩니다.

### 14-3 제약 조건
- `where T : class` 같은 제약으로 허용 타입 범위를 제한할 수 있습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 제네릭(generic) | 타입에 독립적인 코드를 작성하는 방식 |
| 제약 조건(constraint) | 제네릭 타입에 허용되는 타입 범위를 제한하는 규칙 |

예시 코드:
```csharp
using System;

class Program
{
    static T Max<T>(T a, T b) where T : IComparable<T>
    {
        return a.CompareTo(b) >= 0 ? a : b;
    }

    static void Main()
    {
        Console.WriteLine(Max(3, 7));
        Console.WriteLine(Max("aa", "ab"));
    }
}
```

예상 출력:
```text
7
ab
```

코드 해석:
- `where T : IComparable<T>` 제약으로 비교 가능한 타입만 허용합니다.
- 같은 `Max<T>` 로직을 `int`와 `string`에 재사용합니다.
