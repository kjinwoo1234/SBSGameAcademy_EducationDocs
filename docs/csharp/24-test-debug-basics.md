# Chapter 24 테스트/디버깅 기초

## 학습 목표
- 재현 가능한 디버깅 절차를 따라 문제를 추적할 수 있다.
- 최소 단위 테스트를 작성해 핵심 로직을 검증할 수 있다.

## 세부 주제
### 24-1 디버깅 기본 흐름
- 재현, 관찰, 가설, 검증

### 24-2 중단점과 로그
- breakpoint, watch, 로그 출력

### 24-3 단위 테스트 기초
- assert 기반 검증

## 실습 체크리스트
- 계산 함수 하나를 디버깅하고 테스트 케이스 3개 이상 작성한다.

## 본문

### 24-1 디버깅 기본 흐름
디버깅은 감으로 고치는 작업이 아니라 절차입니다.
1. 재현: 같은 입력에서 같은 오류가 나는지 확인
2. 관찰: 변수 값/흐름을 수집
3. 가설: 원인 후보를 하나로 좁힘
4. 검증: 수정 후 재현 케이스 재실행

### 24-2 중단점과 로그
- **중단점(breakpoint)**: 특정 줄에서 실행을 멈춰 변수 상태를 확인합니다.
- **watch**: 추적하고 싶은 식/변수를 지속적으로 확인합니다.
- 로그는 빠른 확인에 좋지만, 과도한 로그는 원인 파악을 어렵게 할 수 있습니다.

### 24-3 단위 테스트 기초
- 단위 테스트(Unit Test)는 작은 기능을 독립 검증해 회귀(regression)를 줄입니다.
- 테스트는 "정상 입력", "경계값", "예외 상황"을 최소 1개씩 포함하는 것이 좋습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 회귀(regression) | 수정 이후 기존 기능이 다시 깨지는 문제 |
| 단위 테스트 | 함수/메서드 단위의 자동 검증 |

예시 코드:
```csharp
using System;
using System.Diagnostics;

class Calculator
{
    public static int DivideSafe(int a, int b)
    {
        if (b == 0) throw new ArgumentException("b는 0일 수 없습니다.");
        return a / b;
    }
}

class Program
{
    static void Main()
    {
        Debug.Assert(Calculator.DivideSafe(10, 2) == 5);
        Debug.Assert(Calculator.DivideSafe(-9, 3) == -3);

        try
        {
            Calculator.DivideSafe(1, 0);
        }
        catch (ArgumentException)
        {
            Console.WriteLine("예외 테스트 통과");
        }
    }
}
```

예상 출력:
```text
예외 테스트 통과
```

코드 해석:
- `Debug.Assert`는 기대 결과가 맞는지 즉시 확인합니다.
- `try-catch`로 예외가 "정상적으로 발생하는지"도 테스트합니다.

연습문제:
1. 문제 1 - 점수 등급 함수 테스트
   - 문제: 점수에 따라 등급을 반환하는 함수를 만들고 테스트 5개를 작성하세요.
   - 입력: 0~100 점수
   - 출력: 등급 문자열
   - 조건: 경계값(59, 60, 89, 90) 포함
2. 문제 2 - 디버깅 로그 축소
   - 문제: 과도한 로그 코드를 읽고 핵심 로그 3개만 남기세요.
   - 입력: 로그가 많은 함수
   - 출력: 정리된 함수
   - 조건: 남긴 로그의 이유를 1줄씩 작성
3. 문제 3 - 예외 재현 루틴
   - 문제: 재현 절차를 문서화하고 수정 후 검증 체크리스트를 작성하세요.
   - 입력: 임의 버그 1개
   - 출력: 재현/검증 기록

정답 포인트:
- 디버깅은 반드시 "재현 가능한 증거" 중심으로 진행
- 단위 테스트는 핵심 로직 안정성을 유지하는 최소 안전망
