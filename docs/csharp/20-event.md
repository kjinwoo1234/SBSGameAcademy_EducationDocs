# Chapter 20 event

- 발행자-구독자 패턴을 C# 이벤트로 구현할 수 있다.

## 세부 주제
### 20-1 이벤트 기본
- `event` 키워드

### 20-2 구독/해지
- `+=`, `-=`

### 20-3 발행자-구독자 패턴
- 느슨한 결합 구조

## 실습 체크리스트
- 플레이어 체력 변화 이벤트를 만들어 UI 알림과 연결한다.

## 본문

### 20-1 이벤트 기본
- 이벤트는 외부에서 직접 호출하지 못하고, 발행 클래스 내부에서만 발생시킬 수 있습니다.
- 델리게이트를 기반으로 안전한 알림 구조를 만듭니다.

### 20-2 구독/해지
- 구독은 `+=`, 해지는 `-=`로 처리합니다.
- 사용이 끝난 구독을 해지하지 않으면 메모리/동작 문제가 생길 수 있습니다.

### 20-3 발행자-구독자 패턴
- 발행자는 이벤트를 알리고, 구독자는 필요한 반응만 구현합니다.
- 두 객체를 느슨하게 연결해 결합도를 줄입니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 발행자(publisher) | 이벤트를 발생시키는 객체 |
| 구독자(subscriber) | 이벤트를 받아 처리하는 객체 |

예시 코드:
```csharp
using System;

class Player
{
    public event Action<int>? OnHpChanged;
    private int hp = 100;
    public void Damage(int d)
    {
        hp -= d;
        OnHpChanged?.Invoke(hp);
    }
}

class Program
{
    static void Main()
    {
        Player p = new Player();
        p.OnHpChanged += hp => Console.WriteLine($"HP: {hp}");
        p.Damage(20);
    }
}
```

예상 출력:
```text
HP: 80
```

코드 해석:
- `event`를 통해 외부에서 임의 호출을 막고 구독/해지만 허용합니다.
- `OnHpChanged?.Invoke(hp)`는 구독자가 있을 때만 안전하게 이벤트를 발생시킵니다.
- 람다 구독으로 UI 갱신 같은 후속 동작을 분리할 수 있습니다.

   - 입력: 이벤트 트리거 2회
   - 출력: 첫 호출만 로그 출력
   - 힌트: `-=` 사용
