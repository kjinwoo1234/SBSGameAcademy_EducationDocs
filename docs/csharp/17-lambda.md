# Chapter 17 람다

## 학습 목표
- 람다식의 문법과 사용 목적을 이해한다.
- 메서드 대신 짧은 동작을 인라인으로 전달할 수 있다.
- `Func`/`Action`으로 반환 유무를 구분해 쓸 수 있다.

## 본문

### 17-1 람다 기본

**람다(lambda)**는 이름 없는 **짧은 함수 표현식**입니다. 형태는 `(매개변수) => 식` 또는 `x => 식`입니다. 정렬 기준·필터 조건처럼 “한 번 넘기고 끝낼” 동작에 메서드를 따로 만들지 않아도 됩니다.

### 17-2 Func/Action

동작을 변수·인자로 넘길 때 자주 쓰는 타입이 **`Func`**(결과 있음)와 **`Action`**(결과 없음, `void`)입니다. (둘 다 **델리게이트**—메서드를 가리키는 타입—계열입니다. 델리게이트 문법은 이후 장에서 더 다룹니다. 지금은 “람다를 담는 상자” 정도로 쓰면 됩니다.)

| 타입 | 반환 | 예 | 이럴 때 |
|---|---|---|---|
| `Func<int, int>` | 있음 | `x => x * 2` | 계산·변환 |
| `Action<int>` | 없음 | `x => Console.WriteLine(x)` | 출력·부수 효과 |

### 17-3 컬렉션 처리

`List`의 **`Sort`**, **`FindAll`**, **`ForEach`** 등에 람다를 넘기면 필터·정렬·순회를 짧게 적을 수 있습니다. 로직이 길어지면 람다 대신 별도 메서드가 읽기 쉽습니다.

| 용어 | 의미 |
|---|---|
| 람다(lambda) | 이름 없이 즉시 전달해 사용하는 함수 표현식 |
| 델리게이트(delegate) | 메서드를 참조하는 타입 |

아래 코드를 실행해 정렬된 `1 3 5 8` 출력을 확인해 보세요.

```csharp
using System;
using System.Collections.Generic;

class Program
{
    static void Main()
    {
        List<int> nums = new List<int> { 3, 1, 8, 5 };
        nums.Sort((a, b) => a.CompareTo(b));
        nums.ForEach(x => Console.Write(x + " "));
    }
}
```

**예상 출력**

```text
1 3 5 8
```

**코드 해석**
- `Sort((a, b) => ...)`에 비교 람다를 넘겨 오름차순 정렬합니다.
- `ForEach(x => ...)`로 각 원소를 공백과 함께 출력합니다.

### 연습문제

**문제 1**
- 문제: 리스트에서 짝수만 골라 출력하세요.
- 입력: 없음(예: `{ 1, 2, 3, 4, 6 }`)
- 출력: `2 4 6` (공백 구분, 형식 유사 OK)
- 조건: 람다 사용 (`FindAll` 또는 동등 필터)

**문제 2**
- 문제: 문자열 리스트를 길이 순으로 정렬하세요.
- 입력: 없음(예: `{ "hi", "a", "hello" }`)
- 출력: `a hi hello` (짧은 것부터)
- 조건: `Sort((a, b) => a.Length.CompareTo(b.Length))` 형태

### 정답 포인트

- 문제 1: `nums.FindAll(x => x % 2 == 0)` 후 출력
- 문제 2: `Sort((a, b) => a.Length.CompareTo(b.Length))`
- 공통: 짧고 목적이 분명할 때 람다, 복잡하면 메서드로 분리
