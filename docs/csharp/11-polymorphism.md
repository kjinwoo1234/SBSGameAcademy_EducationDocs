# C# 다형성

다형성은 "같은 이름의 기능을 상황에 맞게 다르게 동작"시키는 개념입니다.  
초보자 기준으로는 **"호출은 똑같이 하는데, 실제 실행 내용은 객체 종류에 따라 바뀌는 것"**으로 이해하면 쉽습니다.

## 1) 왜 필요한가?
예를 들어 `Work()`라는 기능이 있을 때,
- 학생은 공부
- 개발자는 코딩
- 디자이너는 디자인  
처럼 같은 기능 이름이라도 실제 행동은 다릅니다.

## 2) 오버라이딩(재정의)
```csharp
class Person
{
    public virtual void Work() => Console.WriteLine("사람이 일합니다.");
}

class Developer : Person
{
    public override void Work() => Console.WriteLine("개발자가 코딩합니다.");
}
```

`virtual`은 "자식이 바꿔도 됨", `override`는 "실제로 바꿔서 사용함"을 의미합니다.

## 3) 부모 타입으로 자식 객체 다루기 (핵심)
```csharp
Person p = new Developer();
p.Work(); // "개발자가 코딩합니다."
```

코드상 변수 타입은 `Person`이지만, 실제 객체는 `Developer`이므로  
실행 시 `Developer.Work()`가 호출됩니다. 이것이 다형성의 핵심입니다.

## 4) 추상 클래스와 인터페이스 (초보자 버전)
- 추상 클래스: 공통 기능 + "꼭 구현해야 할 기능"을 함께 정의
- 인터페이스: "무조건 지켜야 하는 기능 목록"만 정의

## 5) 언제 쓰면 좋은가?
- 결제 방식(카드/계좌/간편결제)을 같은 함수로 호출하고 싶을 때
- 게임 캐릭터(전사/궁수/마법사) 공격 메서드를 통일하고 싶을 때
- 기능 확장 시 기존 코드를 최소 수정하고 싶을 때

## 6) 초보자 실수 포인트
- 부모 메서드에 `virtual`을 빼먹어서 오버라이드가 안 되는 경우
- `new`와 `override`의 차이를 모르는 경우
- "변수 타입"과 "실제 객체 타입"을 혼동하는 경우

## 7) 연습 문제

2. `IMovable` 인터페이스 구현 클래스 작성
