# Chapter 08 포인터와 참조자

## 학습 목표
- 포인터와 참조자의 차이를 이해한다.
- 주소 출력과 역참조 동작을 설명할 수 있다.

## 세부 주제
### 08-1 포인터 기초
- 주소, 역참조, `nullptr`

### 08-2 참조자 기초
- 별칭과 초기화 규칙

### 08-3 포인터 vs 참조자
- 사용 기준 비교

## 실습 체크리스트
- 같은 변수에 포인터/참조자를 각각 적용해 값 변경을 확인한다.

## 본문

### 08-1 포인터 기초
- 포인터는 메모리 주소를 저장하는 변수입니다.
- `*p`는 포인터가 가리키는 실제 값에 접근할 때 사용합니다.
- 유효한 대상을 가리키지 않을 때는 `nullptr`를 사용합니다.

### 08-2 참조자 기초
- 참조자는 기존 변수의 또 다른 이름(별칭)입니다.
- 참조자는 선언 시 반드시 초기화해야 하며 다른 대상을 다시 가리킬 수 없습니다.

### 08-3 포인터 vs 참조자
- 포인터는 `nullptr` 상태를 가질 수 있어 "가리키는 대상 없음"을 표현하기 쉽습니다.
- 참조자는 문법이 간결해 항상 유효한 대상을 다룰 때 읽기 좋습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 주소(address) | 메모리에서 데이터가 저장된 위치 번호 |
| 역참조(dereference) | 주소를 따라가 실제 값에 접근하는 동작(`*p`) |

비교 표:
| 항목 | 포인터 | 참조자 |
|---|---|---|
| null 가능성 | 가능(`nullptr`) | 불가 |
| 재대상 지정 | 가능 | 불가 |
| 문법 | `*`, `->` 필요 | 일반 변수처럼 사용 |

예시 코드:
```cpp
#include <iostream>
using namespace std;

int main() {
    int n = 10;
    int* p = &n;
    int& r = n;

    *p = 20;
    r = 30;

    cout << "n: " << n << endl;
    cout << "address: " << p << endl;
    return 0;
}
```

예상 출력(주소는 실행마다 다름):
```text
n: 30
address: 0x...
```

코드 해석:
- `int* p = &n;`에서 `p`는 `n`의 주소를 저장합니다.
- `*p = 20;`은 포인터를 통해 원본 변수 `n`의 값을 바꿉니다.
- `int& r = n;`은 별칭을 만든 뒤 `r = 30;`으로 다시 원본을 변경합니다.

연습문제:
1. 문제 1 - 포인터 값 변경
   - 문제: 정수 변수 하나를 선언하고 포인터로 값을 2배로 바꾸세요.
   - 입력: 없음
   - 출력: 변경 전/후 값
   - 조건: `*p` 사용
2. 문제 2 - 참조자 별칭 확인
   - 문제: 참조자로 값을 변경했을 때 원본이 바뀌는지 출력하세요.
   - 입력: 없음
   - 출력: 원본 변수 값
   - 힌트: 참조자는 별칭 개념

정답 포인트:
- 포인터는 주소 저장, 참조자는 별칭
- null 상태가 필요하면 포인터가 더 적합
