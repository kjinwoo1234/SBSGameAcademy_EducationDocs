# Chapter 09 동적 메모리와 스마트 포인터

## 학습 목표
- 동적 할당(`new/delete`)의 기본 원리를 이해한다.
- 스마트 포인터의 필요성과 기본 사용법을 익힌다.

## 세부 주제
### 09-1 `new`와 `delete`
- 동적 객체 생성/해제

### 09-2 메모리 누수와 소유권
- 해제 책임 관리

### 09-3 스마트 포인터
- `unique_ptr`, `shared_ptr` 기초

## 실습 체크리스트
- 동적 할당 코드와 스마트 포인터 코드를 비교해본다.

## 본문

### 09-1 `new`와 `delete`
- `new`는 힙(Heap) 영역에 객체를 생성하고 주소를 반환합니다.
- `delete`를 호출하지 않으면 확보한 메모리가 반환되지 않습니다.

### 09-2 메모리 누수와 소유권
- 메모리 누수는 더 이상 접근할 수 없는 동적 메모리가 해제되지 않은 상태입니다.
- 소유권은 "누가 이 메모리를 해제할 책임이 있는가"를 의미합니다.

### 09-3 스마트 포인터
- 스마트 포인터는 객체 수명 관리를 자동화해 `delete` 실수를 줄여줍니다.
- `unique_ptr`는 단독 소유, `shared_ptr`는 참조 횟수 기반 공동 소유를 제공합니다.

스마트 포인터 비교:
| 타입 | 소유 방식 | 복사 가능 여부 | 적합한 상황 |
|---|---|---|---|
| `unique_ptr` | 단독 소유 | 불가(이동만 가능) | 소유자가 명확한 자원 |
| `shared_ptr` | 공동 소유 | 가능 | 여러 객체가 함께 참조 |

용어 설명:
| 용어 | 의미 |
|---|---|
| 메모리 누수(memory leak) | 해제되지 않아 계속 남는 동적 메모리 |
| 소유권(ownership) | 자원의 생성/해제 책임을 가지는 주체 |

예시 코드:
```cpp
#include <iostream>
#include <memory>
using namespace std;

int main() {
    unique_ptr<int> p = make_unique<int>(42);
    cout << *p << endl;
    return 0;
}
```

예상 출력:
```text
42
```

코드 해석:
- `make_unique<int>(42)`는 정수 객체를 만들고 소유권을 `p`에 부여합니다.
- `*p`로 저장된 값을 읽어 출력합니다.
- `main` 종료 시 `unique_ptr`가 자동으로 자원을 해제합니다.

연습문제:
1. 문제 1 - 동적 배열 생성
   - 문제: `new`로 정수 배열을 만들고 값을 출력한 뒤 해제하세요.
   - 입력: 배열 크기 1개
   - 출력: 배열 원소
   - 조건: `delete[]` 사용
2. 문제 2 - unique_ptr 전환
   - 문제: 기존 `new/delete` 코드를 `unique_ptr` 기반으로 바꾸세요.
   - 입력: 없음
   - 출력: 동일 결과
   - 힌트: `make_unique` 사용

정답 포인트:
- `new[]`에는 `delete[]`를 대응
- 가능하면 스마트 포인터를 기본 선택으로 사용
