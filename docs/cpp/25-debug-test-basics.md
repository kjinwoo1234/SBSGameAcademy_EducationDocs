# Chapter 25 디버깅과 테스트 기초

## 학습 목표
- 디버깅 기본 루틴(재현, 관찰, 가설, 검증)을 익힌다.
- 간단한 단위 테스트를 작성해 회귀를 줄이는 방법을 이해한다.

## 세부 주제
### 25-1 디버깅 루틴
- 중단점, 변수 관찰

### 25-2 로그 기반 점검
- 입력/상태/출력 추적

### 25-3 테스트 기초
- 최소 단위 검증

## 실습 체크리스트
- 버그 1개를 재현하고 수정 전/후 테스트 결과를 비교한다.

## 본문

### 25-1 디버깅 루틴
- 버그를 먼저 안정적으로 재현한 뒤 원인 분석을 시작해야 합니다.
- 중단점(breakpoint)과 단계 실행(step)을 사용하면 상태 변화를 추적하기 쉽습니다.

### 25-2 로그 기반 점검
- 함수 입출력과 핵심 상태를 로그로 남기면 오류 구간을 좁히기 쉽습니다.
- 로그는 너무 많지 않게 "원인 추적에 필요한 지점"만 남기는 것이 좋습니다.

### 25-3 테스트 기초
- 테스트는 함수 단위로 입력과 기대 출력이 맞는지 검증하는 습관입니다.
- 버그 수정 후 테스트를 다시 실행해 회귀(다시 깨짐)를 방지합니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 회귀(regression) | 수정 후 기존에 되던 기능이 다시 깨지는 현상 |
| 단위 테스트(unit test) | 작은 함수/모듈 단위의 자동 검증 |

디버깅 체크리스트:
| 단계 | 질문 |
|---|---|
| 재현 | 같은 입력에서 항상 문제를 다시 볼 수 있는가? |
| 관찰 | 어느 값이 기대와 다르게 바뀌는가? |
| 가설 | 어떤 코드가 원인일 가능성이 높은가? |
| 검증 | 수정 후 동일 케이스가 통과하는가? |

예시 코드:
```cpp
#include <cassert>
#include <iostream>
using namespace std;

int divideSafe(int a, int b) {
    if (b == 0) return 0; // 정책: 0 반환
    return a / b;
}

int main() {
    assert(divideSafe(10, 2) == 5);
    assert(divideSafe(10, 0) == 0);
    cout << "tests passed\n";
    return 0;
}
```

예상 출력:
```text
tests passed
```

코드 해석:
- `assert`는 조건이 거짓이면 즉시 실패를 알려줍니다.
- 정상 케이스/예외 케이스를 함께 검증해 함수 정책을 고정합니다.
- 수정 후 같은 테스트를 반복 실행하면 회귀를 빠르게 감지할 수 있습니다.

연습문제:
1. 문제 1 - 버그 재현 문서화
   - 문제: 정수 평균 함수 버그를 재현하고 원인/수정 내용을 기록하세요.
   - 입력: 테스트 케이스 2개 이상
   - 출력: 재현 로그 + 수정 요약
   - 조건: 수정 전/후 결과 비교 포함
2. 문제 2 - 간단 테스트 추가
   - 문제: 문자열 유틸 함수 2개에 대해 `assert` 테스트를 작성하세요.
   - 입력: 문자열 케이스
   - 출력: 통과/실패 결과
   - 힌트: 경계값(빈 문자열) 케이스 포함

정답 포인트:
- 디버깅은 "감"이 아니라 재현 가능한 절차
- 테스트 케이스는 정상/경계/오류 입력을 균형 있게 구성
