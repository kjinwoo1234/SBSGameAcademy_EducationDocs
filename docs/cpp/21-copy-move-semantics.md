# Chapter 21 복사/이동 시맨틱

## 학습 목표
- 복사와 이동의 차이를 이해한다.
- Rule of 3/5 개념을 바탕으로 자원 소유 클래스를 안전하게 설계한다.

## 세부 주제
### 21-1 복사 시맨틱
- 복사 생성자, 복사 대입 연산자

### 21-2 이동 시맨틱
- 이동 생성자, 이동 대입 연산자, `std::move`

### 21-3 Rule of 3/5
- 자원 관리 클래스의 필수 멤버 함수

## 실습 체크리스트
- 동적 메모리를 소유하는 클래스를 만들고 복사/이동 로그를 확인한다.

## 본문

### 21-1 복사 시맨틱
- 복사는 "같은 값을 가진 새 객체"를 만드는 동작입니다.
- 포인터 같은 자원 핸들을 얕게 복사하면 이중 해제 문제가 생길 수 있습니다.

### 21-2 이동 시맨틱
- 이동은 자원을 복사하지 않고 소유권을 옮겨 성능을 높이는 방식입니다.
- `std::move`는 "이 객체를 이동 대상으로 써도 된다"는 의도 표현입니다.

### 21-3 Rule of 3/5
- 소멸자/복사 생성자/복사 대입이 필요하면 Rule of 3를 고려합니다.
- 이동 생성자/이동 대입까지 필요하면 Rule of 5를 적용합니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 시맨틱(semantic) | 코드 동작의 의미/의도 |
| 얕은 복사(shallow copy) | 포인터 주소만 복사해 같은 자원을 공유하는 복사 |
| 깊은 복사(deep copy) | 새 자원을 따로 확보해 내용까지 복사하는 방식 |

복사 vs 이동 비교:
| 항목 | 복사 | 이동 |
|---|---|---|
| 자원 처리 | 새 자원 복제 | 소유권 이전 |
| 비용 | 상대적으로 큼 | 상대적으로 작음 |
| 원본 상태 | 유지 | 유효하지만 비어 있을 수 있음 |

예시 코드:
```cpp
#include <iostream>
#include <utility>
using namespace std;

class Buffer {
public:
    int* data;
    explicit Buffer(int v) : data(new int(v)) {}
    ~Buffer() { delete data; }

    Buffer(const Buffer& other) : data(new int(*other.data)) {
        cout << "copy ctor\n";
    }
    Buffer(Buffer&& other) noexcept : data(other.data) {
        other.data = nullptr;
        cout << "move ctor\n";
    }
};

int main() {
    Buffer a(10);
    Buffer b = a;              // 복사
    Buffer c = std::move(a);   // 이동
    return 0;
}
```

예상 출력:
```text
copy ctor
move ctor
```

코드 해석:
- `Buffer b = a;`에서 복사 생성자가 호출되어 새 메모리를 확보합니다.
- `std::move(a)`는 `a`를 이동 가능 상태로 만들어 이동 생성자를 호출합니다.
- 이동 후 `a.data`를 `nullptr`로 바꿔 이중 해제를 방지합니다.

   - 입력: 없음
   - 출력: 복사/이동 로그

정답 포인트:
- 자원 소유 클래스는 복사/이동/소멸 설계를 함께 고려
- 이동 후 원본 객체는 "안전한 비어 있는 상태"로 유지
