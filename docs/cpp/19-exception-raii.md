# Chapter 19 예외 처리와 RAII

## 학습 목표
- 예외 처리의 기본 흐름(`try/catch/throw`)을 이해한다.
- RAII를 통해 자원 누수를 방지하는 방법을 익힌다.

## 세부 주제
### 19-1 예외 처리 기초
- `try`, `throw`, `catch`

### 19-2 예외 설계 원칙
- 오류 전파와 메시지

### 19-3 RAII
- 자원 수명과 객체 수명 연결

## 실습 체크리스트
- 파일 열기 실패 상황에서 예외를 던지고 처리한다.

## 본문

### 19-1 예외 처리 기초
- 예외는 "정상 흐름에서 처리하기 어려운 오류 상황"을 표현하는 방식입니다.
- `throw`로 예외를 발생시키고 `catch`에서 처리합니다.

### 19-2 예외 설계 원칙
- 어디서 처리할지 명확하지 않으면 상위 호출자에게 예외를 전파합니다.
- 사용자에게 보여줄 메시지는 상황을 이해할 수 있게 작성합니다.

### 19-3 RAII
- RAII는 자원을 객체 수명에 묶어 자동 정리하는 설계 원칙입니다.
- 스코프가 끝날 때 소멸자가 실행되어 자원 누수를 줄일 수 있습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 예외(exception) | 정상 흐름에서 분리된 오류 처리 메커니즘 |
| RAII | **Resource Acquisition Is Initialization**의 약자. 자원 획득/해제를 객체 생성/소멸에 연결하는 C++ 관용구 |

RAII 이름 풀이:
- **Resource**: 메모리, 파일 핸들, 뮤텍스 같은 자원
- **Acquisition**: 자원을 "얻는 것"
- **Is Initialization**: 자원을 얻는 시점을 객체 초기화(생성)와 묶는다는 뜻
- 즉 "객체가 만들어질 때 자원을 얻고, 객체가 사라질 때 자원을 반납"하는 방식입니다.

예시 코드:
```cpp
#include <fstream>
#include <iostream>
#include <stdexcept>
using namespace std;

int main() {
    try {
        ifstream fin("not_found.txt");
        if (!fin) throw runtime_error("파일 열기 실패");
        cout << "open success\n";
    } catch (const exception& e) {
        cout << e.what() << endl;
    }
    return 0;
}
```

예상 출력:
```text
파일 열기 실패
```

코드 해석:
- 파일 스트림 객체는 스코프 종료 시 자동으로 정리되어 RAII 이점을 제공합니다.
- 열기 실패 시 `runtime_error`를 던져 오류 원인을 분리합니다.
- `catch`에서 `what()` 메시지를 출력해 사용자/개발자 모두 원인 파악이 쉬워집니다.

연습문제:
1. 문제 1 - 나눗셈 예외 처리
   - 문제: 0으로 나누기 시 예외를 던지고 처리하세요.
   - 입력: 피제수, 제수
   - 출력: 결과 또는 오류 메시지
   - 조건: `throw` 사용
2. 문제 2 - 파일 열기 래퍼
   - 문제: 파일 열기 유틸 함수를 만들고 실패 시 예외를 던지세요.
   - 입력: 파일 경로
   - 출력: 성공/실패 메시지
   - 힌트: `runtime_error` 활용

정답 포인트:
- 예외는 "복구 불가/상위 정책 필요" 상황에서 사용
- RAII 객체를 사용하면 예외 경로에서도 정리 코드 누락 가능성이 줄어듦
