# Chapter 20 도전! 콘솔 RPG 구조화

## 학습 목표
- 지금까지 배운 C++ 문법을 작은 프로젝트에 통합한다.
- 클래스 분리, 컨테이너, 예외 처리, 자원 관리를 함께 적용한다.

## 세부 주제
### 20-1 프로젝트 범위 정의
- 전투/인벤토리/저장 기능 최소 범위

### 20-2 모듈 분리
- `model`, `system`, `ui` 계층 분리

### 20-3 품질 체크
- 예외 처리, 메모리 안전, 코드 확장성

## 실습 체크리스트
- 최소 5개 클래스와 1개 이상 STL 컨테이너를 사용하는 콘솔 RPG를 구현한다.

## 본문

### 20-1 프로젝트 범위 정의
- 기능을 한 번에 많이 넣기보다 "플레이어 생성 -> 전투 -> 결과 출력" 최소 흐름부터 완성합니다.
- 최소 기능 완성 후 인벤토리, 아이템, 저장 기능을 단계적으로 추가합니다.

약어 설명(RPG):
- **RPG**는 **Role-Playing Game**의 약자입니다.
- 한국어로는 "역할 수행 게임"이며, 플레이어가 캐릭터를 성장시키며 진행하는 게임 장르입니다.
- 이 챕터에서는 RPG의 핵심 루프(상태 관리 -> 전투 -> 저장)를 코드 구조화 연습 대상으로 사용합니다.

### 20-2 모듈 분리
- 데이터 모델(`Player`, `Enemy`)과 규칙(`BattleSystem`)과 화면 출력(`ConsoleUI`)을 나눕니다.
- 분리 구조를 쓰면 버그 수정과 기능 확장이 쉬워집니다.

### 20-3 품질 체크
- 자원은 RAII 기반 객체로 관리하고, 예외 발생 경로도 고려합니다.
- 게임 루프 종료 조건과 입력 오류 처리 루틴을 반드시 둡니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 게임 루프(game loop) | 입력 처리 -> 상태 업데이트 -> 출력 과정을 반복하는 실행 구조 |
| 모듈 분리 | 기능을 파일/클래스 단위로 나누어 수정 범위를 줄이는 설계 방식 |

프로젝트 체크 표:
| 항목 | 최소 완료 기준 |
|---|---|
| 전투 시스템 | 턴 기반 공격/피격/종료 판단 |
| 데이터 구조 | `vector` 또는 `unordered_map` 사용 |
| 안정성 | 잘못된 입력/파일 실패 처리 |
| 확장성 | 새 직업/몬스터 추가 시 수정 범위 최소화 |

예시 코드(구조 예시):
```cpp
class Player {};
class Enemy {};
class BattleSystem {};
class Inventory {};
class Game {
public:
    void run();
};
```

코드 해석:
- 이 챕터의 핵심은 "문법"보다 "구조화"입니다.
- 클래스 개수보다 책임 분리가 명확한지가 더 중요합니다.
- 최종 목표는 유지보수 가능한 프로젝트 뼈대를 만드는 것입니다.

연습문제:
1. 문제 1 - 최소 플레이 루프
   - 문제: 시작/전투/종료 3단계 루프를 구현하세요.
   - 입력: 메뉴 선택값
   - 출력: 단계별 로그
   - 조건: 잘못된 입력 처리 포함
2. 문제 2 - 저장/불러오기
   - 문제: 플레이어 상태를 파일에 저장하고 다시 불러오세요.
   - 입력: 이름, 레벨, 체력
   - 출력: 저장/복원 결과
   - 힌트: `fstream` + 실패 체크
