# Chapter 10 클래스와 객체

## 학습 목표
- 클래스와 객체의 기본 구조를 이해한다.
- 캡슐화, 생성자, 접근 지정자 개념을 설명할 수 있다.

## 세부 주제
### 10-1 클래스 기본 문법
- 멤버 변수/멤버 함수

### 10-2 생성자와 초기화
- 객체 생성 시점 초기화

### 10-3 접근 지정자와 캡슐화
- `public`, `private`

## 실습 체크리스트
- 간단한 `Player` 클래스를 만들어 상태 변경 함수를 작성한다.

## 본문

### 10-1 클래스 기본 문법
- 클래스는 데이터(멤버 변수)와 기능(멤버 함수)을 하나로 묶는 사용자 정의 타입입니다.
- 객체는 클래스를 기반으로 실제로 생성된 인스턴스입니다.

### 10-2 생성자와 초기화
- 생성자는 객체가 만들어질 때 자동 호출되는 특별한 함수입니다.
- 멤버 초기화 목록을 사용하면 의도한 초기값을 안정적으로 설정할 수 있습니다.

### 10-3 접근 지정자와 캡슐화
- `public`은 외부 접근 허용, `private`는 클래스 내부에서만 접근 가능합니다.
- 캡슐화는 내부 데이터를 보호하고, 정해진 함수로만 상태를 바꾸게 하는 설계 방식입니다.

접근 지정자 비교:
| 지정자 | 접근 가능 범위 | 사용 목적 |
|---|---|---|
| `public` | 클래스 외부/내부 모두 | 외부에 공개할 기능 |
| `private` | 클래스 내부 | 내부 상태 보호 |

용어 설명:
| 용어 | 의미 |
|---|---|
| 객체(object) | 클래스 설계도로부터 만들어진 실제 데이터 단위 |
| 캡슐화(encapsulation) | 데이터와 기능을 묶고 내부 상태를 보호하는 설계 원칙 |

예시 코드:
```cpp
#include <iostream>
using namespace std;

class Player {
private:
    int hp;
public:
    Player(int h) : hp(h) {}
    void damage(int d) { hp -= d; }
    int getHp() const { return hp; }
};

int main() {
    Player p(100);
    p.damage(15);
    cout << p.getHp() << endl;
    return 0;
}
```

예상 출력:
```text
85
```

코드 해석:
- `Player(int h) : hp(h) {}`는 생성 시 `hp`를 초기화합니다.
- `damage` 함수는 내부 상태 `hp`를 변경하는 전용 통로 역할을 합니다.
- `getHp() const`는 객체 상태를 바꾸지 않고 읽기만 하는 함수입니다.

   - 출력: 누적 경험치
