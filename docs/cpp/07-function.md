# Chapter 07 함수와 참조

## 학습 목표
- 함수 선언/정의/호출 흐름을 이해한다.
- 값 전달과 참조 전달의 차이를 설명할 수 있다.

## 세부 주제
### 07-1 함수 기본 문법
- 선언, 정의, 반환값

### 07-2 참조 전달(`&`)
- 원본 수정이 필요한 함수

### 07-3 기본 인자와 오버로딩
- 같은 이름 함수 확장

## 실습 체크리스트
- 두 수를 교환하는 함수를 값 전달/참조 전달로 비교한다.

## 본문

### 07-1 함수 기본 문법
- 함수는 특정 작업을 이름으로 묶어 재사용성을 높입니다.
- 선언(프로토타입)을 먼저 두면 아래쪽 정의를 나중에 작성할 수 있습니다.

### 07-2 참조 전달(`&`)
- 참조 전달은 변수의 별명을 통해 원본에 직접 접근하는 방식입니다.
- 함수 안에서 값이 바뀌면 호출한 쪽 변수도 함께 바뀝니다.

### 07-3 기본 인자와 오버로딩
- 기본 인자는 인자를 생략했을 때 자동으로 채워지는 값입니다.
- 오버로딩은 같은 함수 이름으로 매개변수 형태를 달리해 여러 버전을 만드는 기능입니다.

전달 방식 비교:
| 방식 | 선언 형태 | 원본 변경 |
|---|---|---|
| 값 전달 | `int x` | 불가 |
| 참조 전달 | `int& x` | 가능 |

용어 설명:
| 용어 | 의미 |
|---|---|
| 참조(reference) | 기존 변수에 붙는 또 다른 이름 |
| 오버로딩(overloading) | 같은 이름 함수를 매개변수로 구분해 여러 개 정의 |

예시 코드:
```cpp
#include <iostream>
using namespace std;

void swapByRef(int& a, int& b) {
    int t = a;
    a = b;
    b = t;
}

int main() {
    int x = 3, y = 7;
    swapByRef(x, y);
    cout << x << ' ' << y << endl;
    return 0;
}
```

예상 출력:
```text
7 3
```

코드 해석:
- `swapByRef(int& a, int& b)`는 원본 변수 자체를 참조로 받습니다.
- 함수 내부 교환 결과가 `main`의 `x`, `y`에 그대로 반영됩니다.
- 참조 전달은 원본 변경이 필요한 함수에서 자주 사용됩니다.

   - 입력: 점수 1개
   - 출력: 증가 후 점수
   - 조건: 원본 값이 실제로 변경되어야 함

정답 포인트:
- 원본 변경 목적이면 참조(`&`) 또는 포인터 전달 사용
- 오버로딩 시 반환형만 다르게 해서는 구분 불가
