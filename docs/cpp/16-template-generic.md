# Chapter 16 템플릿과 제네릭 프로그래밍

## 학습 목표
- 함수/클래스 템플릿의 기본 문법을 이해한다.
- 중복 코드를 줄이는 제네릭 설계 개념을 익힌다.

## 세부 주제
### 16-1 함수 템플릿
- 타입 일반화

### 16-2 클래스 템플릿
- 타입 매개변수 클래스

### 16-3 템플릿 사용 주의점
- 컴파일 에러 해석 기초

## 실습 체크리스트
- 타입에 독립적인 `maxValue` 함수를 템플릿으로 작성한다.

## 본문

### 16-1 함수 템플릿
- 함수 템플릿은 타입만 바뀌는 중복 함수를 하나로 합칠 때 유용합니다.
- 호출 시 전달 인자 타입을 보고 템플릿 타입이 결정됩니다.

### 16-2 클래스 템플릿
- 클래스 템플릿은 내부 데이터 타입을 외부에서 지정받는 클래스입니다.
- 예: `Box<int>`, `Box<string>`.

### 16-3 템플릿 사용 주의점
- 템플릿 오류 메시지는 길 수 있어 "첫 번째 핵심 오류"부터 읽는 습관이 중요합니다.
- 타입이 요구 연산(예: `<`)을 지원하는지 확인해야 합니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 템플릿(template) | 타입을 일반화해 재사용 코드를 만드는 문법 |
| 제네릭(generic) | 특정 타입에 고정되지 않은 일반화 프로그래밍 방식 |

예시 코드:
```cpp
#include <iostream>
using namespace std;

template <typename T>
T maxValue(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    cout << maxValue(3, 5) << endl;
    cout << maxValue(2.5, 1.8) << endl;
    return 0;
}
```

예상 출력:
```text
5
2.5
```

코드 해석:
- `template <typename T>`는 "타입 자리에 T를 쓰겠다"는 선언입니다.
- 호출 시 `int`, `double` 각각에 맞는 함수 버전이 컴파일 시 생성됩니다.
- 동일 로직을 타입별로 중복 작성하지 않아도 됩니다.

정답 포인트:
- 타입만 다른 중복 코드는 템플릿으로 일반화
- 템플릿 오류는 연산 가능 여부부터 점검
