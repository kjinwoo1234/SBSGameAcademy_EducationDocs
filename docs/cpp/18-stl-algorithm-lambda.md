# Chapter 18 STL 알고리즘과 람다

## 학습 목표
- STL 알고리즘과 람다식을 함께 사용하는 방법을 익힌다.
- 정렬/탐색/조건 처리 코드를 간결하게 작성할 수 있다.

## 세부 주제
### 18-1 `<algorithm>` 기초
- `sort`, `find_if`

### 18-2 람다식 문법
- 캡처, 매개변수, 반환

### 18-3 알고리즘 + 람다 조합
- 조건 기반 처리

## 실습 체크리스트
- 점수 벡터를 정렬하고 조건에 맞는 값만 출력한다.

## 본문

### 18-1 `<algorithm>` 기초
- STL 알고리즘은 컨테이너 데이터 처리 코드를 줄여줍니다.
- 대표적으로 `sort`, `find_if`, `count_if`를 자주 사용합니다.

약어 설명(STL):
- **STL**은 **Standard Template Library**의 약자입니다.
- 한국어로는 "표준 템플릿 라이브러리"이며, 컨테이너/알고리즘/반복자 같은 재사용 컴포넌트 모음입니다.
- 이 챕터에서는 STL 중에서도 알고리즘(`sort`, `find_if`)과 람다 조합에 집중합니다.

### 18-2 람다식 문법
- 람다는 이름 없는 짧은 함수 객체입니다.
- `[캡처](인자){본문}` 형태를 사용합니다.

### 18-3 알고리즘 + 람다 조합
- 정렬 기준을 람다로 전달하면 다양한 규칙을 간단히 적용할 수 있습니다.
- 조건 필터링 역시 람다와 알고리즘을 조합하면 코드가 짧아집니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 람다(lambda) | 필요 위치에서 즉시 정의해 쓰는 익명 함수 |
| 캡처(capture) | 외부 변수를 람다 내부에서 사용할 수 있게 가져오는 방식 |

예시 코드:
```cpp
#include <algorithm>
#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> scores = {70, 95, 82, 60, 88};
    sort(scores.begin(), scores.end(), [](int a, int b) { return a > b; });

    for (int s : scores) cout << s << ' ';
    cout << endl;
    return 0;
}
```

예상 출력:
```text
95 88 82 70 60
```

코드 해석:
- `sort`에 람다 비교 함수를 전달해 내림차순 정렬을 수행합니다.
- `scores.begin(), scores.end()`는 정렬 범위를 나타냅니다.
- 별도 비교 함수를 만들지 않아도 조건을 인라인으로 표현할 수 있습니다.

연습문제:
1. 문제 1 - 조건 필터
   - 문제: 정수 벡터에서 80 이상 점수 개수를 세세요.
   - 입력: 점수 목록
   - 출력: 조건 충족 개수
   - 조건: `count_if` + 람다 사용
2. 문제 2 - 사용자 정의 정렬
   - 문제: 문자열 벡터를 길이 기준으로 오름차순 정렬하세요.
   - 입력: 문자열 목록
   - 출력: 정렬된 문자열
   - 힌트: `sort` 비교 람다에서 `a.size() < b.size()`

정답 포인트:
- 알고리즘은 반복문 대체가 아니라 의도 명확화 도구
- 람다는 짧고 목적이 분명할 때 효과적
