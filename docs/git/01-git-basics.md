# Chapter 01 Git 기본과 저장소 시작

## 학습 목표
- Git 저장소를 생성하고 상태를 확인할 수 있다.
- 작업 영역/스테이징 영역/커밋의 차이를 설명할 수 있다.
- `.gitignore`를 사용해 불필요 파일 추적을 방지할 수 있다.

## 세부 주제
### 01-1 저장소 초기화
- `git init`, `git status`

### 01-2 스테이징과 커밋
- `git add`, `git commit`

### 01-3 `.gitignore`
- 추적 제외 규칙

## 실습 체크리스트
- 빈 폴더에서 첫 커밋까지 완료한다.

## 본문

### 01-1 저장소 초기화
- `git init`은 현재 폴더를 Git 저장소로 만듭니다.
- `git status`는 변경 상태를 확인하는 가장 중요한 기본 명령입니다.

### 01-2 스테이징과 커밋
- 스테이징은 "이번 커밋에 포함할 변경"을 고르는 단계입니다.
- 커밋은 선택된 변경을 이력으로 고정하는 작업입니다.

### 01-3 `.gitignore`
- 빌드 산출물/임시 파일처럼 추적이 필요 없는 파일을 제외합니다.
- 팀 전체가 같은 제외 규칙을 공유하면 커밋 잡음을 줄일 수 있습니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 작업 영역(working tree) | 현재 편집 중인 파일이 있는 영역 |
| 스테이징 영역(staging area) | 커밋 후보 변경을 임시로 모아두는 영역 |
| 커밋(commit) | 변경 이력의 저장 단위 |

예시 명령:
```bash
git init
git status
git add README.md
git commit -m "학습 노트 초기 파일 추가"
```

예상 결과:
```text
초기 커밋이 생성되고 git log에서 이력이 확인된다.
```

코드 해석:
- `git add README.md`는 특정 파일만 선택적으로 스테이징합니다.
- `git commit`은 스테이징된 내용만 이력에 저장합니다.

연습문제:
1. 문제 1 - 첫 커밋 만들기
   - 문제: 새 저장소를 만들고 `README.md` 첫 커밋을 생성하세요.
   - 입력: 없음
   - 출력: 커밋 1개
   - 조건: `git status`를 커밋 전/후로 확인
2. 문제 2 - `.gitignore` 적용
   - 문제: `*.log`, `bin/`을 추적 제외하도록 설정하세요.
   - 입력: 없음
   - 출력: 상태에서 제외 파일 미표시
   - 힌트: `.gitignore` 저장 후 `git status` 재확인

정답 포인트:
- 스테이징과 커밋을 구분하면 이력 품질이 올라감
- `git status` 습관화가 실수 예방의 핵심
