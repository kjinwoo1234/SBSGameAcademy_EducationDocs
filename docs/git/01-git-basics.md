# Git 기본과 저장소 시작

Git은 분산 버전 관리 도구입니다. 로컬에서도 전체 이력을 관리할 수 있어 실험과 복구가 쉽습니다.

## 학습 목표
- Git 저장소를 초기화하고 기본 상태를 확인할 수 있다.
- 스테이징과 커밋의 차이를 설명할 수 있다.
- `.gitignore`를 활용해 불필요한 파일 커밋을 방지할 수 있다.

## 핵심 개념
- `working tree`: 현재 작업 중인 파일 상태
- `staging area`: 커밋 후보를 모아두는 임시 영역
- `commit`: 변경 이력의 최소 단위

## 기본 명령어
```bash
git init
git status
git add .
git commit -m "초기 프로젝트 구조 추가"
```

## 단계별 실습
1. 빈 폴더에서 `git init`을 실행한다.
2. `README.md`를 만들고 `git status`로 변경 상태를 확인한다.
3. `git add README.md` 후 `git commit`으로 첫 커밋을 만든다.
4. `.gitignore`에 임시 파일 패턴을 추가하고 다시 상태를 확인한다.

## 실수 포인트
- `git add .`를 습관적으로 사용해 불필요 파일을 함께 스테이징하는 문제
- 커밋 메시지를 "수정", "업데이트"처럼 목적 없이 작성하는 문제

## 셀프 퀴즈
1. 스테이징 영역을 거치는 이유는 무엇인가?
2. `git status`를 커밋 전후로 확인해야 하는 이유는?

## 정답
1. 어떤 변경을 이번 커밋에 포함할지 선택적으로 제어하기 위해서
2. 누락/오포함 파일을 방지하고 이력 품질을 유지하기 위해서
