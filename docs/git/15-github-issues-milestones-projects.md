# Chapter 15 이슈·마일스톤·프로젝트로 작업 관리하기

## 학습 목표

- **이슈(issue)**로 무엇을 기록·논의하는지 설명한다.
- **마일스톤(milestone)**으로 작업 묶음과 기한을 관리하는 목적을 말한다.
- **프로젝트(Projects)** 칸반에서 이슈 카드를 열 사이로 옮기며 진행 상태를 나타내는 방법을 이해한다.

## 세부 주제

### 15-1 이슈란?

- 버그·기능·질문·작업 단위 기록

### 15-2 마일스톤이란?

- 기한·버전·스프린트 단위 묶음

### 15-3 프로젝트 탭과 칸반 보드

- 이슈를 카드로 보고 열(단계) 사이로 이동

## 실습 체크리스트

- 학습용 저장소에서 **이슈 2개**를 만들고 라벨을 하나 이상 붙인다.
- **마일스톤 하나**를 만들고 두 이슈를 모두 그 마일스톤에 연결한다.
- **Projects**에서 보드(보통 **Board** 레이아웃)를 만들고, 이슈 카드를 **열이 다른 칸**으로 옮겨 상태를 바꿔 본다.

## 본문

<a id="ch15-1"></a>

### 15-1 이슈란?

**이슈(issue)**는 저장소 안에서 **할 일·버그·아이디어·질문**을 **번호가 매겨진 티켓**으로 남기는 기능입니다. PR과 달리 **코드 변경이 필수는 아니고**, 논의와 결정 이력이 쌓입니다. 제목은 **한 줄로 무엇인지** 드러나게, 본문에는 **재현 절차(버그)·기대 동작·스크린샷**을 넣으면 나중에 읽는 사람이 시간을 덜 씁니다.

용어 설명:

| 용어 | 의미 |
|------|------|
| 라벨(label) | 이슈·PR에 붙이는 색 태그(`bug`, `enhancement` 등). 필터·통계에 쓰임 |
| 담당자(assignee) | 맡은 사람(비워 두면 팀 풀에서 가져가기도 함) |

**이슈를 쓰지 않으면** 채팅·구두로만 전달되어 **누가 무엇을 끝냈는지**, **왜 그렇게 결정했는지**가 사라지고, 같은 버그가 **반복 보고**되기 쉽습니다.

---

<a id="ch15-2"></a>

### 15-2 마일스톤이란?

**마일스톤(milestone)**은 이슈(와 PR)를 **한 묶음**으로 묶고, 보통 **목표 날짜·버전 이름**(예: `v1.2`, `Sprint 5`)을 붙여 진행률을 보는 단위입니다. 진행률 막대는 연결된 이슈의 **열림/닫힘** 비율로 계산되는 경우가 많아, “이번 주 목표가 얼마나 닫혔는지”를 한눈에 볼 수 있습니다.

**마일스톤 없이만 쓰면** 이슈는 많은데 **언제까지 어떤 묶음**인지가 불분명해져, 우선순위와 **출시 범위**를 설명하기 어려워질 수 있습니다.

---

<a id="ch15-3"></a>

### 15-3 프로젝트 탭에 칸반 보드로 이슈 카드 관리하기

**Projects**(프로젝트)는 GitHub에서 **이슈·PR을 카드**로 모아 **보드·표** 형태로 관리하는 영역입니다. **Board** 형식은 **열(column)**이 흔히 **진행 단계**를 뜻합니다(예: **Todo**, **In progress**, **Done**). 이슈를 보드에 **추가**하면 카드가 생기고, 카드를 **다른 열로 드래그**하면 GitHub가 이슈 상태를 바꿀 수 있도록 **자동화**를 연결해 둔 경우가 많습니다(저장소 설정·워크플로에 따라 다름). 최소한 팀원에게는 “지금 이 카드는 이 단계”라는 **시각적 합의**가 됩니다.

**칸반을 쓰지 않으면** 목록만으로는 **동시에 진행 중인 작업 수**가 한눈에 들어오지 않아, 같은 영역을 **여러 사람이 겹쳐 고치거나** 막힌 작업이 **오래 방치**되기 쉽습니다.

GitHub 웹에서의 대략적 경로(버전에 따라 메뉴 이름이 조금 다를 수 있음)는 다음과 같습니다.

1. 저장소 상단 **Projects** 탭 → **New project**.
2. **Board** 등 템플릿 선택 후 프로젝트 이름 저장.
3. **Add item** 또는 기존 이슈를 보드에 연결한 뒤, 카드를 열 사이로 이동.

연습문제:

1. 문제: 버그 리포트 이슈 본문에 **최소 세 가지** 포함하면 좋은 항목을 나열하세요(형식 자유).
2. 문제: 마일스톤 `v1.0`에 이슈 5개를 묶었을 때, 진행률이 의미 있으려면 이슈를 어떻게 관리해야 하는지 한 문장으로 쓰세요.
3. 문제: 칸반에서 **In progress** 열에 카드가 너무 많으면 어떤 팀 문제를 시사할 수 있는지 적으세요.

정답 포인트:

버그 이슈에는 재현 절차, 기대/실제, 환경(OS·브라우저 등)이 대표적으로 들어갑니다. 마일스톤 진행률은 이슈를 **열림/닫힘**으로 끝까지 가져가야 퍼센트가 의미 있습니다. **In progress**가 과다하면 **진행 중 작업 과다·우선순위 불명**을 시사할 수 있습니다.
