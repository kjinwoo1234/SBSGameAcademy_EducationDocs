# Chapter 04 원격 저장소와 협업

## 학습 목표
- 원격 저장소 연결과 브랜치 푸시를 수행할 수 있다.
- Pull Request(PR) 기반 리뷰 흐름을 적용할 수 있다.
- 병합 후 로컬 동기화 루틴을 습관화할 수 있다.

## 세부 주제
### 04-1 원격 브랜치 동기화
- `push`, 추적 브랜치 설정

### 04-2 PR 리뷰 흐름
- 변경 이유 + 테스트 결과 공유

### 04-3 병합 후 정리
- `main` 동기화, 브랜치 정리

## 실습 체크리스트
- 기능 브랜치 푸시부터 PR 병합 후 로컬 동기화까지 완료한다.

## 본문

### 04-1 원격 브랜치 동기화
- 협업에서는 로컬 커밋을 원격 브랜치에 공유해야 리뷰가 가능합니다.
- `git push -u`를 쓰면 추적 브랜치가 설정되어 다음 푸시가 간편해집니다.

### 04-2 PR 리뷰 흐름
- PR에는 변경 내용만이 아니라 변경 이유와 검증 결과를 함께 적어야 합니다.
- 리뷰 피드백은 작은 커밋으로 나눠 반영하면 추적이 쉽습니다.

### 04-3 병합 후 정리
- 병합 후에는 `main`을 최신화하고 사용한 작업 브랜치를 정리합니다.
- 이 루틴이 없으면 다음 작업 시작점이 오래된 상태가 될 수 있습니다.

약어 설명(PR):
- **PR**은 **Pull Request**의 약자입니다.
- 한국어로는 "병합 요청"이며, 브랜치 변경을 검토받고 합치는 협업 절차입니다.
- PR을 통해 코드 품질과 변경 의도를 팀이 함께 확인합니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 원격 추적 브랜치 | 로컬 브랜치가 대응되는 원격 브랜치 정보를 기억하는 연결 |
| 리뷰 라운드 | PR 피드백 반영과 재검토 요청이 한 번 순환되는 단위 |

예시 명령:
```bash
git push -u origin feature/input-validation
git switch main
git pull origin main
```

예상 결과:
```text
원격 브랜치 공유 -> PR 리뷰 -> 병합 -> 로컬 기준 브랜치 최신화가 완료된다.
```

코드 해석:
- `-u` 옵션은 현재 로컬 브랜치와 원격 브랜치 연결을 저장합니다.
- 병합 후 `main`을 최신화해야 다음 브랜치를 올바른 기준에서 시작할 수 있습니다.

연습문제:
1. 문제 1 - 원격 추적 브랜치 설정
   - 문제: 기능 브랜치를 원격에 처음 푸시하며 추적 브랜치를 설정하세요.
   - 입력: 브랜치 이름
   - 출력: 푸시 성공 로그
   - 조건: `-u` 옵션 포함
2. 문제 2 - PR 템플릿 작성
   - 문제: "변경 이유/검증 결과" 2개 섹션이 있는 PR 본문을 작성하세요.
   - 입력: 변경 요약
   - 출력: PR 본문 초안
   - 힌트: 체크리스트 형태 사용

정답 포인트:
- 협업 품질은 코드 + 커뮤니케이션 품질의 합
- 병합 후 동기화는 다음 작업 품질을 보장하는 필수 루틴
