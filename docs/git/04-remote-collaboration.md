# 원격 저장소와 협업

팀 협업에서는 로컬 이력 관리뿐 아니라 원격 저장소 동기화, 리뷰 절차가 중요합니다.

## 학습 목표
- 원격 저장소 연결과 브랜치 푸시를 수행할 수 있다.
- Pull Request 기반 리뷰 흐름을 적용할 수 있다.
- 병합 후 동기화 루틴을 습관화할 수 있다.

## 기본 협업 흐름
1. 기능 브랜치 작업 후 원격에 푸시
2. Pull Request 생성
3. 리뷰 반영 후 재푸시
4. 병합 완료 후 로컬 `main` 동기화

## 핵심 명령어
```bash
git remote -v
git push -u origin feature/input-validation
git switch main
git pull origin main
```

## 단계별 실습
1. 로컬 브랜치에서 작업 완료 후 `git status`로 변경 상태를 확인한다.
2. `git push -u origin <branch>`로 원격 추적 브랜치를 설정하며 푸시한다.
3. Pull Request를 생성하고 변경 목적/테스트 결과를 작성한다.
4. 리뷰 피드백 반영 후 재푸시하고 최종 승인 후 병합한다.
5. 병합 뒤 `main`으로 전환해 `git pull origin main`으로 동기화한다.

## 리뷰 반영 루틴
- 리뷰 코멘트 단위로 수정 후 작은 커밋 생성
- 재검토 요청 전 로컬 빌드/테스트 수행
- 병합 뒤 사용한 브랜치 정리

## 이해 점검 체크리스트
- 원격 추적 브랜치 설정(`-u`)을 적용했는가?
- PR 설명에 변경 이유와 검증 결과를 포함했는가?
- 병합 후 로컬 기준 브랜치를 최신화했는가?

## 셀프 퀴즈
1. `-u` 옵션으로 푸시하는 이유는?
2. 병합 후 로컬 `main`을 동기화해야 하는 이유는?

## 정답
1. 해당 로컬 브랜치의 기본 원격 추적 브랜치를 설정하기 위해서
2. 다음 작업 시작 시 최신 기준에서 브랜치를 만들기 위해서
