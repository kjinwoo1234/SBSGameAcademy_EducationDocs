# 병합 충돌 해결

충돌은 협업에서 자연스럽게 발생합니다. 중요한 것은 "안전하게 해결하고 검증하는 절차"입니다.

## 학습 목표
- 충돌이 발생하는 원리를 설명할 수 있다.
- 충돌 마커를 읽고 올바르게 수정할 수 있다.
- 충돌 해결 후 검증 절차를 수행할 수 있다.

## 충돌 마커 읽기
```text
<<<<<<< HEAD
현재 브랜치 내용
=======
병합 대상 브랜치 내용
>>>>>>> feature/xxx
```

## 해결 절차
1. 충돌 파일을 열어 두 변경 의도를 파악한다.
2. 최종 코드로 수동 정리한다.
3. `git add <file>`로 해결 완료를 표시한다.
4. 병합 커밋 또는 rebase 계속 진행 후 테스트한다.

## 핵심 명령어
```bash
git status
git add src/player_move.cpp
git commit -m "이동 속도 계산 충돌 해결 및 조건 통합"
```

## 검증 포인트
- 충돌 해결 지점 관련 테스트를 우선 실행했는가?
- 동일 파일의 다른 분기 로직이 깨지지 않았는가?

## 셀프 퀴즈
1. 충돌 파일을 수정한 뒤 바로 해야 하는 Git 명령은?
2. 충돌 해결 후 왜 테스트가 필수인가?

## 정답
1. `git add <충돌 파일>`
2. 수동 병합 과정에서 논리 누락/오타가 쉽게 발생하기 때문
