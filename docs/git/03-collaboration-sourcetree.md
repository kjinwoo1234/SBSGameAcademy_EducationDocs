# Chapter 3 여러 명이 함께 Git으로 협업하기

## 학습 목표
- 원격 협업 흐름(가져오기 → 작업 → 푸시)을 반복할 수 있다.
- 브랜치를 만들고 전환해 병렬 작업을 한다.
- 병합·충돌 해결·PR까지의 역할을 설명한다.

## 세부 주제
- 원격 저장소 기준 협업
- 브랜치 생성과 체크아웃
- 병합(merge)
- 충돌(conflict) 해결
- Pull Request
- 릴리즈 태그·GitHub Releases 개념

## 실습 체크리스트
- 동일 원격을 두 클론(또는 두 계정)으로 시뮬레이션해 충돌 1회 해결
- 기능 브랜치에서 PR 생성·머지(또는 로컬 병합) 완료

## 본문

<a id="ch3-1"></a>
### 3-1 원격 저장소에서 협업하기

- 작업 전 `git pull`(또는 SourceTree의 **Pull**)로 **최신 `main`을 맞춥니다.
- 각자 브랜치에서 작업하면 `main` 직접 푸시 충돌을 줄일 수 있습니다.
- 푸시가 거절되면 보통 원격에 새 커밋이 있으므로 **먼저 가져와 병합/리베이스**합니다.

예시 명령:
```bash
git checkout main
git pull origin main
```

---

<a id="ch3-2"></a>
### 3-2 브랜치 만들고, 이동하기

- 브랜치 이름은 `feature/login`, `fix/typo`처럼 **목적**이 드러나게 짓습니다.
- `git switch -c feature/foo` 또는 `git checkout -b feature/foo`로 생성 겸 전환합니다.

예시 명령:
```bash
git switch -c feature/add-readme
# 작업 후
git push -u origin feature/add-readme
```

용어 설명:
| 용어 | 의미 |
|------|------|
| 체크아웃/checkout | 특정 브랜치·커밋으로 작업 트리를 맞추는 것 |

---

<a id="ch3-3"></a>
### 3-3 병합: 브랜치와 브랜치 합치기

- **병합(merge)**은 한 브랜치의 이력을 다른 브랜치에 **합치는** 동작입니다.
- Fast-forward는 갈래가 없을 때 포인터만 앞으로 이동하고, 그렇지 않으면 **병합 커밋**이 생길 수 있습니다.

예시 명령:
```bash
git checkout main
git merge feature/add-readme
```

---

<a id="ch3-4"></a>
### 3-4 충돌: 앗! 둘이 똑같은 코드를 고쳤어요

- 같은 줄을 서로 다르게 수정하면 Git이 **자동 병합에 실패**하고 충돌 마커를 남깁니다.
- 파일에서 `<<<<<<<`, `=======`, `>>>>>>>`를 찾아 **의도한 최종 코드**로 정리한 뒤 스테이징하고 커밋합니다.

오류 방지 패턴:
1. **언제**: 동일 라인·인접 라인을 양쪽에서 수정했을 때.
2. **왜 문제**: 잘못 고르면 빌드 오류·로직 버그.
3. **어떻게**: 한쪽만 남기지 말고 **실행 가능한 결과**로 수동 편집 후 `add` + `commit`.

---

<a id="ch3-5"></a>
### 3-5 풀 리퀘스트: 브랜치를 합치는 예의 바른 방법

- PR은 **코드 리뷰·CI 확인·토론**을 거친 뒤 병합하기 위한 절차입니다.
- GitHub에서 base(보통 `main`)와 compare(작업 브랜치)를 선택해 PR을 엽니다.
- 리뷰어 지정, 설명에 **변경 이유·테스트 방법**을 적는 것이 좋습니다.

---

<a id="ch3-6"></a>
### 3-6 릴리즈: 개발이 완료되었습니다. 출시하자!

- **태그(tag)**는 특정 커밋에 붙는 **이름 붙은 포인터**로, 배포 버전(`v1.0.0`)을 표시합니다.
- GitHub **Releases**는 태그에 빌드 산출물·릴리즈 노트를 붙이는 화면입니다.

예시 명령:
```bash
git tag -a v1.0.0 -m "First stable release"
git push origin v1.0.0
```

연습문제:
1. 문제: 두 브랜치에서 같은 파일의 같은 함수를 수정해 충돌을 낸 뒤, 수동으로 해결하고 병합 커밋까지 완료하세요.
2. 문제: PR 설명에 포함하면 좋은 항목 3가지를 나열하세요(예: 변경 요약, 스크린샷, 테스트 방법).

정답 포인트:
- 협업 = 최신 맞춤 + 브랜치 + PR + 병합; 충돌은 **의미 있는 최종 코드**로 해결.
