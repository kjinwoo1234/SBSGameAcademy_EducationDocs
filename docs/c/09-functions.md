### 09-1 함수 정의/선언

함수는 반복되는 코드를 한 덩어리로 묶어 **재사용**하기 쉽게 만드는 도구입니다. **선언(프로토타입)**은 “이런 이름·인자·반환형의 함수가 있다”고 미리 알려 주는 약속이고, **정의**는 실제로 어떤 일을 하는 본문입니다. 입력(매개변수)과 출력(반환값)을 분명히 적어 두면 나중에 고치거나 나누어 쓰기도 편해집니다. `printf` 역시 C 표준 라이브러리에 들어 있는 **함수**의 하나입니다.

우리가 `printf`를 직접 선언하지 않아도 쓸 수 있는 이유는 `#include <stdio.h>`를 넣으면 **`stdio.h` 안에 이미 `printf`의 선언**이 들어 있기 때문입니다. 실제 `printf`의 구현은 컴파일·링크 과정에서 표준 라이브러리와 연결되어 붙습니다. 정리하면 **헤더 포함 + 라이브러리 링크** 덕분에 직접 선언·정의를 쓰지 않아도 호출할 수 있습니다.

용어를 표로 정리하면 다음과 같습니다.

| 용어 | 의미 |
|---|---|
| 함수 선언(프로토타입) | 함수 이름/인자/반환형을 먼저 알려 주는 문장 |
| 함수 정의 | 함수가 실제로 어떤 일을 하는지 작성한 본문 |
| 매개변수 | 함수가 입력으로 받는 값 |
| 반환값 | 함수가 실행 후 돌려주는 값 |

전형적인 작성 순서는 (1) 함수 원형(선언)을 적고, (2) `main` 등에서 호출하고, (3) 아래쪽이나 다른 위치에 정의를 두는 흐름입니다. 반대로 **정의를 호출보다 위에** 두면 선언을 생략할 수도 있습니다. 컴파일러가 정의를 먼저 읽었기 때문입니다. 다만 파일이 길어질수록 아래에 정의가 숨는 경우가 많아서, 실무에서는 **선언과 정의를 나누는 방식**을 더 자주 씁니다.

| 방식 | 사용 조건 | 장점 | 단점 |
|---|---|---|---|
| 선언 + 정의 분리 | 정의가 아래에 있거나 파일 분리 시 | 구조화/협업에 유리 | 코드가 조금 길어짐 |
| 선언 생략 + 정의 먼저 | 정의를 호출보다 위에 둘 수 있을 때 | 간단함 | 파일이 커지면 관리 어려움 |

```c
#include <stdio.h>

int add(int a, int b);  // 1) 선언(프로토타입)

int main(void) {
    printf("add: %d\n", add(3, 4));  // 2) 호출
    return 0;
}

int add(int a, int b) {  // 3) 정의
    return a + b;
}
```

**예상 출력**

```text
add: 7
```

```c
#include <stdio.h>

int add(int a, int b) {  // 정의를 먼저 작성
    return a + b;
}

int main(void) {
    printf("add: %d\n", add(3, 4));  // 아래에서 호출
    return 0;
}
```

### 09-2 지역변수

지역변수는 **함수 안**에서 선언된 변수로, 그 함수 블록 안에서만 쓸 수 있습니다. 함수가 끝나면 지역변수는 사라지는 쪽으로 이해하면 됩니다. 그래서 서로 다른 함수에서 같은 이름을 써도 **서로 다른 변수**로 동작합니다.

| 용어 | 의미 |
|---|---|
| 스코프 | 변수를 사용할 수 있는 코드 범위 |
| 생명주기 | 변수가 생성되고 사라질 때까지의 시간 범위 |

### 09-3 전역변수 / `static` / `register`

전역변수는 파일 전체에서 접근할 수 있어 편하지만, 남용하면 함수끼리 **의존성**이 커져 디버깅이 어려워질 수 있습니다. `static`은 붙이는 위치에 따라 **접근 범위나 생명주기**를 조절할 때 씁니다. `register`는 예전에는 CPU 레지스터에 넣어 달라는 힌트로 쓰이기도 했으나, 요즘 컴파일러에서는 의미가 제한적인 편입니다.

### 09-4 재귀함수

재귀함수는 **자기 자신을 다시 호출**하는 함수입니다. 반드시 **종료 조건(기저 조건)**이 있어야 하며, 없으면 호출이 끝나지 않아 프로그램이 멈추거나 종료될 수 있습니다. 팩토리얼·피보나치·트리 탐색처럼 문제를 더 작은 같은 모양으로 나눌 수 있을 때 유리합니다.

| 유형 | 설명 | 예시 |
|---|---|---|
| 일반 함수 | 인자를 받아 결과 반환 | `add(3, 4)` |
| 재귀 함수 | 자기 자신을 호출 | `factorial(n)` |
| void 함수 | 반환값 없음 | `print_menu()` |

```c
#include <stdio.h>

int factorial(int n);

int factorial(int n) {
    if (n <= 1) return 1;  // 종료 조건
    return n * factorial(n - 1);
}

int main(void) {
    printf("5! = %d\n", factorial(5));
    return 0;
}
```

**예상 출력**

```text
5! = 120
```

### 정답 포인트

함수 선언은 `반환형 함수명(매개변수);` 형태로 쓰면 됩니다. 재귀 함수에는 반드시 종료 조건이 있어야 하고, `sum_to_n(n)`은 `n + sum_to_n(n - 1)` 같은 식으로 작은 문제로 넘기면 됩니다.
