## 본문

### 09-1 함수 정의/선언
- 함수는 반복되는 코드를 묶어 재사용성을 높이는 도구입니다.
- 선언(프로토타입)은 "어떤 함수가 있다"는 약속이고, 정의는 실제 동작 코드입니다.
- 입력(매개변수)과 출력(반환값)을 명확히 설계하면 유지보수가 쉬워집니다.
- `printf`도 C 표준 라이브러리에 포함된 "함수"입니다.

`printf`를 선언/정의 없이 바로 쓰는 이유:
- 우리가 `printf`를 직접 선언하지 않아도 `#include <stdio.h>`를 작성하면 됩니다.
- `stdio.h` 안에는 `printf`의 함수 선언(프로토타입)이 이미 들어 있습니다.
- 실제 `printf` 함수 정의(구현)는 컴파일/링크 과정에서 표준 라이브러리와 연결되어 사용됩니다.
- 즉, **직접 선언/정의를 안 해도 되는 이유는 헤더 포함(`#include`) + 라이브러리 링크** 덕분입니다.

용어 설명(09-1):
| 용어 | 의미 |
|---|---|
| 함수 선언(프로토타입) | 함수 이름/인자/반환형을 먼저 알려주는 문장 |
| 함수 정의 | 함수가 실제로 어떤 일을 하는지 작성한 본문 |
| 매개변수 | 함수가 입력으로 받는 값 |
| 반환값 | 함수가 실행 후 돌려주는 값 |

함수 선언/정의 작성 순서:
1. 함수 원형(선언)을 먼저 작성
2. `main` 또는 다른 함수에서 호출
3. 함수 정의를 작성

선언을 생략하고 정의를 먼저 쓰는 방식:
- 함수 정의가 호출 코드보다 위에 있다면, 별도 함수 선언(프로토타입)을 생략할 수 있습니다.
- 즉 "정의 자체가 먼저 등장"하면 컴파일러가 그 정보를 바로 알 수 있습니다.
- 다만 코드가 길어지면 흐름이 복잡해질 수 있어, 실무에서는 보통 선언/정의 분리를 더 많이 사용합니다.

두 방식 비교:
| 방식 | 사용 조건 | 장점 | 단점 |
|---|---|---|---|
| 선언 + 정의 분리 | 정의가 아래에 있거나 파일 분리 시 | 구조화/협업에 유리 | 코드가 조금 길어짐 |
| 선언 생략 + 정의 먼저 | 정의를 호출보다 위에 둘 수 있을 때 | 간단함 | 파일이 커지면 관리 어려움 |

예시 코드(선언/정의 분리):
```c
#include <stdio.h>

int add(int a, int b);  // 1) 선언(프로토타입)

int main(void) {
    printf("add: %d\n", add(3, 4));  // 2) 호출
    return 0;
}

int add(int a, int b) {  // 3) 정의
    return a + b;
}
```

예상 출력:
```text
add: 7
```

예시 코드(선언 생략, 정의 먼저):
```c
#include <stdio.h>

int add(int a, int b) {  // 정의를 먼저 작성
    return a + b;
}

int main(void) {
    printf("add: %d\n", add(3, 4));  // 아래에서 호출
    return 0;
}
```

### 09-2 지역변수
- 지역변수는 함수 안에서 선언되며, 해당 함수 블록 안에서만 사용할 수 있습니다.
- 함수가 끝나면 지역변수는 메모리에서 사라집니다.
- 서로 다른 함수에서 같은 이름을 써도 별개 변수로 동작하는 이유가 여기에 있습니다.

용어 설명(09-2):
| 용어 | 의미 |
|---|---|
| 스코프 | 변수를 사용할 수 있는 코드 범위 |
| 생명주기 | 변수가 생성되고 사라질 때까지의 시간 범위 |

### 09-3 전역변수 / `static` / `register`
- 전역변수는 파일 전체에서 접근 가능해 편리하지만, 남용하면 의존성이 커질 수 있습니다.
- `static`은 사용 위치에 따라 생명주기/접근 범위를 제한할 때 사용합니다.
- `register`는 과거 최적화 힌트 용도로 쓰였고, 현대 컴파일러에서는 의미가 제한적인 편입니다.

### 09-4 재귀함수
- 재귀함수는 함수가 자기 자신을 다시 호출하는 방식입니다.
- 반드시 종료 조건(기저 조건)이 있어야 하며, 없으면 무한 호출로 프로그램이 종료될 수 있습니다.
- 팩토리얼, 피보나치, 트리 탐색처럼 문제를 작은 단위로 나눌 수 있을 때 유용합니다.

함수 유형 정리:
| 유형 | 설명 | 예시 |
|---|---|---|
| 일반 함수 | 인자를 받아 결과 반환 | `add(3, 4)` |
| 재귀 함수 | 자기 자신을 호출 | `factorial(n)` |
| void 함수 | 반환값 없음 | `print_menu()` |

예시 코드(재귀 함수):
```c
#include <stdio.h>

int factorial(int n);

int factorial(int n) {
    if (n <= 1) return 1;  // 종료 조건
    return n * factorial(n - 1);
}

int main(void) {
    printf("5! = %d\n", factorial(5));
    return 0;
}
```

예상 출력:
```text
5! = 120
```
