## 본문

### 09-1 함수 정의/선언
- 함수는 반복되는 코드를 묶어 재사용성을 높이는 도구입니다.
- 선언(프로토타입)은 "어떤 함수가 있다"는 약속이고, 정의는 실제 동작 코드입니다.
- 입력(매개변수)과 출력(반환값)을 명확히 설계하면 유지보수가 쉬워집니다.

### 09-2 지역변수
- 지역변수는 함수 안에서 선언되며, 해당 함수 블록 안에서만 사용할 수 있습니다.
- 함수가 끝나면 지역변수는 메모리에서 사라집니다.
- 서로 다른 함수에서 같은 이름을 써도 별개 변수로 동작하는 이유가 여기에 있습니다.

### 09-3 전역변수 / `static` / `register`
- 전역변수는 파일 전체에서 접근 가능해 편리하지만, 남용하면 의존성이 커질 수 있습니다.
- `static`은 사용 위치에 따라 생명주기/접근 범위를 제한할 때 사용합니다.
- `register`는 과거 최적화 힌트 용도로 쓰였고, 현대 컴파일러에서는 의미가 제한적인 편입니다.

### 09-4 재귀함수
- 재귀함수는 함수가 자기 자신을 다시 호출하는 방식입니다.
- 반드시 종료 조건(기저 조건)이 있어야 하며, 없으면 무한 호출로 프로그램이 종료될 수 있습니다.
- 팩토리얼, 피보나치, 트리 탐색처럼 문제를 작은 단위로 나눌 수 있을 때 유용합니다.

예시 코드:
```c
#include <stdio.h>

int add(int a, int b);     // 선언
int factorial(int n);      // 선언

int add(int a, int b) {    // 정의
    return a + b;
}

int factorial(int n) {
    if (n <= 1) return 1;  // 종료 조건
    return n * factorial(n - 1);
}

int main(void) {
    printf("add: %d\n", add(3, 4));
    printf("5! = %d\n", factorial(5));
    return 0;
}
```

정답 포인트:
- 함수 선언: `반환형 함수명(매개변수);`
- 재귀 함수는 반드시 종료 조건 포함
- `sum_to_n(n)`: `n + sum_to_n(n - 1)` 형태
