## 본문

### 23-1 `typedef`

`typedef`는 이미 있는 타입에 **짧은 별칭**을 붙여 코드를 읽기 쉽게 만드는 기능입니다. 구조체나 함수 포인터처럼 선언이 길어질 때 특히 도움이 됩니다.

### 23-2 구조체 전달과 반환

값으로 넘기면 복사가 일어나고, 포인터로 넘기면 **같은 메모리**를 함수 안에서 다루게 됩니다. 덩어리가 크면 포인터 전달이 성능과 표현 면에서 유리한 경우가 많습니다.

### 23-3 중첩 구조체

구조체 멤버로 **다른 구조체**를 넣으면, 위치·속도·상태처럼 여러 층의 정보를 한 타입으로 묶을 수 있습니다.

### 23-4 공용체(Union)

공용체(union)는 여러 멤버가 **같은 메모리 영역**을 공유합니다. 한 시점에는 그중 **하나의 의미**로만 다루는 것이 안전하고, 어떤 멤버가 유효한지는 프로그램이 따로 기억해야 합니다.

### 23-5 열거형(Enum)

열거형(`enum`)은 상태를 숫자 대신 **이름**으로 쓰게 해 줍니다. `if (state == 2)`보다 `if (state == RUN)`이 읽기 쉬운 이유입니다.

| 요소 | 핵심 목적 |
|---|---|
| `typedef` | 긴 타입 선언 단순화 |
| 중첩 구조체 | 복합 데이터 표현 |
| `union` | 메모리 공유 |
| `enum` | 상태값 이름화 |

```c
#include <stdio.h>

typedef struct {
    int x, y;
} Point;

enum State { IDLE, RUN, JUMP };

int main(void) {
    Point p = {10, 20};
    enum State st = RUN;
    printf("(%d, %d), state=%d\n", p.x, p.y, st);
    return 0;
}
```

**예상 출력**

```text
(10, 20), state=1
```
