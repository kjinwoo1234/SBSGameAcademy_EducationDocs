## 본문

### 02-1 Hello World! 출력해보기

아래 예시를 따라 파일에 적어 보고, Visual Studio에서는 `Ctrl + Shift + B` 또는 `Ctrl + B`로 빌드한 뒤 `Ctrl + F5` 또는 `F5`로 실행해 결과를 확인하면 됩니다. 지금은 이해가 잘 안되더라도 우선 따라서 적은 후 실행해봅시다.

```c
#include <stdio.h>

int main() {
    printf("Hello, World!");
    return 0;
}
```

#### `main` 함수: 프로그램의 출발점

위 `Hello, World!` 코드에서 한눈에 들어오는 큰 덩어리는 **`int main() { ... }`**입니다. C 프로그램은 실행될 때 **항상 `main`부터** 들어가므로, `main`을 **프로그램의 출발점(시작점)**이라고 부릅니다. 그리고 이 `main`은 **함수(function, 이름 붙은 작업 묶음)**입니다. 함수가 무엇인지에 대한 설명은 곧바로 해드릴 예정이니 지금 단계에서는 “프로그램이 실행되면 컴퓨터가 맨 먼저 들어가서 코드를 실행하는 곳이 `int main()` 이후 `{ ... }` 부분이구나”라고 기억해 두면 이후 설명이 덜 꼬입니다.

아래는 **Hello, World 글자 출력 없이** 끝나는 가장 작은 `main` 예시로, 위 코드의 뼈대와 같습니다.

```c
int main() {
    return 0;
}
```

`int main()`은 “`main`이라는 이름의 함수가 있다는 뜻으로 읽을 수 있습니다. 괄호 안에 매개변수 이름을 적지 않은 형태로, **인자 없음**을 뜻합니다. `return 0;`은 `main`이 운영체제(OS)에 **정상 종료**를 알리는 관례적인 값으로, 우선은 “프로그램이 성공적으로 끝났다”는 신호로 기억하면 충분합니다.

**한 줄로 읽기(초보용)**  
“정수(`int`)를 반환하는 함수 `main`이 있고, 이 함수는 0을 반환한다.”

#### ‘함수’는 어떤 의미일까요?

앞에서는 **`main`이 시작점**이고, 그 `main` 자체가 **함수**라는 사실만 먼저 잡았습니다. 이제부터는 같은 말인 **함수**가 조금 더 넓게 무엇을 뜻하는지, 작은 예시 하나로 확장해 봅니다. **함수에 대한 자세한 내용**(선언과 정의의 구분, 지역·전역 변수, 재귀 등)은 **[Chapter 09 함수](./09-functions.md)**에서 배웁니다.

수학에서의 **함수**는 흔히 “식이 있고, 숫자를 넣으면 결과가 나온다”고 배웁니다. C 언어의 함수도 **비슷하게** 이해할 수 있습니다. 정해진 일을 하고, 경우에 따라 **결과 값**을 돌려줍니다.

다만 C의 함수는 화면에 글자를 찍는 일처럼 **숫자 결과 말고도 여러 동작**을 할 수 있습니다. 그래서 “수학의 함수와 100% 같다”고 단정하기보다는, **지금은 입력 → 계산 → 결과** 순서로 연습한다고 생각하면 됩니다.

#### `main` 말고 다른 함수도 만들 수 있나요?

**만들 수 있습니다.** `helper`처럼 **이름만 다르게** 함수를 여러 개 두어도 됩니다. 다만 실행이 시작될 때 **자동으로 호출되는 것은 `main`뿐**이라는 점이 C의 특별한 규칙입니다. 그래서 다른 함수(예를 들어 강의에서 말하는 `a`, `b`, `c` 같은 이름의 함수)를 돌리고 싶다면 **`main` 안에서 이름을 불러** 주어야 합니다. 이렇게 이름을 불러 실행하는 것을 **함수 호출(function call, 콜)**이라고 합니다.

```c
#include <stdio.h>

void helper(void) {
    printf("helper 실행\n");
}

int main() {
    helper();
    return 0;
}
```

위 예시에서는 `main`이 시작된 뒤, `main` 안의 **`helper();`** 때문에 `helper` 함수가 호출됩니다.

#### 예시: 수식 `f(x) = 3x + 4`를 C 함수로 만들기

`x`가 0이면 `3 * 0 + 4 = 4`, `x`가 1이면 7입니다. 같은 규칙을 코드로 옮기면 아래와 같습니다.

```c
#include <stdio.h>

int f(int x) {
    return 3 * x + 4;
}

int main() {
    printf("%d\n", f(0));
    printf("%d\n", f(1));
    return 0;
}
```

`int`는 **integer(정수, 소수점 없는 수)**를 다룬다는 뜻의 자료형 이름입니다. `int`와 **`float`·`double`·`char` 등 다른 자료형**, 변수 선언·형식 지정자를 체계적으로 익히는 것은 **[Chapter 03 변수와 연산자](./03-variables-operators.md)**에서 다룹니다. `int x`는 함수 `f`가 받는 입력을 `x`라는 이름으로 쓰겠다는 뜻이고, 이런 입력을 **매개변수(parameter, 파라미터)**라고 부릅니다. `int f(int x)` 한 줄은 “정수를 받아 정수를 돌려주는 함수 `f`”의 **머리(head, 윗줄)**에 해당합니다. 입문 강의에서는 이 줄을 **함수를 선언한다**고 말하기도 하지만, C 문법에서는 윗줄과 중괄호 몸통이 합쳐진 전체를 **함수 정의(function definition)**라고 부르는 경우가 많습니다. 지금은 “이름과 입력 모양이 적힌 윗줄”로 이해하면 됩니다.

`{`와 `}`(중괄호, curly brace)는 **어디부터 어디까지가 이 함수의 내용인지**를 묶는 기호입니다. `return`은 함수가 돌려줄 값을 정하는 줄이고, 값을 돌려준다는 말을 **반환(return)한다**고 합니다. 세미콜론(`;`)은 한 문장이 끝났다는 표시로, 지금은 “문장 끝에 붙는 기호”로 익혀 두면 됩니다.

**코드 해석(한 줄 요약)**  
`f`는 “`x`를 받아 `3 * x + 4`를 계산한 값을 돌려주는 함수”이고, `main`은 그 결과를 `printf`로 화면에 출력합니다.

**예상 출력**

```text
4
7
```

**용어 설명(`%d`)**  
`printf`의 `%d`는 **정수를 출력하기 위한 형식 지정자(format specifier)**입니다. 지금은 “`f(0)` 같은 정수 결과를 출력할 때 쓴다”만 알아 두면 되고, 출력 형식과 변수 타입을 맞추는 규칙은 **[Chapter 03 변수와 연산자](./03-variables-operators.md)**에서 이어집니다.

#### 왜 이 파일에 `printf`를 만들지 않았는데 쓸 수 있을까요? (`#include`와 `stdio.h`)

**`printf` 함수는 이 파일 안에서 따로 만들지 않았는데, 어떻게 이름만 쓰고 사용할 수 있을까요?** 앞에서 **함수**는 이름을 불러 **호출**해서 쓴다고 했습니다. `printf`도 마찬가지로 **누군가가 미리 이름과 호출 모양을 정해 둔 함수**인데, 그 정보가 들어 있는 것이 바로 **헤더 파일(header file, 헤더, 보통 `.h` 확장자)**입니다.

C에서는 **`#include` 지시문**(전처리 단계에서 **다른 파일** 내용을 이 소스에 읽어 넣는 줄)으로 그런 헤더를 가져올 수 있습니다. **`stdio.h`**는 **standard input/output**(표준 입출력)과 관련된 이름들이 모인 헤더이고, `printf`처럼 자주 쓰는 함수의 **이름·인자 모양을 알려 주는 선언(declaration, “이런 식으로 호출할 수 있다”는 안내)**이 여기에 적혀 있습니다. 그래서 **`#include <stdio.h>`**를 넣어야 컴파일러가 `printf`를 **함수 호출**로 이해할 수 있습니다. (화면에 글자를 찍는 **실행 코드 본체**는 **연결(link)** 단계에서 **표준 라이브러리**와 이어지는 경우가 많지만, 지금은 **“`stdio.h` 헤더를 포함해야 `printf`를 쓴다”**로 기억하면 됩니다.)

#### 위의 `Hello, World!` 코드를 다시 읽기

위에서 짚은 순서대로, 맨 처음 예제 한 덩어리를 다시 정리하면 다음과 같습니다.

- **`#include <stdio.h>`** — 바로 위 절에서 설명한 대로, 외부 **헤더**를 읽어 넣고 **`printf` 선언** 등을 알려 줍니다.
- **`int main() { ... }`** — 프로그램의 시작점인 **`main` 함수**를 만듭니다.
- **`printf("Hello, World!");`** — `printf`를 **호출**해 괄호 안 문자열을 출력하라고 넘깁니다.
- **`return 0;`** — `main`이 0을 반환하며 **정상 종료**했다는 뜻입니다.

**한 줄 요약(말로 설명 연습)**  
“정수를 반환하는 `main`을 만들고, 그 안에서 `printf`로 문장을 출력한 뒤, 0을 반환한다.”

#### 자주 하는 실수(발생 조건 → 영향 → 점검)

`return 0` 뒤에 세미콜론(`;`)이 빠지면 컴파일 오류가 날 수 있어 빌드가 막힙니다. `printf(...);`, `return 0;`처럼 문장 끝을 습관적으로 확인합니다. 또 `main`이라는 이름을 바꿔 버리면 시작점이 사라져 예제·과제가 통째로 어긋날 수 있으므로, 시작 함수 이름은 **`main`**으로 유지하는 것이 안전합니다.

**이 챕터의 범위**  
**함수에 대한 자세한 내용**(선언·정의 분리, 지역·전역·`static`, 재귀 등)은 **[Chapter 09 함수](./09-functions.md)**에서 배웁니다. **`int`와 다른 자료형**, 변수·연산·입출력 형식 지정자를 본격적으로 정리하는 것은 **[Chapter 03 변수와 연산자](./03-variables-operators.md)**에서 다룹니다. 지금은 **`main`이 시작점**이고 **`printf`는 호출해서 쓰는 도구**라는 흐름만 굳혀 두면 됩니다.


### 02-2 주석이란?

주석은 컴퓨터가 실행하지 않고, **사람이 읽기 위한 메모**입니다. 한 줄 주석은 `//`, 여러 줄 주석은 `/* ... */`를 씁니다. 초보 단계에서는 “왜 이렇게 썼는지”를 짧게 남겨 두면 나중에 복습할 때 시간이 많이 절약됩니다.

```c
#include <stdio.h>

int main() {
    // 한 줄 주석: 아래 문장은 화면에 인사말을 출력합니다.
    printf("안녕하세요!\n");

    /*
      여러 줄 주석:
      프로그램의 목적, 작성자, 수정 날짜처럼
      길게 설명할 내용을 정리할 때 사용합니다.
    */
    return 0;
}

//main 밖에도 자유롭게 주석을 작성할 수 있습니다.
```

### 02-3 `printf`로 다양한 텍스트 출력

`printf`는 콘솔(검은 창에 가까운 화면)에 글자를 출력할 때 가장 기본으로 쓰는 함수입니다. `\n`은 줄바꿈을 뜻하며, 출력을 여러 줄로 나누어 읽기 좋게 만들 때 자주 붙입니다.

이 챕터에서는 **문자열과 줄바꿈**에 집중합니다. `02-1`에서 `printf`의 `%d`를 잠깐 본 적이 있다면, **`%f` 같은 다른 형식**과 출력 규칙을 체계적으로 정리하는 것은 **[Chapter 03 변수와 연산자](./03-variables-operators.md)**에서 이어집니다.

```c
#include <stdio.h>

int main() {
    printf("안녕하세요.\n");
    printf("C 언어를 시작합니다.\n");
    printf("줄바꿈 연습 중입니다.\n");

    return 0;
}
```

**예상 출력**

```text
안녕하세요.
C 언어를 시작합니다.
줄바꿈 연습 중입니다.
```
