## 본문

### 13-1 포인터와 배열의 관계
- 배열 이름은 대부분의 표현식에서 첫 번째 원소 주소로 변환됩니다.
- `arr`와 `&arr[0]`는 같은 주소 값을 가리킵니다.

### 13-2 포인터 연산
- `p + 1`은 1바이트가 아니라 "해당 타입 크기만큼" 이동합니다.
- `int*`는 보통 4바이트 단위로, `char*`는 1바이트 단위로 이동합니다.

### 13-3 상수 문자열 포인터
- 문자열 리터럴은 읽기 전용 영역에 있을 수 있으므로 수정하면 안 됩니다.
- `const char *msg = "Hello";` 형태를 권장합니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 문자열 리터럴 | 코드에 직접 적는 문자열 값 (예: `"Hello"`) |
| 읽기 전용 영역 | 프로그램 실행 중 수정하면 안 되는 메모리 영역 |

### 13-4 포인터 배열
- 포인터 배열은 "포인터 여러 개를 담는 배열"입니다.
- 문자열 목록 처리(`char *names[]`)에 자주 쓰입니다.

핵심 비교:
| 표현 | 의미 |
|---|---|
| `arr` | 배열 첫 원소 주소 |
| `&arr[0]` | 첫 원소 주소(동일) |
| `p[i]` | `*(p + i)`와 동일 |

예시 코드:
```c
#include <stdio.h>

int main(void) {
    int arr[3] = {10, 20, 30};
    int *p = arr;
    printf("%d %d %d\n", p[0], *(p + 1), *(p + 2));

    const char *names[] = {"Kim", "Lee", "Park"};
    printf("%s %s %s\n", names[0], names[1], names[2]);
    return 0;
}
```

예상 출력:
```text
10 20 30
Kim Lee Park
```

   - 출력: 정렬된 문자열 목록

- 배열 순회: `for (i = 0; i < n; i++)`
- 포인터 접근: `*(p + i)` 또는 `p[i]`
