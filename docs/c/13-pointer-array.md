## 본문

### 13-1 포인터와 배열의 관계

배열 이름은 대부분의 표현식에서 **첫 번째 원소의 주소**로 해석됩니다. 그래서 `arr`와 `&arr[0]`은 같은 주소 값을 가리키는 경우가 많습니다.

### 13-2 포인터 연산

`p + 1`은 메모리에서 1바이트만 이동하는 것이 아니라, **포인터가 가리키는 타입의 크기만큼** 한 칸 이동합니다. `int *`는 보통 4바이트씩, `char *`는 1바이트씩 건너뛰는 식으로 이해하면 됩니다.

### 13-3 상수 문자열 포인터

문자열 리터럴(코드에 직접 적은 `"Hello"` 같은 값)은 **읽기 전용** 영역에 놓일 수 있어 수정하면 안 됩니다. `const char *msg = "Hello";`처럼 **읽기 전용** 의도를 타입에 드러내는 편이 안전합니다.

| 용어 | 의미 |
|---|---|
| 문자열 리터럴 | 코드에 직접 적는 문자열 값 (예: `"Hello"`) |
| 읽기 전용 영역 | 실행 중 수정하면 안 되는 메모리 영역 |

### 13-4 포인터 배열

포인터 배열은 **포인터 여러 개를 담는 배열**입니다. `char *names[]`처럼 여러 문자열의 시작 주소를 한꺼번에 들고 있을 때 자주 씁니다.

| 표현 | 의미 |
|---|---|
| `arr` | 배열 첫 원소 주소 |
| `&arr[0]` | 첫 원소 주소(동일) |
| `p[i]` | `*(p + i)`와 동일 |

```c
#include <stdio.h>

int main(void) {
    int arr[3] = {10, 20, 30};
    int *p = arr;
    printf("%d %d %d\n", p[0], *(p + 1), *(p + 2));

    const char *names[] = {"Kim", "Lee", "Park"};
    printf("%s %s %s\n", names[0], names[1], names[2]);
    return 0;
}
```

**예상 출력**

```text
10 20 30
Kim Lee Park
```

**문제 2**

### 정답 포인트

배열 순회는 `for (i = 0; i < n; i++)` 형태로 범위를 맞추고, 포인터 접근은 `*(p + i)`와 `p[i]`가 같은 그림임을 이용하면 됩니다.
