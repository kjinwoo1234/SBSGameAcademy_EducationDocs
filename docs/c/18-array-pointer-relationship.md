# Chapter 18 다차원 배열과 포인터의 관계

## 학습 목표
- 2차원 배열 이름의 타입을 정확히 해석한다.
- 포인터와 배열 선언의 함정 포인트를 파악한다.

## 세부 주제
### 18-1 2차원 배열 이름의 포인터형
- `int (*)[N]` 형태 이해

### 18-2 2차원 배열 이름의 특성과 주의사항
- 함수 인자 전달 시 열 크기 필요성

## 실습 체크리스트
- 2차원 배열을 함수에 전달해 합계를 구하는 코드를 작성한다.

## 본문

### 18-1 2차원 배열 이름의 포인터형
- `int arr[2][3]`의 배열 이름 `arr` 타입은 `int (*)[3]`에 가깝습니다.
- 즉 "int 3개짜리 배열을 가리키는 포인터"입니다.

### 18-2 2차원 배열 이름의 특성과 주의사항
- 함수 인자로 2차원 배열을 받을 때 열 크기를 명시해야 합니다.
- 예: `void print_map(int arr[][3], int row)`

함수 인자 선언 예시:
| 선언 | 의미 |
|---|---|
| `int arr[][3]` | 열 크기 3 고정 |
| `int (*arr)[3]` | "int[3]" 배열을 가리키는 포인터 |

예시 코드:
```c
#include <stdio.h>

void print_sum(int arr[][3], int row) {
    int sum = 0;
    for (int r = 0; r < row; r++)
        for (int c = 0; c < 3; c++)
            sum += arr[r][c];
    printf("sum: %d\n", sum);
}

int main(void) {
    int data[2][3] = {{1,2,3},{4,5,6}};
    print_sum(data, 2);
    return 0;
}
```

예상 출력:
```text
sum: 21
```
