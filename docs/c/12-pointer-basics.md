### 12-1 포인터 개념

**주소(address)**는 메모리 안에서 데이터가 놓인 칸을 구분하기 위한 위치 번호입니다. **포인터(pointer)**는 일반 숫자 값이 아니라 이런 주소를 저장하는 변수입니다. `int *p`는 “`int`형 데이터가 있는 위치의 주소를 담는 포인터”라는 뜻으로 읽습니다. 포인터의 타입과 가리키는 대상의 타입은 **맞추는 것**이 원칙입니다.

| 용어 | 의미 |
|---|---|
| 메모리(memory) | 프로그램 실행 중 데이터가 저장되는 공간 |
| 주소(address) | 메모리 안에서 데이터가 저장된 위치 번호 |
| 포인터(pointer) | 주소를 저장하는 변수 |

주소가 필요한 대표적인 이유는 다음과 같습니다. 함수 안에서 **원본**을 바꾸려면 복사본이 아니라 원본이 있는 위치를 알아야 하고, 배열·문자열은 **시작 주소**를 기준으로 다음 칸으로 이동하며, 동적 메모리는 실행 중에 잡은 블록의 위치를 포인터에 저장해 두었다가 나중에 쓰거나 돌려줍니다.

| 상황 | 주소가 필요한 이유 |
|---|---|
| 함수에서 원본 값 변경 | 값 복사본이 아니라 원본 위치를 알아야 직접 수정 가능 |
| 배열/문자열 순회 | 첫 위치 주소를 기준으로 다음 원소에 접근 가능 |
| 동적 메모리 사용 | 실행 중 확보한 메모리 위치를 저장해 재사용/해제 가능 |

### 12-2 `&`, `*` 연산자

`&`는 변수의 **주소**를 구할 때 씁니다. `*`는 포인터가 가리키는 주소로 가서 **실제 값**을 읽거나 바꿀 때 씁니다(역참조). 주소를 화면에 찍어 보려면 `printf`의 `%p`를 쓸 수 있는데, 이때는 `(void *)`로 형을 맞춰 출력하는 습관을 들이면 좋습니다.

| 기호 | 의미 | 예시 |
|---|---|---|
| `&` | 주소 얻기 | `p = &num;` |
| `*` | 역참조(값 읽기/수정) | `*p = 20;` |

```c
#include <stdio.h>

int main(void) {
    int num = 10;
    int *p = &num;

    printf("num: %d\n", num);
    printf("&num: %p\n", (void *)&num);
    printf("p   : %p\n", (void *)p);
    printf("&num == p: %d\n", (&num == p));

    *p = 20;
    printf("num after *p = 20: %d\n", num);
    return 0;
}
```

**예상 출력(주소는 실행마다 다름)**

```text
num: 10
&num: 0x...
p   : 0x...
&num == p: 1
num after *p = 20: 20
```

### 연습문제

**문제 1**
- 문제: 정수 변수 `num`을 선언하고, 포인터를 이용해 값을 두 배로 바꾸세요.
- 입력: 정수 `num` 1개
- 출력: 변경 전 값과 변경 후 값
- 조건(힌트): 값 변경은 반드시 `*포인터`를 통해 수행하세요.

**문제 2**
- 문제: 정수 변수 두 개를 선언한 뒤, 포인터를 사용해 합계를 출력하세요.
- 입력: 정수 2개
- 출력: 합계 1개
- 조건(힌트): 변수 이름을 직접 더하지 말고 역참조(`*p`)로만 계산하세요.

### 정답 포인트

포인터 선언은 `자료형 *이름` 형태이고, 값을 바꿀 때는 `*포인터 = 새값`입니다.
