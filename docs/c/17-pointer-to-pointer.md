# Chapter 17 포인터의 포인터

## 학습 목표
- 이중 포인터가 필요한 상황을 이해한다.
- 다중 포인터 선언을 해석할 수 있다.

## 세부 주제
### 17-1 이중 포인터
- 주소의 주소와 간접 접근 단계

### 17-2 다중 포인터의 필요성
- 함수에서 포인터 자체를 변경해야 하는 경우

## 실습 체크리스트
- 이중 포인터를 이용해 문자열 포인터를 교체하는 함수를 작성한다.

## 본문

### 17-1 이중 포인터
- 이중 포인터(`type **pp`)는 "포인터를 가리키는 포인터"입니다.
- 한 단계 더 간접 접근이 필요할 때 사용합니다.

### 17-2 다중 포인터의 필요성
- 함수 안에서 "포인터 변수 자체"를 바꾸고 싶을 때 이중 포인터가 필요합니다.
- 문자열 포인터 교체, 동적 메모리 재할당 함수에서 자주 등장합니다.

예시 코드:
```c
#include <stdio.h>

void change_text(const char **p) {
    *p = "Changed";
}

int main(void) {
    const char *msg = "Original";
    change_text(&msg);
    printf("%s\n", msg);
    return 0;
}
```

연습문제:
1. `int*`를 인자로 받아 다른 배열 시작 주소로 바꾸는 함수를 작성하세요.
2. 문자열 포인터를 함수에서 교체해보세요.

정답 포인트:
- 포인터 자체를 수정하려면 `type **` 필요
- 호출 시 `&포인터변수` 전달
