## 본문

### 26-1 전처리기와 매크로

**전처리기(preprocessor)**는 컴파일이 본격적으로 돌기 전에 소스를 한 번 손보는 단계입니다. `#define` 매크로는 같은 패턴이 반복될 때 짧은 이름으로 묶어 줄이거나, 컴파일 옵션에 따라 코드를 바꿀 때 자주 씁니다.

### 26-2 대표 선행처리(전처리) 명령문

`#include`는 헤더를 끼워 넣고, `#define`과 `#undef`는 매크로를 정의하거나 해제합니다. `#pragma`는 컴파일러에게 특별한 힌트를 줄 때 쓰는 지시문입니다.

### 26-3 조건부 컴파일

`#if`, `#ifdef`, `#ifndef`는 **빌드 조건**에 따라 소스 일부를 포함할지 말지 정합니다. 디버그 로그를 켜고 끄는 용도로 자주 등장합니다.

### 26-4 매개변수 결합/문자열화

`##`는 전처리 단계에서 **토큰을 이어 붙여** 새 이름을 만들 때 쓰고, `#`는 매크로 인자를 **문자열 리터럴**로 바꿀 때 씁니다.

| 용어 | 의미 |
|---|---|
| 토큰 | 컴파일러가 인식하는 코드의 최소 단위(식별자, 숫자, 기호 등) |
| 토큰 결합(`##`) | 두 토큰을 붙여 새 토큰으로 만드는 전처리 연산 |
| 문자열화(`#`) | 매크로 인자를 문자열 리터럴로 바꾸는 전처리 연산 |

| 지시문 | 역할 |
|---|---|
| `#include` | 헤더 파일 포함 |
| `#define` | 매크로 정의 |
| `#undef` | 매크로 해제 |
| `#if/#ifdef/#ifndef` | 조건부 컴파일 |

```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))
#define LOG(msg) printf("[LOG] %s\n", msg)

int main(void) {
    int n = 4;
    printf("%d\n", SQUARE(n));
    LOG("start");
    return 0;
}
```

**예상 출력**

```text
16
[LOG] start
```
