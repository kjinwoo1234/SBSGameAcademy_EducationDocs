# Chapter 26 매크로와 전처리기

## 학습 목표
- 전처리 단계의 역할을 이해한다.
- 매크로의 장단점과 안전한 사용법을 익힌다.

## 세부 주제
### 26-1 전처리기와 매크로
- `#define` 기반 치환

### 26-2 대표 선행처리(전처리) 명령문
- `#include`, `#undef`, `#pragma` 개요

### 26-3 조건부 컴파일
- `#if`, `#ifdef`, `#ifndef` 활용

### 26-4 매개변수 결합/문자열화
- `##`, `#` 연산자

## 실습 체크리스트
- 디버그 빌드 전용 로그 매크로를 정의해 사용한다.

## 본문

### 26-1 전처리기와 매크로
- 전처리기는 컴파일 전에 소스 코드를 미리 가공합니다.
- `#define` 매크로는 반복되는 코드 패턴을 줄이는 데 유용합니다.

### 26-2 대표 선행처리(전처리) 명령문
- `#include`: 헤더 포함
- `#define`, `#undef`: 매크로 정의/해제
- `#pragma`: 컴파일러 지시

### 26-3 조건부 컴파일
- `#if`, `#ifdef`, `#ifndef`로 빌드 조건에 따라 코드 포함 여부를 제어합니다.
- 디버그 로그 ON/OFF에 자주 사용됩니다.

### 26-4 매개변수 결합/문자열화
- `##`는 토큰 결합, `#`는 문자열화 연산입니다.

예시 코드:
```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))
#define LOG(msg) printf("[LOG] %s\n", msg)

int main(void) {
    int n = 4;
    printf("%d\n", SQUARE(n));
    LOG("start");
    return 0;
}
```

1. 두 수 최댓값 매크로 `MAX(a, b)` 작성

정답 포인트:
- 매크로 매개변수는 괄호로 감싸기
- 부작용(예: `x++`) 인자 사용 주의
