## 실습 체크리스트

## 본문

### 26-1 전처리기와 매크로
- 전처리기는 컴파일 전에 소스 코드를 미리 가공합니다.
- `#define` 매크로는 반복되는 코드 패턴을 줄이는 데 유용합니다.

### 26-2 대표 선행처리(전처리) 명령문
- `#include`: 헤더 포함
- `#define`, `#undef`: 매크로 정의/해제
- `#pragma`: 컴파일러 지시

### 26-3 조건부 컴파일
- `#if`, `#ifdef`, `#ifndef`로 빌드 조건에 따라 코드 포함 여부를 제어합니다.
- 디버그 로그 ON/OFF에 자주 사용됩니다.

### 26-4 매개변수 결합/문자열화
- `##`는 토큰 결합, `#`는 문자열화 연산입니다.

용어 설명:
| 용어 | 의미 |
|---|---|
| 토큰 | 컴파일러가 인식하는 코드의 최소 단위(식별자, 숫자, 기호 등) |
| 토큰 결합(`##`) | 두 토큰을 붙여 새 토큰으로 만드는 전처리 연산 |
| 문자열화(`#`) | 매크로 인자를 문자열 리터럴로 바꾸는 전처리 연산 |

전처리기 지시문 정리:
| 지시문 | 역할 |
|---|---|
| `#include` | 헤더 파일 포함 |
| `#define` | 매크로 정의 |
| `#undef` | 매크로 해제 |
| `#if/#ifdef/#ifndef` | 조건부 컴파일 |

예시 코드:
```c
#include <stdio.h>

#define SQUARE(x) ((x) * (x))
#define LOG(msg) printf("[LOG] %s\n", msg)

int main(void) {
    int n = 4;
    printf("%d\n", SQUARE(n));
    LOG("start");
    return 0;
}
```

예상 출력:
```text
16
[LOG] start
```
