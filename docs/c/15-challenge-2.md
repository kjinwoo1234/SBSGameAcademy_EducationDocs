# Chapter 15 도전! 프로그래밍 2

## 학습 목표
- 콘솔 제어와 함수 분할을 실전 문제에 적용한다.
- 작은 프로젝트 단위로 기능을 조립해본다.

## 세부 주제
### 15-1 커서 이동 함수
- 콘솔 좌표 제어 기본 아이디어

### 15-2 글자 색상 변경 함수
- 출력 스타일 변경 함수 분리

### 15-3 글자 전부 지우는 함수
- 화면 초기화와 출력 갱신

### 15-4 머드게임 만들기
- 입력/출력/상태 관리 결합

## 실습 체크리스트
- 3개 이상의 유틸 함수로 콘솔 미니게임을 구성한다.

## 본문

### 15-1 커서 이동 함수
- 콘솔 좌표 이동 함수(`gotoxy`)를 만들면 UI 표현이 좋아집니다.
- 화면을 다시 그릴 위치를 제어할 수 있어 게임 느낌을 만들기 좋습니다.

### 15-2 글자 색상 변경 함수
- 색상 출력 함수(`setColor`)를 분리하면 경고/상태 메시지를 직관적으로 표현할 수 있습니다.

### 15-3 글자 전부 지우는 함수
- 화면 지우기 함수(`clearScreen`)를 통일해 쓰면 프레임 갱신이 쉬워집니다.

### 15-4 머드게임 만들기
- 입력 처리, 상태 업데이트, 렌더링을 함수로 분리하면 유지보수가 쉬워집니다.

권장 함수 분리:
- `init_game()`: 초기화
- `handle_input()`: 입력 처리
- `update_game()`: 상태 갱신
- `render_game()`: 화면 출력

루프 구조 요약:
| 단계 | 역할 |
|---|---|
| 입력 | 키/명령 수집 |
| 업데이트 | 캐릭터/상태 계산 |
| 렌더링 | 화면 반영 |

예시 코드(구조 템플릿):
```c
#include <stdio.h>

void init_game(void) { printf("init\n"); }
void handle_input(void) { /* 입력 처리 */ }
void update_game(void) { /* 상태 갱신 */ }
void render_game(void) { /* 화면 출력 */ }

int main(void) {
    init_game();
    for (int frame = 0; frame < 3; frame++) {
        handle_input();
        update_game();
        render_game();
    }
    return 0;
}
```
- 이 템플릿을 기반으로, 실제 게임에서는 `main` 루프에 **종료 조건**(예: ESC 입력)과 **입력 실패 처리**(잘못된 입력 시 재입력)를 추가해 완성도를 높이세요.

예상 출력:
```text
init
```

연습문제:
1. 문제 1 - 상태판 출력 도구 만들기
   - 문제: 커서 이동 함수와 색상 출력 함수를 만들어 상태판(UI)을 출력하세요.
   - 입력: 플레이어 이름, HP, EXP(코드 내부 상수 가능)
   - 출력: 콘솔 상태판 1회
   - 조건: 출력 관련 코드를 최소 2개 함수로 분리
2. 문제 2 - 게임 루프 함수 분리
   - 문제: 플레이어 HP/EXP 갱신 로직을 함수로 분리하고 메인 루프에서 호출하세요.
   - 입력: 이벤트 값(예: 몬스터 처치 여부)
   - 출력: 갱신된 상태 출력
   - 조건: `handle_input`, `update_game`, `render_game` 구조 유지

정답 포인트:
- 전역변수 최소화
- 기능별 함수 분리
- 입력/업데이트/출력 루프 구조 유지
