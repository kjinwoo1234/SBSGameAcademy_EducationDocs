## 본문

### 20-1 스트림과 데이터 이동

표준 입력·표준 출력·표준 오류(`stdin`, `stdout`, `stderr`)는 프로그램과 바깥(콘솔·파이프 등) 사이를 오가는 **스트림(stream, 흐름)**입니다. 대부분의 콘솔 입출력은 이 통로를 통해 이루어진다고 보면 됩니다.

### 20-2 문자 단위 입출력

`getchar`와 `putchar`는 **문자 한 개** 단위로 읽고 쓸 때 씁니다. 한 글자씩 상태를 보거나 필터를 만들 때 작은 단위 제어가 필요하면 선택지가 됩니다.

### 20-3 문자열 단위 입출력

`fgets`는 한 줄을 읽되 **최대 길이**를 정해 둘 수 있어, 공백이 섞인 문장을 받을 때 `scanf("%s")`보다 안전한 경우가 많습니다. `puts`는 문자열을 출력한 뒤 **자동으로 줄바꿈**을 붙입니다.

### 20-4 표준 입출력과 버퍼

입력은 내부 **버퍼(buffer, 잠시 쌓아 두는 공간)**를 거친 뒤 함수에 전달되는 경우가 많습니다. 그래서 이전 입력에서 남은 개행 문자(`\n`) 때문에 다음 `scanf`나 `getchar`가 기대와 다르게 동작하는 일이 자주 생깁니다. 어떤 함수를 어떤 순서로 부를지, 필요하면 버퍼를 비울지까지 습관으로 잡아 두면 좋습니다.

### 20-5 문자열 관련 함수

`strlen`, `strcpy`, `strcat`, `strcmp`는 문자열 처리에서 가장 자주 만나는 함수들입니다. 복사나 이어붙이기는 **대상 배열의 남은 칸**을 항상 확인해야 하며, 크기를 넘기면 **오버플로우(overflow, 넘침)**로 이어질 수 있습니다.

| 용어 | 의미 |
|---|---|
| 버퍼(buffer) | 입력/출력을 잠시 저장해두는 메모리 공간 |
| 오버플로우(overflow) | 저장 공간을 초과해 데이터가 넘치는 오류 |

| 함수 | 용도 |
|---|---|
| `strlen` | 문자열 길이 계산 |
| `strcmp` | 문자열 비교 |
| `strcpy` | 문자열 복사 |
| `strcat` | 문자열 이어붙이기 |

```c
#include <stdio.h>
#include <string.h>

int main(void) {
    char a[30], b[30];
    fgets(a, sizeof(a), stdin);
    fgets(b, sizeof(b), stdin);

    a[strcspn(a, "\n")] = '\0';
    b[strcspn(b, "\n")] = '\0';

    printf("len(a): %zu\n", strlen(a));
    printf("compare: %d\n", strcmp(a, b));
    return 0;
}
```

**예상 출력(입력: `abc`, `abd`)**

```text
len(a): 3
compare: -1
```

### 연습문제

**문제 1**
- 문제: 두 문자열을 입력받아 각각의 길이를 출력하세요.
- 입력: 공백 포함 가능 문자열 2줄
- 출력: 첫 번째 문자열 길이, 두 번째 문자열 길이
- 조건(힌트): `fgets`로 입력받고, 개행 문자(`\n`)를 제거한 뒤 `strlen`으로 길이를 계산하세요.

**문제 2**
- 문제: 두 문자열이 같으면 `same`, 다르면 `different`를 출력하세요.
- 입력: 문자열 2개
- 출력: `same` 또는 `different`
- 조건(힌트): 포인터 비교 `==` 대신 `strcmp` 결과가 `0`인지 확인하세요.

### 정답 포인트

`fgets`로 읽은 뒤에는 끝의 개행을 정리하고, 문자열 내용 비교는 `strcmp`로 처리합니다.
