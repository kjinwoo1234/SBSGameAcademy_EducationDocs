## 본문

### 08-1 조건적 실행과 분기

조건문은 상황에 따라 **서로 다른 코드**를 실행하게 만드는 도구입니다. `if`, `else if`, `else`를 이어 붙이면 여러 단계의 조건을 위에서 아래로 검사할 수 있습니다. 조건식을 단순하고 읽기 쉽게 쓸수록 나중에 버그를 찾기도 쉬워집니다.

- **어디서 쓰나?** 점수 계산, 로그인 결과 처리, 입력값 검증처럼 결과가 여러 갈래로 나뉘는 상황에서 사용합니다.
- **왜 중요한가?** 조건 순서가 잘못되면 의도한 분기가 실행되지 않아, 맞는 입력에도 잘못된 결과가 나올 수 있습니다.
- **자주 하는 실수** `if (score >= 60)`를 먼저 쓰고 아래에 `else if (score >= 90)`를 두면 A 등급 조건이 도달하지 못합니다.
- **오류 방지 팁** 더 엄격한 조건(범위가 좁은 조건)부터 위에 배치합니다.

```c
#include <stdio.h>

int main(void) {
    int score = 95;

    if (score >= 90) {
        printf("A\n");
    } else if (score >= 80) {
        printf("B\n");
    } else {
        printf("C 이하\n");
    }
    return 0;
}
```

**코드 해석**
- 점수 구간처럼 범위를 나눌 때는 `if -> else if -> else` 순서로 위에서부터 검사합니다.
- 위 코드는 `95`이므로 첫 번째 조건에서 바로 `A`를 출력합니다.

### 08-2 `continue` / `break`

`continue`는 **지금 반복**에서 남은 본문을 건너뛰고 곧바로 다음 반복으로 넘어갑니다. `break`는 반복문이나 `switch` 전체를 **즉시 종료**합니다. 두 키워드는 흐름을 세밀하게 조절할 때 유용하지만, 너무 많이 쓰면 읽는 사람이 따라가기 어려워질 수 있으므로 필요한 곳에만 쓰는 편이 좋습니다.

- **발생 조건** 반복 중 특정 값은 건너뛰고 싶을 때 `continue`, 목표를 달성해 더 볼 필요가 없을 때 `break`를 사용합니다.
- **왜 문제인가?** `continue` 위치가 앞쪽이면 아래 핵심 처리 코드가 실행되지 않아 결과가 비어 보일 수 있습니다.
- **어떻게 막나?** `continue`/`break` 직전에 현재 반복 값과 분기 이유를 출력해 흐름을 먼저 확인합니다.

```c
#include <stdio.h>

int main(void) {
    int i;
    for (i = 1; i <= 10; i++) {
        if (i % 3 == 0) {
            continue;  // 3의 배수는 건너뜀
        }
        if (i >= 8) {
            break;     // 8 이상이면 반복 종료
        }
        printf("%d ", i);
    }
    return 0;
}
```

**예상 출력**

```text
1 2 4 5 7
```

### 08-3 `switch`와 `goto`

`switch`는 **하나의 값**을 기준으로 여러 경우를 나눌 때 읽기 좋은 편입니다. 각 `case` 끝에서 `break`를 빼먹으면 다음 `case`까지 이어 실행되는 **fall-through**가 생길 수 있으므로, 의도가 아니라면 `break`를 넣는 습관이 안전합니다.

`goto`는 코드 실행을 **레이블(label)**이 붙은 위치로 즉시 옮깁니다. 레이블은 `이름:` 형태로 붙이고, `goto 이름;`으로 점프합니다. 매우 제한적인 탈출이나 예외적인 구조에서 가끔 쓰이지만, 남용하면 흐름이 꼬여 유지보수가 어려워지므로 학습 단계에서는 “있다” 정도로만 이해해도 됩니다.

```c
#include <stdio.h>

int main(void) {
    int n = 0;

start:  // 레이블
    n++;
    printf("n = %d\n", n);
    if (n < 3) goto start;  // start 레이블 위치로 이동

    return 0;
}
```

분기 방식을 비교하면 다음과 같습니다.

| 방식 | 적합한 상황 | 장점 | 주의점 |
|---|---|---|---|
| `if / else if / else` | 범위 조건, 복합 조건 | 유연함 | 조건식이 길어지면 가독성 저하 |
| `switch` | 값 1개에 대한 다중 분기 | 읽기 쉬움 | `break` 누락 시 fall-through |
| `goto` | 매우 제한적인 탈출 흐름 | 즉시 이동 가능 | 남용 시 유지보수 어려움 |

**fall-through(폴스루, 다음 case로 연속 실행)**는 `switch`에서 `break`를 쓰지 않아 다음 `case` 코드까지 연달아 실행되는 현상입니다. 의도한 경우가 아니면 버그 원인이 되기 쉽습니다.

**오류 방지 3요소 (`break` 누락)**
- **발생 조건**: `switch`의 `case` 끝에 `break`를 작성하지 않았을 때
- **영향**: 다음 `case` 코드까지 함께 실행되어 메뉴/상태가 잘못 처리될 수 있음
- **해결 원리**: `break`가 현재 `case` 실행을 끝내고 `switch` 블록 밖으로 제어 흐름을 이동시킴

```c
#include <stdio.h>

int main(void) {
    int score = 85;

    if (score >= 90) printf("A\n");
    else if (score >= 80) printf("B\n");
    else printf("C 이하\n");

    int menu = 2;
    switch (menu) {
        case 1: printf("시작\n"); break;
        case 2: printf("설정\n"); break;
        default: printf("종료\n"); break;
    }
    return 0;
}
```

`if`는 점수 구간을 위에서부터 나누어 검사하고, `switch`는 `menu` 값 하나로 분기합니다. 각 `case` 뒤의 `break`는 그 분기만 끝내고 `switch` 밖으로 나가게 합니다.

**예상 출력**

```text
B
설정
```

### 연습문제

**문제 1**
- 문제: 점수(`0~100`)를 입력받아 A/B/C/F를 출력하세요.
- 입력: 점수 정수 1개
- 출력: `A`, `B`, `C`, `F` 중 하나
- 조건(힌트): `if - else if - else`를 사용하고, 높은 점수 조건부터 검사하세요.

**문제 2**
- 문제: 숫자 1~3을 입력받아 각 메뉴 이름을 `switch`로 출력하세요.
- 입력: 메뉴 번호 정수 1개
- 출력: `1: 시작`, `2: 설정`, `3: 종료`, 그 외는 `잘못된 입력`
- 조건(힌트): 각 `case` 끝에 `break`를 넣고, `default`를 포함하세요.

**문제 3**
- 문제: 1부터 증가시키며 출력하되, 3의 배수는 건너뛰고(`continue`) 20 이상이면 종료(`break`)하는 코드를 작성하세요.
- 입력: 해당 없음
- 출력: 조건에 맞게 필터링된 숫자 목록
- 조건(힌트): `for` 또는 `while` 중 하나를 사용하고, `continue`와 `break`를 모두 포함하세요.

### 정답 포인트

- 문제 1: `if (score >= 90)`, `else if (score >= 80)`, `else if (score >= 70)`, `else` 순서로 작성합니다.
- 문제 2: `switch (menu)`에 `case 1`, `case 2`, `case 3`, `default`를 만들고 각 `case` 끝에 `break`를 둡니다.
- 문제 3: 반복문 안에서 `if (i % 3 == 0) continue;`와 `if (i >= 20) break;`를 사용해 흐름을 분기합니다.
- 공통 점검: 결과가 예상과 다르면 조건 순서와 `break` 누락 여부를 먼저 확인합니다.
