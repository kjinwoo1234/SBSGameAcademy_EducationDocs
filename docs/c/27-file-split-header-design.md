## 본문

### 27-1 파일 분할

기능마다 `.c` 파일을 나누면 **역할이 분명**해져 유지보수와 협업이 쉬워집니다. 예를 들어 `main.c`, `calc.c`, `calc.h`처럼 “시작점”, “계산 구현”, “계산 약속(선언)”으로 나눌 수 있습니다.

### 27-2 다중 파일 컴파일과 `static`

여러 `.c` 파일을 각각 컴파일한 뒤 **링크(link)**로 하나의 실행 파일을 만듭니다. `static` 함수는 **이 파일 안에서만** 보이게 막아, 바깥 이름과 충돌하지 않게 할 때 씁니다.

### 27-3 헤더파일 디자인/활용

헤더에는 **선언**(함수 원형, 타입)을 두고, 실제 코드는 `.c`에 둡니다. 같은 헤더가 여러 번 포함될 때 생기는 중복 정의 오류를 막으려면 **include guard**( `#ifndef` … `#define` … `#endif` )를 두는 것이 일반적입니다.

| 파일 | 역할 |
|---|---|
| `*.h` | 선언(인터페이스) |
| `*.c` | 구현(동작 코드) |
| `main.c` | 프로그램 시작/조합 |

**인터페이스(interface)**는 다른 파일이 쓸 수 있도록 밖에 내놓은 약속(함수·타입 선언)이라고 보면 됩니다. **include guard**는 헤더가 중복 포함될 때 같은 내용이 두 번 들어가지 않게 막는 전처리 패턴이고, **링크**는 컴파일된 목표 파일들을 실행 파일 하나로 묶는 단계입니다.

```c
// calc.h
#ifndef CALC_H
#define CALC_H
int add(int a, int b);
#endif
```

```c
// calc.c
#include "calc.h"
int add(int a, int b) { return a + b; }
```

```c
// main.c
#include <stdio.h>
#include "calc.h"

int main(void) {
    printf("3 + 4 = %d\n", add(3, 4));
    return 0;
}
```

**예상 출력**

```text
3 + 4 = 7
```

### 연습문제

**문제 1**
- 문제: `sub`, `mul` 함수를 각각 소스와 헤더에 맞게 나누고, `add`와 함께 `main`에서 호출해 결과를 출력하세요.
- 입력: 피연산자 정수 2개
- 출력: `add`, `sub`, `mul` 결과
- 조건(힌트): 헤더에는 선언만 두고 구현은 `.c` 파일에 작성하세요.

**문제 2**
- 문제: `main.c`와 `calc.c`를 함께 컴파일/링크해 실행 파일을 만들고 동작을 확인하세요.
- 입력: 소스 파일 2개와 헤더 파일 1개
- 출력: 실행 결과 화면(연산 결과)
- 조건(힌트): include guard를 유지하고 링크 오류 없이 빌드가 끝나야 합니다.

### 정답 포인트

헤더에는 선언만, 소스에는 정의만 두고, 헤더마다 include guard를 붙이면 됩니다.
