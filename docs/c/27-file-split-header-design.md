# Chapter 27 파일 분할과 헤더파일 디자인

## 학습 목표
- 소스/헤더 분리 구조를 이해한다.
- 다중 파일 프로젝트의 컴파일 흐름을 익힌다.

## 세부 주제
### 27-1 파일 분할
- 기능 단위 소스 분리 기준

### 27-2 다중 파일 컴파일과 `static`
- 내부 연결과 외부 연결 개념

### 27-3 헤더파일 디자인/활용
- include guard, 선언/정의 분리 원칙

## 실습 체크리스트
- 2개 이상의 소스 파일로 계산기 프로젝트를 분리 구성한다.

## 본문

### 27-1 파일 분할
- 기능별로 `.c` 파일을 나누면 유지보수와 협업이 쉬워집니다.
- 예: `main.c`, `calc.c`, `calc.h`

### 27-2 다중 파일 컴파일과 `static`
- 여러 `.c` 파일을 함께 컴파일/링크해 하나의 실행 파일을 만듭니다.
- `static` 함수는 현재 파일 내부에서만 사용하도록 제한할 수 있습니다.

### 27-3 헤더파일 디자인/활용
- 헤더에는 선언(함수 원형, 타입)을 두고, 구현은 `.c`에 둡니다.
- 중복 포함 방지를 위해 include guard를 사용합니다.

예시 구조:
```c
// calc.h
#ifndef CALC_H
#define CALC_H
int add(int a, int b);
#endif
```

```c
// calc.c
#include "calc.h"
int add(int a, int b) { return a + b; }
```

연습문제:
1. `sub`, `mul` 함수를 별도 파일로 분리
2. 헤더 파일에 함수 선언만 남기고 컴파일 통과시키기

정답 포인트:
- 헤더에는 선언, 소스에는 정의
- include guard 필수
