# Chapter 27 파일 분할과 헤더파일 디자인

## 학습 목표
- 소스/헤더 분리 구조를 이해한다.
- 다중 파일 프로젝트의 컴파일 흐름을 익힌다.

## 세부 주제
### 27-1 파일 분할
- 기능 단위 소스 분리 기준

### 27-2 다중 파일 컴파일과 `static`
- 내부 연결과 외부 연결 개념

### 27-3 헤더파일 디자인/활용
- include guard, 선언/정의 분리 원칙

## 실습 체크리스트
- 2개 이상의 소스 파일로 계산기 프로젝트를 분리 구성한다.

## 본문

### 27-1 파일 분할
- 기능별로 `.c` 파일을 나누면 유지보수와 협업이 쉬워집니다.
- 예: `main.c`, `calc.c`, `calc.h`

### 27-2 다중 파일 컴파일과 `static`
- 여러 `.c` 파일을 함께 컴파일/링크해 하나의 실행 파일을 만듭니다.
- `static` 함수는 현재 파일 내부에서만 사용하도록 제한할 수 있습니다.

### 27-3 헤더파일 디자인/활용
- 헤더에는 선언(함수 원형, 타입)을 두고, 구현은 `.c`에 둡니다.
- 중복 포함 방지를 위해 include guard를 사용합니다.

파일 역할 분리:
| 파일 | 역할 |
|---|---|
| `*.h` | 선언(인터페이스) |
| `*.c` | 구현(동작 코드) |
| `main.c` | 프로그램 시작/조합 |

용어 설명:
- 인터페이스(interface)는 "다른 파일이 사용할 수 있도록 외부에 공개한 약속(함수 선언/타입 선언)"을 의미합니다.
- include guard는 헤더 파일이 중복 포함되어 생기는 재정의 오류를 막기 위한 전처리 패턴입니다.
- 링크(link)는 여러 소스 파일의 컴파일 결과를 하나의 실행 파일로 묶는 과정입니다.

예시 구조:
```c
// calc.h
#ifndef CALC_H
#define CALC_H
int add(int a, int b);
#endif
```

```c
// calc.c
#include "calc.h"
int add(int a, int b) { return a + b; }
```

```c
// main.c
#include <stdio.h>
#include "calc.h"

int main(void) {
    printf("3 + 4 = %d\n", add(3, 4));
    return 0;
}
```

예상 출력:
```text
3 + 4 = 7
```

연습문제:
1. 문제 1 - 함수 파일 분리
   - 문제: `sub`, `mul` 함수를 각각 소스/헤더에 분리해 프로젝트를 구성하세요.
   - 입력: 정수 2개
   - 출력: `add`, `sub`, `mul` 결과
   - 조건: 헤더에는 선언만, 구현은 `.c` 파일에 작성
2. 문제 2 - 다중 파일 컴파일
   - 문제: `main.c + calc.c`를 함께 컴파일해 실행 파일을 만들고 동작을 확인하세요.
   - 입력: 코드 내부 상수 사용 가능
   - 출력: 계산 결과
   - 조건: include guard 유지, 링크 에러 없이 컴파일 통과

정답 포인트:
- 헤더에는 선언, 소스에는 정의
- include guard 필수
