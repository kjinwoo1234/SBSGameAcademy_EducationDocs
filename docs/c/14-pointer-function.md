## 본문

### 14-1 함수 인자로 배열 전달

C에서 배열 이름을 함수 인자로 넘기면 **첫 원소의 주소**가 전달되는 쪽으로 이해하면 됩니다. 그래서 함수 안에서는 배열의 “전체 크기”를 자동으로 알 수 없는 경우가 많아, **길이를 별도 인자**로 같이 넘기는 패턴이 안전합니다.

### 14-2 Call-by-value vs Call-by-reference

C의 기본 인자 전달은 **값 복사(Call-by-value)**입니다. 함수 안에서 매개변수를 바꿔도 호출한 쪽의 원본 변수는 그대로인 경우가 많습니다. 원본을 바꾸고 싶다면 **주소를 넘겨** 같은 메모리를 가리키게 만들어야 하며, 이를 흔히 참조에 가까운 효과라고 말합니다.

| 용어 | 의미 |
|---|---|
| Call-by-value | 값의 복사본을 전달하는 방식(원본 직접 변경 불가) |
| Call-by-reference(효과) | 주소를 전달해 원본 값을 변경하는 방식 |

### 14-3 포인터 대상 `const`

`const int *p`처럼 쓰면 “이 포인터로 가리키는 **값을 바꾸지 않겠다**”는 약속에 가깝습니다. 읽기 전용 데이터를 함수에 넘길 때 실수로 수정하는 것을 막는 데 유용합니다.

**API**는 **Application Programming Interface(응용 프로그램 프로그래밍 인터페이스)**의 약자로, 외부에서 어떤 함수를 어떻게 호출할지에 대한 약속을 뜻합니다. 이 챕터에서는 `const`를 붙여 “이 함수는 데이터를 읽기만 한다”는 **API 의도**를 분명히 하는 예를 봅니다.

| 방식 | 인자 형태 | 원본 변경 가능 |
|---|---|---|
| 값 전달 | `int x` | 불가 |
| 주소 전달 | `int *x` | 가능 |

```c
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int sum_array(const int *arr, int n) {
    int i, sum = 0;
    for (i = 0; i < n; i++) sum += arr[i];
    return sum;
}

int main(void) {
    int x = 3, y = 7;
    int nums[3] = {1, 2, 3};
    swap(&x, &y);
    printf("%d %d\n", x, y);
    printf("sum: %d\n", sum_array(nums, 3));
    return 0;
}
```

**예상 출력**

```text
7 3
sum: 6
```
