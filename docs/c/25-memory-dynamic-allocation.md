## 본문

### 25-1 메모리 구조

프로그램이 쓰는 메모리는 크게 **코드 영역**(실행할 명령), **데이터 영역**(전역·정적 변수), **스택 영역**(지역 변수와 호출 정보), **힙 영역**(`malloc` 등으로 잡는 공간)으로 나누어 이해할 수 있습니다.

| 영역 | 보통 생성 시점 | 보통 소멸 시점 |
|---|---|---|
| 코드 영역 | 프로그램 로드 시 | 프로그램 종료 시 |
| 데이터 영역 | 프로그램 시작 시 | 프로그램 종료 시 |
| 스택 영역 | 함수 호출 시 | 함수 종료 시 |
| 힙 영역 | `malloc`/`calloc`/`realloc` 호출 시 확보 | `free` 호출 시 반환 |

### 25-2 동적 할당

`malloc`은 요청한 **바이트 크기**만큼 메모리를 잡고, 내용은 초기화되지 않을 수 있습니다. `calloc`은 개수와 크기를 곱한 만큼 잡으면서 **0으로 채우는** 쪽에 가깝습니다. `realloc`은 이미 잡아 둔 블록의 크기를 바꿀 때 쓰며, 실패 시 포인터 처리에 주의해야 합니다. `free`는 더 이상 쓰지 않을 때 힙을 **돌려주는** 함수입니다.

| 함수 | 특징 |
|---|---|
| `malloc` | 초기화하지 않음 |
| `calloc` | 0으로 초기화 |
| `realloc` | 크기 변경(기존 데이터 유지 가능) |
| `free` | 메모리 반환 |

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int n = 3;
    int *arr = (int *)malloc(sizeof(int) * n);
    if (!arr) return 1;

    for (int i = 0; i < n; i++) arr[i] = (i + 1) * 10;
    for (int i = 0; i < n; i++) printf("%d ", arr[i]);
    printf("\n");

    free(arr);
    arr = NULL;
    return 0;
}
```

**예상 출력**

```text
10 20 30
```

### 연습문제

**문제 1**  
사용자에게 배열 크기 `n`을 입력받아 동적 배열을 만든 뒤 정수 `n`개를 받아 합계를 출력하세요. `malloc` 직후 `NULL` 검사와 사용 후 `free`는 필수입니다.

**문제 2**  
`realloc`으로 배열 크기를 두 배로 늘린 뒤 추가 값을 저장하고 전체를 출력하세요. `realloc`이 실패하면 원래 포인터를 잃지 않도록 임시 변수에 받는 패턴을 검토합니다.

### 정답 포인트

할당 직후에는 `NULL`인지 확인하고, `free`한 뒤에는 같은 포인터를 다시 쓰지 않도록 `NULL`을 대입하는 습관이 안전합니다.
