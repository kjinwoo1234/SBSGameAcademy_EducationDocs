## 본문

### 23-1 `typedef`
- `typedef`는 타입 이름에 별칭을 붙여 코드 가독성을 높입니다.
- 긴 선언을 간단히 만들 때 특히 유용합니다.

### 23-2 구조체 전달과 반환
- 값 전달은 복사가 발생하고, 포인터 전달은 원본 접근이 가능합니다.
- 큰 구조체는 포인터 전달이 성능상 유리한 경우가 많습니다.

### 23-3 중첩 구조체
- 구조체 멤버로 다른 구조체를 포함해 복합 데이터를 구성할 수 있습니다.

### 23-4 공용체(Union)
- 공용체는 여러 멤버가 같은 메모리를 공유합니다.
- 한 시점에는 하나의 의미로만 사용해야 안전합니다.

### 23-5 열거형(Enum)
- 열거형은 상태값을 숫자 대신 이름으로 표현해 가독성을 높입니다.

유형별 사용 목적:
| 요소 | 핵심 목적 |
|---|---|
| `typedef` | 긴 타입 선언 단순화 |
| 중첩 구조체 | 복합 데이터 표현 |
| `union` | 메모리 공유 |
| `enum` | 상태값 이름화 |

예시 코드:
```c
#include <stdio.h>

typedef struct {
    int x, y;
} Point;

enum State { IDLE, RUN, JUMP };

int main(void) {
    Point p = {10, 20};
    enum State st = RUN;
    printf("(%d, %d), state=%d\n", p.x, p.y, st);
    return 0;
}
```

예상 출력:
```text
(10, 20), state=1
```
