# C 포인터

포인터는 C 언어의 핵심 개념으로, 메모리 주소를 다루기 위한 도구입니다.  
배열, 함수, 동적 메모리와 모두 연결되므로 반드시 깊게 이해해야 합니다.

## 학습 목표
- 포인터 변수와 주소/역참조 개념을 구분할 수 있다.
- 배열과 포인터 관계를 코드로 설명할 수 있다.
- NULL, 댕글링 포인터 같은 위험 상황을 예방할 수 있다.

## 핵심
- 주소 연산자: `&`
- 역참조 연산자: `*`
- 안전 초기화: `NULL` 사용

## 1) 주소와 포인터
- `&변수`: 변수의 주소
- `*포인터`: 그 주소가 가리키는 실제 값(역참조)

```c
int n = 10;
int* p = &n;
printf("주소: %p\n", (void*)p);
printf("값: %d\n", *p);
```

## 2) 포인터를 통한 값 변경
```c
int n = 10;
int* p = &n;
*p = 30;
printf("%d\n", n); // 30
```

## 3) 포인터와 배열 관계
배열 이름은 첫 요소의 주소처럼 동작합니다.

```c
int arr[3] = {10, 20, 30};
int* p = arr;
printf("%d\n", *(p + 1)); // 20
printf("%d\n", p[2]);     // 30
```

## 4) NULL 포인터
아직 유효한 주소를 가리키지 않을 때 `NULL`로 초기화해 안전성을 높입니다.

```c
int* p = NULL;
```

## 5) 자주 하는 실수
- 초기화하지 않은 포인터 역참조
- 해제된 메모리를 계속 사용하는 댕글링 포인터
- 잘못된 포인터 연산으로 범위를 벗어나는 접근

## 단계별 실습
1. 정수 변수 하나를 만들고 주소를 출력한다.
2. 포인터로 값을 읽고 수정해 원본 값 변경을 확인한다.
3. 배열을 포인터로 순회하며 합계를 계산한다.
4. `NULL` 검사 후 역참조하는 안전 코드를 작성한다.
