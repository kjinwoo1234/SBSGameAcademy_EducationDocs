# Chapter 01 개론

## 학습 목표
- C언어의 특징과 학습 방향을 이해한다.
- 컴파일 기반 언어의 학습 흐름을 파악한다.

## 세부 주제
- C언어의 탄생 배경과 활용 분야

- 개발 환경에서 첫 C 소스 파일을 생성해본다.
- 컴파일과 실행의 차이를 말로 설명할 수 있다.

## 본문

### c언어의 특징

C언어는 “컴퓨터에게 일을 시키는 방법”을 가장 기초부터 배우기 좋은 언어입니다. 다른 언어보다 메모리·주소·데이터가 저장되는 방식처럼 컴퓨터 내부 동작을 직접 다루기 때문에 처음에는 부담이 될 수 있지만, 한 번 몸에 익으면 이후에 실력이 단단해지는 편입니다.

집을 지을 때 기초 공사가 중요하듯, C언어는 나중에 C++, C#, Java 등을 배 때도 받쳐 주는 **기초 체력**에 가깝습니다. 그래서 “당장 화려한 결과”보다는 **동작 원리를 정확히** 잡는 쪽에 초점을 두고 배우면 좋습니다.

### 컴파일이란

컴파일은 사람이 읽기 쉬운 소스 코드(`.c` 파일)를 컴퓨터가 실행할 수 있는 형태(Windows에서는 보통 `.exe`)로 바꾸는 과정입니다. 쉽게 말하면 **사람의 언어를 컴퓨터가 이해하는 형태로 번역**하는 단계라고 보면 됩니다.

일반적인 흐름은 `코드 작성 → 컴파일(번역) → 실행` 순서입니다. C언어는 실행 전에 반드시 컴파일이 필요하고, 문법이 틀리면 오류 메시지가 나오며 실행 파일이 만들어지지 않을 수 있습니다.

### c언어의 탄생 배경과 활용 분야

C언어는 1970년대 초 벨 연구소에서 시스템 소프트웨어를 만들기 위해 설계되었습니다. 이후 운영체제(예: UNIX), 임베디드, 드라이버, 게임 엔진의 코어 모듈처럼 **성능과 제어**가 중요한 분야에서 널리 쓰입니다.

학습 관점에서는 “컴퓨터가 실제로 어떻게 동작하는지”를 이해하는 출발점으로도 좋습니다. 게임 개발을 목표로 한다면 성능과 메모리 개념을 잡는 데 특히 도움이 됩니다.

### Visual Studio란?

Visual Studio는 Microsoft에서 제공하는 프로그래밍용 **IDE(Integrated Development Environment, 통합 개발 환경)**입니다. 코드 작성, 빌드(컴파일), 실행, 오류 확인, 디버깅(문제 추적)을 한 화면에서 이어 가기 쉽게 묶어 두었기 때문에 초보자에게도 선택지로 자주 쓰입니다.

비유하자면 “프로그래밍 작업실”에 가깝고, 필요한 도구가 한곳에 모여 있다고 생각하면 됩니다. 이 과정의 C 학습에서는 **코드를 쓰는 자리**이자, 동시에 **MSVC 컴파일러를 설치하고 쓰는 중심 도구** 역할을 합니다.

### `cl`이란?

`cl`은 Visual Studio에 포함된 C/C++ 컴파일러 **MSVC(Microsoft Visual C++)**를 명령줄에서 실행할 때 쓰는 프로그램 이름입니다. 우리가 만든 `main.c` 같은 소스 파일을 읽어 실행 가능한 프로그램으로 만들어 줍니다.

컴파일이 끝나면 같은 폴더에 `main.obj`처럼 기계어에 가까운 중간 결과물과, 실제로 더블클릭해 실행하는 `main.exe` 같은 파일이 생기는 경우가 많습니다. 같은 C라도 컴파일러(`cl`, `gcc` 등)에 따라 옵션과 오류 메시지 표현이 조금씩 달라질 수 있으므로, 이 문서는 **Windows + Visual Studio + `cl`** 기준으로 맞춰 두었습니다.

### IDE를 쓰면 `cl`을 직접 몰라도 되나요?

대부분의 경우 Visual Studio는 빌드 버튼을 누를 때 내부에서 `cl` 같은 도구를 대신 호출해 줍니다. 그래서 매번 명령어를 직접 치지 않아도 학습은 진행할 수 있습니다.

다만 오류가 났을 때 “환경 문제인지, 코드 문제인지”를 가르려면, **컴파일러가 소스를 실행 파일로 바꾼다**는 그림은 알아 두는 편이 좋습니다. 정리하면 **일상은 IDE에 맡기고**, **원리는 `cl`이 무슨 역할인지 가볍게** 알아 두면 충분합니다.

### 개발환경 세팅하기

이 문서는 **Windows + Visual Studio + MSVC(`cl`)** 조합을 기준으로 합니다.

#### STEP 1) Visual Studio Installer 준비

Visual Studio 공식 사이트에서 **Visual Studio Community**를 내려받아 설치합니다. 설치가 끝나면 보통 **Visual Studio Installer**가 함께 실행됩니다. 이미 설치되어 있다면 시작 메뉴에서 Installer만 따로 열어도 됩니다.

#### STEP 2) C/C++ 개발 워크로드 선택

Installer에서 설치할 제품을 고른 뒤 **수정(Modify)**을 누릅니다. **Desktop development with C++**(C++를 사용한 데스크톱 개발) 워크로드를 체크합니다. 오른쪽 설치 요약에 `MSVC`, `Windows SDK`, `C++ CMake tools` 등이 포함돼 있는지 확인한 다음 **수정/설치**로 마칩니다. 이 단계를 빼먹으면 이후에 `cl` 명령이 인식되지 않는 경우가 많으니 가장 중요한 단계라고 보면 됩니다.

#### STEP 3) Developer PowerShell 실행

시작 메뉴에서 **Developer PowerShell for VS 2022**를 실행합니다. 일반 PowerShell이 아니라 Visual Studio가 준비한 **개발자용 셸**을 써야 `cl` 경로가 자동으로 잡히는 경우가 대부분입니다. 개발자 셸은 컴파일러 위치와 빌드에 필요한 설정을 미리 불러 옵니다.

#### STEP 4) 컴파일러 설치 확인

개발자 PowerShell에서 `cl`만 입력해 봅니다. 버전 안내와 사용법 도움말처럼 긴 텍스트가 **오류 없이** 출력되면 설치가 정상적으로 잡힌 것입니다.

#### STEP 5) 인식이 안 될 때 점검

`cl`을 찾을 수 없다는 메시지가 나오면, 먼저 **Desktop development with C++** 워크로드가 실제로 설치됐는지 확인합니다. 그다음 **Developer PowerShell for VS 2022**에서 실행했는지, 설치 직후라면 터미널을 닫았다가 다시 열었는지, 그래도 안 되면 PC를 한 번 재부팅한 뒤 다시 시도해 봅니다.

### 첫 c 소스 파일 생성하기

프로젝트용 폴더를 하나 만들고 그 안에 `main.c` 파일을 둡니다. 아래는 그 파일에 넣을 수 있는 최소 예시입니다.

`#include <stdio.h>`는 화면 출력에 필요한 선언들을 가져오는 줄입니다. `#`으로 시작하는 줄은 전처리기에 해당하는데, 세부 동작은 뒤 챕터에서 다시 정리합니다. `main` 함수는 프로그램이 시작되는 **입구**라고 생각하면 됩니다.

```c
#include <stdio.h>

int main(void) {
    printf("Hello, C!\n");
    return 0;
}
```

### cli로 컴파일하고 실행하기

**Developer PowerShell for VS 2022**를 연 뒤, `main.c`가 있는 폴더로 이동합니다. 흔한 흐름은 **폴더 이동 → `cl`로 컴파일 → `.\실행파일.exe`로 실행**입니다. 화면에 `Hello, C!`가 보이면 환경이 정상인 경우가 많습니다. 실행 파일이 생기지 않았다면 컴파일 단계에서 출력된 오류 메시지를 먼저 읽어 보세요.

#### `cl` 명령어 사용 예시

표에 나오는 **옵션(option, 선택 스위치)**은 컴파일러에게 “실행 파일 이름을 이렇게 바꿔라”, “경고를 더 자세히 보여줘” 같은 **추가 지시**를 줄 때 씁니다. 옵션 없이 `cl main.c`만으로도 기본 동작은 됩니다.

| 목적 | 예시 명령 | 생성되는 실행 파일(기본 가정: 소스 이름이 `main.c`) | 비고 |
|---|---|---|---|
| 기본 컴파일 | `cl main.c` | `main.exe` | 같은 폴더에 `main.obj`도 함께 생깁니다. |
| 실행 파일 이름 바꾸기 | `cl /Fegreet.exe main.c` | `greet.exe` | `/Fe`는 MSVC에서 **실행 파일(.exe) 이름과 위치를 지정**할 때 쓰는 옵션입니다. `/Fe` 바로 뒤에 **공백 없이** 파일 이름을 이어 붙이는 형태가 흔합니다. 경로에 공백이 있으면 전체를 큰따옴표로 감싸야 할 수 있습니다. |
| 경고를 더 많이 보기 | `cl /W4 main.c` | `main.exe` | `/W4`는 **W**arning(경고) 수준 **4**로 올려서, 나중에 버그로 이어질 수 있는 패턴을 더 일찍 보여 주는 설정입니다. |

**전체 흐름 예시**는 아래처럼 이어서 입력하면 됩니다. `cd`는 **c**hange **d**irectory(현재 작업 폴더를 바꾼다)는 뜻입니다. 아래의 `cd` 뒤 경로는 **예시 이름**이므로, 복사할 때는 **본인 PC에 `main.c`를 저장해 둔 실제 경로**로 바꿉니다.

```text
cd D:\예시폴더\내프로젝트
cl main.c
.\main.exe
```

실행 파일 이름을 바꿔 컴파일했다면, 실행할 때도 그 이름을 씁니다.

```text
cl /Fegreet.exe main.c
.\greet.exe
```

실행 직후 화면에는 대략 아래처럼 나오면 됩니다. 컴파일할 때는 컴파일러 버전 안내 등 긴 메시지가 먼저 나올 수 있는데, PC나 설정에 따라 문구는 조금 달라질 수 있습니다.

```text
Hello, C!
```

**명령과 절차를 한 번에 읽기**하면 다음과 같습니다. `cd`로 작업 폴더를 맞춰 두어야 `cl`이 같은 위치의 `main.c`를 찾습니다. 폴더가 어긋나면 “파일을 찾을 수 없습니다” 같은 메시지가 나올 수 있습니다. `cl main.c`가 컴파일을 수행해 성공 시 `main.exe`를 만들고, `.\main.exe`의 `.\`는 PowerShell에서 “**지금 폴더**에 있는 프로그램을 실행한다”는 뜻입니다.

#### 자주 막히는 지점(발생 조건 → 영향 → 점검)

`cl`을 **일반 PowerShell**에서 실행했거나, **Desktop development with C++** 워크로드가 빠진 상태이면 `cl`이 인식되지 않아 첫 컴파일부터 막힐 수 있습니다. Visual Studio가 제공하는 **개발자용 셸**과 **C++ 빌드 도구**를 함께 갖춰야 `cl` 경로가 잡히는 경우가 많으므로, 위 **STEP 3~5**를 다시 확인하면 됩니다.

### 비주얼 스튜디오에서 바로 컴파일하고 실행하기

상단 메뉴의 **빌드 → 솔루션 빌드** 또는 `Ctrl + Shift + B`, 혹은 **프로젝트 빌드** / `Ctrl + B`로 컴파일할 수 있습니다. 실행은 `Ctrl + F5`(디버깅 없이 실행) 또는 `F5`(디버깅 실행)를 씁니다. 콘솔에 `Hello, C!`가 보이면 같은 목표는 달성한 것입니다.

### 연습문제

**문제 1**
- 문제: `Hello, C!` 대신 본인 이름과 오늘 날짜가 나오도록 바꿔 출력해 보세요.
- 입력: 해당 없음
- 출력: 이름과 날짜를 포함한 한 줄 이상 메시지
- 조건(힌트): `printf` 문자열 리터럴만 바꿔도 됩니다.

**문제 2**
- 문제: 줄바꿈(`\n`)을 사용해 문장을 3줄 이상 출력해 보세요.
- 입력: 해당 없음
- 출력: 서로 다른 문장 3줄 이상
- 조건(힌트): 한 번의 `printf` 또는 여러 번의 `printf`를 사용해도 됩니다.

**문제 3**
- 문제: 같은 코드를 CLI(`cl`)와 IDE(`Ctrl + F5`) 방식으로 각각 한 번 이상 실행해 보세요.
- 입력: 해당 없음
- 출력: 두 실행 방식 모두 동일한 결과 화면
- 조건(힌트): 실행 결과가 다르면 저장 파일/빌드 대상을 먼저 확인하세요.

### 정답 포인트

`printf`의 문자열만 원하는 내용으로 바꾸면 됩니다. 줄바꿈은 `\n`을 문자열 안에 넣으면 됩니다. 두 방식 모두에서 같은 문구가 출력되면 실행 경로는 정상으로 보면 됩니다.
