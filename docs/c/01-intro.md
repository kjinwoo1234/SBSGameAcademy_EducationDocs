# Chapter 01 개론

## 학습 목표
- C언어의 특징과 학습 방향을 이해한다.
- 컴파일 기반 언어의 학습 흐름을 파악한다.

## 세부 주제
- C언어의 탄생 배경과 활용 분야
- C 학습이 게임 개발 기초에 주는 이점

## 실습 체크리스트
- 개발 환경에서 첫 C 소스 파일을 생성해본다.
- 컴파일과 실행의 차이를 말로 설명할 수 있다.

## 본문

### c언어의 특징
- C언어는 "컴퓨터에게 일을 시키는 방법"을 가장 기초부터 배우기 좋은 언어입니다.
- 다른 언어보다 컴퓨터 내부 동작(메모리, 주소, 데이터 저장 방식)을 직접 다루기 때문에 처음엔 어렵지만, 한 번 이해하면 실력이 단단해집니다.
- 집을 지을 때 기초 공사가 중요하듯, C언어는 이후 C++, C#, Java 같은 언어를 배울 때 큰 도움이 되는 기초 체력 역할을 합니다.

### 컴파일이란
- 컴파일은 사람이 읽기 쉬운 코드(`.c`)를 컴퓨터가 이해할 수 있는 실행 파일(`.exe`)로 바꾸는 과정입니다.
- 쉽게 말하면, "사람의 언어를 컴퓨터 언어로 번역"하는 단계입니다.
- 일반적인 흐름은 `코드 작성 -> 컴파일(번역) -> 실행` 순서입니다.
- C언어는 실행 전에 반드시 컴파일을 해야 하며, 문법이 틀리면 오류가 나고 실행이 되지 않습니다.

### c언어의 탄생 배경과 활용 분야
- C언어는 1970년대 초 벨 연구소에서 시스템 소프트웨어 개발을 위해 만들어졌습니다.
- 운영체제(예: UNIX), 임베디드, 드라이버, 게임 엔진의 코어 모듈 등 성능과 제어가 중요한 분야에서 널리 사용됩니다.
- 학습 관점에서 C언어는 "컴퓨터가 실제로 어떻게 동작하는지"를 이해하는 가장 좋은 출발점 중 하나입니다.
- 특히 게임 개발을 목표로 한다면, 성능과 메모리 개념을 이해하는 데 매우 유리합니다.

### Visual Studio란?
- Visual Studio는 Microsoft에서 제공하는 프로그래밍 작업용 프로그램(IDE)입니다.
- 한 화면에서 코드 작성, 실행, 오류 확인, 디버깅(문제 추적)까지 할 수 있어서 초보자가 사용하기 편합니다.
- 비유하면, Visual Studio는 "프로그래밍 작업실"이고, 그 안에 필요한 도구들이 한 번에 들어 있습니다.
- C언어 학습에서는 코드를 쓰는 장소이자, 컴파일러(MSVC)를 설치하고 사용하는 중심 도구입니다.

### `cl`이란?
- `cl`은 Visual Studio에 들어 있는 C/C++ 컴파일러(MSVC)를 실행하는 명령어 이름입니다.
- 우리가 작성한 `main.c` 파일을 읽어서 실행 가능한 프로그램 파일로 만들어 줍니다.
- 컴파일 후에는 보통 다음 파일이 생성됩니다.
  - `main.obj`: 기계어에 가까운 중간 결과 파일
  - `main.exe`: 실제 실행 파일
- 같은 C언어라도 컴파일러(`cl`, `gcc` 등)에 따라 옵션/오류 메시지가 조금씩 다를 수 있습니다.
- 이 문서는 Visual Studio 환경에 맞춰 `cl` 기준으로 설명합니다.

### IDE를 쓰면 `cl`을 직접 몰라도 되나요?
- 대부분의 경우 Visual Studio 같은 IDE가 빌드 버튼을 누를 때 내부적으로 `cl`을 자동 실행해 줍니다.
- 그래서 초보자는 매번 명령어를 직접 입력하지 않아도 학습을 진행할 수 있습니다.
- 다만 오류가 발생했을 때 원인을 파악하려면 "컴파일러가 코드를 실행 파일로 바꾼다"는 기본 개념은 알고 있는 것이 좋습니다.
- 정리하면, **일상 사용은 IDE 자동 빌드에 맡기고**, **문제 해결과 기초 이해를 위해 `cl`의 역할은 가볍게 알아두면 충분**합니다.

### 개발환경 세팅하기
- 이 문서는 **Windows + Visual Studio + MSVC(cl)** 기준으로 진행합니다.

#### STEP 1) Visual Studio Installer 준비
- [Visual Studio 공식 사이트](https://visualstudio.microsoft.com/ko/)에서 `Visual Studio Community`를 설치합니다.
- 설치가 끝나면 자동으로 `Visual Studio Installer`가 실행됩니다.
- 이미 설치되어 있다면 시작 메뉴에서 `Visual Studio Installer`를 직접 실행해도 됩니다.

#### STEP 2) C/C++ 개발 워크로드 선택
- Installer에서 설치할 버전을 선택한 뒤 **수정(Modify)**을 누릅니다.
- **Desktop development with C++** 워크로드를 체크합니다.
- 오른쪽 설치 요약에서 `MSVC`, `Windows SDK`, `C++ CMake tools`가 포함되어 있는지 확인합니다.
- **수정/설치**를 눌러 완료합니다.
- 이 단계가 가장 중요합니다. 이 항목을 설치하지 않으면 `cl` 명령이 동작하지 않습니다.

#### STEP 3) Developer PowerShell 실행
- 시작 메뉴에서 **Developer PowerShell for VS 2022**를 실행합니다.
- 일반 PowerShell이 아니라 Visual Studio 개발자 셸을 사용해야 `cl` 명령이 바로 인식됩니다.
- 이유: 개발자 셸은 컴파일러 경로(Path)와 필요한 설정을 자동으로 불러옵니다.

#### STEP 4) 컴파일러 설치 확인
- 아래 명령으로 MSVC 컴파일러를 확인합니다.
  - `cl`
- 버전 정보와 사용법 안내가 출력되면 설치가 정상 완료된 것입니다.
- `cl` 실행 시 에러 없이 텍스트가 길게 나오면 정상입니다.

#### STEP 5) 인식이 안 될 때 점검
- `cl`을 찾을 수 없다는 메시지가 나오면 아래를 확인합니다.
  - `Desktop development with C++` 워크로드가 실제 설치되었는지
  - **Developer PowerShell for VS 2022**에서 실행했는지
  - 설치 후 터미널을 새로 열었는지
  - PC 재부팅 후 다시 시도했는지

### 첫 c 소스 파일 생성하기
- 프로젝트 폴더를 만들고 `main.c` 파일을 생성합니다.
- 아래의 최소 코드로 시작합니다.
- `#include <stdio.h>`는 화면 출력에 필요한 파일을 내 코드에 가져오는 줄입니다. `#`으로 시작하는 코드는 전처리기라고도 불리는데, 나중에 조금 더 자세히 알아보도록 합시다.
- `main` 함수는 프로그램이 시작되는 "입구"라고 생각하면 됩니다.

```c
#include <stdio.h>

int main(void) {
    printf("Hello, C!\n");
    return 0;
}
```

### cli로 컴파일하고 실행하기
- **Developer PowerShell for VS 2022**에서 소스 파일이 있는 폴더로 이동합니다.
- 컴파일: `cl main.c`
- 실행: `.\main.exe`
- 실행 결과로 `Hello, C!`가 출력되면 개발 환경이 정상 동작하는 것입니다.
- 만약 실행 파일이 보이지 않으면, 컴파일 단계에서 나온 오류 메시지를 먼저 확인하세요.

### 비주얼 스튜디오에서 바로 컴파일하고 실행하기
- 상단 메뉴에서 **빌드 -> 솔루션 빌드**를 누르거나 `Ctrl + Shift + B`를 눌러 컴파일합니다. 또는 `프로젝트 빌드`를 누르거나 `Ctrl + B` 를 눌러 컴파일합니다.
- 실행은 `Ctrl + F5`(디버깅 없이 실행) 또는 `F5`(디버깅 실행)를 사용합니다.
- 콘솔 창에 `Hello, C!`가 출력되면 성공입니다.

연습문제:
1. `Hello, C!` 대신 본인 이름과 오늘 날짜를 출력해보세요.
2. 줄바꿈(`\n`)을 사용해 3줄 문장을 출력해보세요.
3. CLI(`cl`)와 IDE(`Ctrl + F5`) 방식으로 각각 1회 이상 실행해보세요.

정답 포인트:
- `printf`를 사용해 출력 문자열만 변경하면 됩니다.
- 줄바꿈은 `\n`으로 처리합니다.
- 실행 결과가 동일하면 두 실행 방식 모두 성공입니다.
