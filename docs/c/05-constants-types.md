## 본문

### 05-1 기본 자료형

자료형은 “어떤 종류의 데이터를 저장할지”를 정하는 규칙입니다. `char`는 문자 한 개, `int`는 정수, `float`와 `double`은 실수를 다루는 데 쓰입니다. 타입마다 저장 가능한 범위와 정밀도가 다르므로, 목적에 맞게 고르는 것이 중요합니다. 크기는 컴파일러와 플랫폼에 따라 달라질 수 있으므로, 실제 프로젝트에서는 `sizeof`로 확인하는 습관이 좋습니다.

입문에서 우선 익히면 좋은 네 가지는 `char`, `int`, `float`, `double`입니다. 그 밖에 `short`는 작은 범위 정수, `long`/`long long`은 큰 범위 정수, `unsigned int`는 음수를 제외하고 양수 범위를 넓히고 싶을 때, `signed`/`unsigned char`는 바이트 데이터를 다룰 때 자주 만납니다. `long double`은 더 높은 실수 정밀도를 노릴 때 쓰지만 환경 의존이 큽니다. `_Bool`(`stdbool.h`를 포함하면 `bool`)은 참/거짓을 저장할 때 쓰고, `void`는 “값이 없음”을 표현하거나 범용 포인터 `void*`와 함께 등장합니다.

확장 타입을 표로 한 번에 보면 아래와 같습니다.

| 타입 | 용도 | 비고 |
|---|---|---|
| `short` | 작은 범위 정수 | 메모리를 조금 아끼고 싶을 때 |
| `long`, `long long` | 큰 범위 정수 | 큰 숫자 계산/카운트 |
| `unsigned int` | 음수 없는 정수 | 범위를 양수 쪽으로 확장 |
| `signed/unsigned char` | 문자/바이트 데이터 | 바이너리 처리에서 자주 사용 |
| `long double` | 더 높은 실수 정밀도 | 환경 의존성 큼 |
| `bool` | 참/거짓 저장 |  |
| `void` | 값 없음 | 반환값 없음, 범용 포인터(`void*`) |

크기 확인 예시는 다음과 같습니다(환경에 따라 출력은 달라질 수 있습니다).

```c
#include <stdio.h>

int main(void) {
    printf("char: %zu byte\n", sizeof(char));
    printf("int: %zu byte\n", sizeof(int));
    printf("float: %zu byte\n", sizeof(float));
    printf("double: %zu byte\n", sizeof(double));
    return 0;
}
```

### 05-2 문자 표현 방식

컴퓨터는 문자를 숫자 코드로 저장합니다. 대표적인 예가 **ASCII**입니다. `'A'` 같은 문자 리터럴도 내부적으로는 정수 값으로 처리되므로, 문자와 숫자의 관계를 알아 두면 입력·출력과 문자열 처리에서 실수가 줄어듭니다.

### 05-3 상수

상수는 실행 중에 값이 바뀌지 않는 데이터입니다. `100`, `3.14`, `'K'`, `"Hello"`는 각각 정수·실수·문자·문자열 상수의 예입니다. 의미 있는 고정값은 상수로 묶어 두면 읽기와 수정이 쉬워집니다. `const`로 선언한 값은 다시 대입할 수 없으므로, 바뀌면 안 되는 설정값에 잘 맞습니다.

### 05-4 자료형 변환

자료형 변환은 한 타입의 값을 다른 타입으로 바꾸는 과정입니다. 컴파일러가 알아서 맞추는 **자동 형 변환**이 있고, 개발자가 `(double)a`처럼 의도를 드러내는 **명시적 형 변환(캐스팅)**이 있습니다. 정수 나눗셈처럼 타입에 따라 결과가 달라질 수 있으므로, 계산 전에 어떤 타입으로 계산될지 확인하는 습관이 필요합니다. 실수 비교에서는 부동소수점 오차 때문에 `==`를 함부로 쓰지 않는 것도 같은 맥락입니다.

예시 코드는 다음과 같습니다.

```c
#include <stdio.h>

int main(void) {
    int a = 7, b = 2;
    double result1 = a / b;           // 정수 나눗셈 후 double 저장
    double result2 = (double)a / b;   // 실수 나눗셈

    const int MAX_LIFE = 3;
    char grade = 'A';

    printf("result1: %.2f\n", result1); // 3.00
    printf("result2: %.2f\n", result2); // 3.50
    printf("grade ASCII: %d\n", grade); // 65
    printf("MAX_LIFE: %d\n", MAX_LIFE);
    return 0;
}
```

### 연습문제

**문제 1**
- 문제: `int x = 5, y = 2;`일 때 `x / y`와 `(double)x / y`의 차이를 출력해 보세요.
- 입력: `x = 5`, `y = 2`
- 출력: 정수 나눗셈 결과와 실수 나눗셈 결과
- 조건(힌트): 둘 다 `double` 변수에 담아 출력하면 차이를 확인하기 쉽습니다.

**문제 2**
- 문제: 문자 `'Z'`의 ASCII 코드를 출력해 보세요.
- 입력: 문자 `'Z'`
- 출력: `Z`의 정수 코드값
- 조건(힌트): `%d` 형식 지정자로 출력하세요.

**문제 3**
- 문제: `const`를 사용해 `PI = 3.14`를 선언해 보세요.
- 입력: 해당 없음
- 출력: `PI`를 사용한 계산 결과 1개 이상
- 조건(힌트): `const double PI = 3.14;` 형태로 선언하세요.

### 정답 예시

```c
double a = x / y;
double b = (double)x / y;
printf("%.2f %.2f\n", a, b);
printf("%d\n", 'Z');
const double PI = 3.14;
```

자주 하는 실수로는 `const` 변수에 다시 값을 넣으려는 시도, `5 / 2`를 실수 나눗셈으로 기대하는 경우, `printf`의 `%d`/`%f`를 변수 타입과 맞추지 않는 경우 등을 들 수 있습니다.
