# Chapter 28 콘솔 제어로 뱀/테트리스 만들기

## 학습 목표
- 콘솔 커서 이동, 색상 변경, 화면 지우기 함수를 이해한다.
- 실시간 입력 루프를 바탕으로 뱀게임/테트리스 구조를 설계할 수 있다.

## 세부 주제
### 28-1 콘솔 제어 함수
- 커서 이동, 텍스트 색상, 화면 클리어

### 28-2 뱀게임 핵심 루프
- 입력 -> 이동 -> 충돌 검사 -> 렌더링

### 28-3 테트리스 핵심 루프
- 블록 낙하 -> 충돌/고정 -> 줄 삭제 -> 렌더링

## 실습 체크리스트
- 콘솔 제어 함수를 분리해 재사용 가능한 `console_util` 모듈을 만든다.

## 본문

### 28-1 콘솔 제어 함수

콘솔 게임에서는 화면 전체를 매번 지우기보다, **커서를 원하는 좌표로 옮겨** 필요한 부분만 다시 찍는 방식이 부담이 적은 경우가 많습니다. Windows 콘솔이라면 커서·색·지우기를 작은 함수로 감싸 두고 뱀과 테트리스에서 **공통 모듈**로 쓰기 좋습니다.

아래 예시는 **Windows 콘솔 API**를 가정합니다. 다른 환경에서는 ANSI 이스케이프 등으로 바꿔야 할 수 있습니다.

| 기능 | 권장 함수(Windows) | 용도 |
|---|---|---|
| 커서 이동 | `SetConsoleCursorPosition` | 원하는 좌표에 출력 위치 이동 |
| 텍스트 색상 | `SetConsoleTextAttribute` | 점수/경고/블록 색 구분 |
| 화면 지우기 | `system("cls")` 또는 버퍼 재렌더링 | 화면 초기화 |

**게임 루프(game loop)**는 입력 처리, 상태 갱신, 화면 출력을 한 바퀴 도는 구조를 말합니다. **틱(tick)**은 그 루프가 한 번 돌아가는 단위 시간으로 이해하면 됩니다.

```c
#include <windows.h>
#include <stdlib.h>

void gotoxy(int x, int y) {
    COORD p = {(SHORT)x, (SHORT)y};
    SetConsoleCursorPosition(GetStdHandle(STD_OUTPUT_HANDLE), p);
}

void set_color(WORD color) {
    SetConsoleTextAttribute(GetStdHandle(STD_OUTPUT_HANDLE), color);
}

void clear_screen(void) {
    system("cls");
}
```

`gotoxy`로 좌표를 직접 잡는 것이 UI의 중심이고, `set_color`로 상태를 색으로 나누면 한눈에 들어옵니다. `clear_screen`은 구현이 간단하지만 호출이 잦으면 깜빡임이 생길 수 있어, 부분 갱신과 함께 설계하는 편이 좋습니다.

### 28-2 뱀게임 핵심 루프

방향 입력을 받고 머리 좌표를 갱신한 다음, 벽과 자기 몸과의 **충돌**를 검사합니다. 먹이를 먹으면 길이와 점수를 바꾸고, `gotoxy`로 필요한 칸만 다시 그리면 효율적입니다.

### 28-3 테트리스 핵심 루프

일정 틱마다 블록을 한 칸 아래로 내리고, 바닥이나 쌓인 블록과 **맞닿으면** 보드에 고정합니다. 한 줄이 가득 차면 지우고, 블록 모양과 회전은 배열로 표현하면 충돌 검사가 단순해집니다.

### 정답 포인트

콘솔 제어는 게임 규칙과 분리해 모듈로 두고, 뱀·테트리스 모두 **입력 → 업데이트 → 렌더링** 루프를 유지합니다. 출력 범위를 최소화하면 깜빡임과 입력 지연이 줄어듭니다.
