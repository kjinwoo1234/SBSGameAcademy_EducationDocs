# Chapter 28 콘솔 제어로 뱀/테트리스 만들기

## 학습 목표
- 콘솔 커서 이동, 색상 변경, 화면 지우기 함수를 이해한다.
- 실시간 입력 루프를 바탕으로 뱀게임/테트리스 구조를 설계할 수 있다.

## 세부 주제
### 28-1 콘솔 제어 함수
- 커서 이동, 텍스트 색상, 화면 클리어

### 28-2 뱀게임 핵심 루프
- 입력 -> 이동 -> 충돌 검사 -> 렌더링

### 28-3 테트리스 핵심 루프
- 블록 낙하 -> 충돌/고정 -> 줄 삭제 -> 렌더링

## 실습 체크리스트
- 콘솔 제어 함수를 분리해 재사용 가능한 `console_util` 모듈을 만든다.

## 본문

### 28-1 콘솔 제어 함수
- 콘솔 게임은 "화면 전체를 다시 출력"하기보다 커서를 원하는 위치로 이동해 필요한 부분만 갱신하는 방식이 효율적입니다.
- Windows 콘솔 기준으로 커서 이동/색상/지우기 함수를 직접 래핑해두면 뱀게임과 테트리스에 공통으로 재사용할 수 있습니다.

환경 주의사항:
- 아래 예시는 Windows 콘솔 API 기준입니다.
- 다른 환경에서는 ANSI 이스케이프 코드 방식으로 대체가 필요할 수 있습니다.

함수 비교:
| 기능 | 권장 함수(Windows) | 용도 |
|---|---|---|
| 커서 이동 | `SetConsoleCursorPosition` | 원하는 좌표에 출력 위치 이동 |
| 텍스트 색상 | `SetConsoleTextAttribute` | 점수/경고/블록 색 구분 |
| 화면 지우기 | `system("cls")` 또는 버퍼 재렌더링 | 화면 초기화 |

용어 설명:
| 용어 | 의미 |
|---|---|
| 게임 루프(game loop) | 입력 처리 -> 상태 업데이트 -> 화면 출력을 반복하는 구조 |
| 틱(tick) | 루프 한 번 실행되는 단위 시간 |

예시 코드(콘솔 유틸):
```c
#include <windows.h>
#include <stdlib.h>

void gotoxy(int x, int y) {
    COORD p = {(SHORT)x, (SHORT)y};
    SetConsoleCursorPosition(GetStdHandle(STD_OUTPUT_HANDLE), p);
}

void set_color(WORD color) {
    SetConsoleTextAttribute(GetStdHandle(STD_OUTPUT_HANDLE), color);
}

void clear_screen(void) {
    system("cls");
}
```

예상 결과:
```text
좌표 이동 출력, 텍스트 색 변경, 화면 초기화 동작 확인
```

코드 해석:
- `gotoxy`는 출력 좌표를 직접 제어해 콘솔 UI를 구성할 때 핵심입니다.
- `set_color`로 게임 상태를 색으로 구분하면 가독성이 크게 좋아집니다.
- `clear_screen`은 간단하지만 빈번 호출 시 깜빡임이 생길 수 있어 부분 갱신 전략과 함께 고려합니다.

### 28-2 뱀게임 핵심 루프
- 방향 입력을 받고 머리 좌표를 갱신한 뒤, 벽/몸통 충돌 여부를 검사합니다.
- 먹이를 먹으면 몸 길이를 늘리고 점수를 갱신합니다.
- 렌더링은 `gotoxy` 기반 좌표 출력으로 구현하면 효율적입니다.

### 28-3 테트리스 핵심 루프
- 일정 틱마다 블록을 한 칸씩 아래로 이동합니다.
- 충돌 시 블록을 보드에 고정하고, 완성된 줄을 삭제합니다.
- 블록 모양/회전 상태를 배열로 관리하면 충돌 검사 로직이 단순해집니다.

연습문제:
1. 문제 1 - 콘솔 유틸 모듈 만들기
   - 문제: `gotoxy`, `set_color`, `clear_screen`를 `console_util.c/.h`로 분리하세요.
   - 입력: 없음
   - 출력: 위치 이동/색상/화면 지우기 데모
   - 조건: 함수 선언/정의 분리
2. 문제 2 - 뱀게임 최소 버전
   - 문제: 머리 1칸과 먹이 1개가 있는 최소 뱀게임 루프를 구현하세요.
   - 입력: 방향키 또는 WASD
   - 출력: 뱀 이동/충돌 시 종료
   - 힌트: 틱 주기 `Sleep` 사용
3. 문제 3 - 테트리스 낙하 루프
   - 문제: 1개 블록이 아래로 떨어지고 바닥 충돌 시 고정되는 로직을 구현하세요.
   - 입력: 좌우 이동 키
   - 출력: 블록 낙하/고정
   - 힌트: 보드 배열(`int board[h][w]`) 사용

정답 포인트:
- 콘솔 제어 함수는 게임 로직과 분리해 재사용 모듈로 관리
- 뱀/테트리스 모두 "입력 -> 업데이트 -> 렌더링" 루프 구조가 핵심
- 깜빡임/입력 지연 문제를 줄이기 위해 출력 범위를 최소화
