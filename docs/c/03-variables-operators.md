## 본문

### 03-1 변수란?
- 변수는 값을 저장하는 "이름이 붙은 상자"라고 생각하면 쉽습니다.
- 값을 처음 넣는 것을 초기화라고 하며, 초기화 습관은 오류를 줄이는 데 매우 중요합니다.
- 변수 이름은 의미 있게 작성해야 나중에 코드를 다시 볼 때 이해가 쉽습니다.

#### 변수 타입(자료형) 기본
- `int`: 정수 저장 (예: `-3`, `0`, `120`)
- `float`: 소수점이 있는 실수 저장 (예: `3.14f`)
- `double`: `float`보다 더 정밀한 실수 저장
- `char`: 문자 1개 저장 (예: `'A'`)
- `char[]`: 문자열 저장 (예: `"Hello"`), 문자의 '배열'이며, 배열에 대해서는 나중에 자세히 학습 예정
- 타입이 다르면 저장 방식과 계산 결과가 달라질 수 있으니, "무엇을 저장할지" 먼저 정하고 선언하는 습관이 중요합니다.

#### 변수 선언/초기화 방법
- 선언만 하기: `int score;`
- 선언과 동시에 초기화: `int score = 100;`
- 같은 타입 여러 개 선언: `int x = 1, y = 2, z = 3;`
- 상수처럼 쓸 값은 `const` 사용: `const int MAX_HP = 100;`
- 초기화하지 않은 지역변수는 쓰레기값이 들어갈 수 있으므로 바로 사용하지 말고 반드시 값부터 넣습니다.

#### 변수 이름 짓기 규칙(필수)
- 예약어(키워드) 사용 금지
  - 예: `int`, `for`, `while`, `return` 같은 단어는 변수명으로 사용할 수 없습니다.
- 사용 가능한 문자
  - 영문 대소문자(`a-z`, `A-Z`), 숫자(`0-9`), 밑줄(`_`)
- 시작 문자 규칙
  - 숫자로 시작할 수 없습니다. (`1score` 불가)
  - 영문자 또는 밑줄로 시작해야 합니다.
- 대소문자 구분
  - `score`, `Score`, `SCORE`는 서로 다른 이름입니다.
- 공백/특수문자 불가
  - 공백(`my score`), 하이픈(`my-score`), 점(`my.score`) 등은 변수명에 사용할 수 없습니다.
- 길이 제한
  - 표준 C에서 구현마다 차이가 있지만, 너무 긴 이름은 피하고 의미가 분명한 길이(예: 3~30자 정도)로 작성하는 것을 권장합니다.

#### 추가로 알아두면 좋은 네이밍 팁
- 변수명은 "값의 의미"가 드러나게 작성합니다. (`a`, `b`보다 `playerHp`, `totalScore` 권장)
- 불리언(참/거짓 의미)은 `is`, `has`, `can` 접두어를 쓰면 읽기 쉽습니다. (예: `isAlive`)
- 전부 대문자 이름은 보통 매크로/상수 스타일에 사용합니다. (예: `MAX_BUFFER_SIZE`)
- 밑줄로 시작하는 이름(특히 `__name`)은 시스템/컴파일러 내부 규칙과 충돌할 수 있어 피하는 것이 안전합니다.

#### 자주 쓰는 네이밍 스타일
- `snake_case`: 단어를 소문자 + 밑줄(`_`)로 연결합니다.
  - 예: `player_hp`, `total_score`
- `camelCase`: 첫 단어는 소문자, 다음 단어부터 첫 글자를 대문자로 씁니다.
  - 예: `playerHp`, `totalScore`
- `PascalCase`: 모든 단어의 첫 글자를 대문자로 씁니다.
  - 예: `PlayerHp`, `TotalScore`
- C 언어에서는 변수명에 `snake_case` 또는 `camelCase`를 많이 사용하고, `PascalCase`는 타입명(구조체/별칭)에서 자주 사용합니다.
- 가장 중요한 것은 "한 프로젝트에서 한 스타일을 정하고 일관되게 유지"하는 것입니다.

#### 형식 지정자와 타입 연결
- 변수 값을 출력할 때는 형식 지정자를 함께 사용합니다.
  - `%d`: 정수(`int`)
  - `%f`: 실수(`float`, `double`)
  - `%c`: 문자(`char`)
  - `%s`: 문자열(`char[]`)

유효/무효 변수명 예시:
```c
int score = 100;      // 가능
int player_hp = 80;   // 가능
int _temp = 0;        // 가능(가능하지만 남용 비권장)

// int 3level = 3;    // 불가: 숫자로 시작
// int my score = 10; // 불가: 공백 포함
// int for = 1;       // 불가: 예약어 사용
```

```c
#include <stdio.h>

int main(void) {
    int age = 20;
    float height = 173.5f;
    char grade = 'A';
    char name[] = "Kim";

    printf("이름: %s\n", name);
    printf("나이: %d\n", age);
    printf("키: %.1f\n", height);
    printf("등급: %c\n", grade);

    return 0;
}
```

### 03-2 연산자란?
- 연산자는 값을 계산하거나 비교할 때 사용하는 기호입니다.
- 산술 연산자(`+`, `-`, `*`, `/`)는 계산, 비교 연산자(`==`, `!=`, `>`)는 조건 판단에 사용합니다.
- 논리 연산자(`&&`, `||`, `!`)는 여러 조건을 조합할 때 사용하며, 조건문/반복문과 함께 자주 등장합니다.

주요 연산자 예시:
```c
#include <stdio.h>

int main(void) {
    int a = 10, b = 3;

    // 산술 연산자
    printf("a + b = %d\n", a + b); // 13
    printf("a - b = %d\n", a - b); // 7
    printf("a * b = %d\n", a * b); // 30
    printf("a / b = %d\n", a / b); // 3 (정수 나눗셈)
    printf("a %% b = %d\n", a % b); // 1 (나머지)

    // 비교 연산자 (참:1, 거짓:0)
    printf("a > b = %d\n", a > b);   // 1
    printf("a == b = %d\n", a == b); // 0

    // 논리 연산자
    printf("(a > 5) && (b < 5) = %d\n", (a > 5) && (b < 5)); // 1
    printf("(a < 5) || (b < 5) = %d\n", (a < 5) || (b < 5)); // 1
    printf("!(a == b) = %d\n", !(a == b)); // 1

    return 0;
}
```

연산자 연습문제:
1. `int x = 8, y = 5;`일 때 아래 결과를 구하세요.
   - `x + y`
   - `x / y`
   - `x % y`
2. `int score = 75;`일 때 `score >= 60`의 결과(참/거짓)는?
3. `int age = 17;`일 때 `(age >= 20) || (age == 17)`의 결과는?

연산자 연습문제 정답:
1. `13`, `1`, `3`
2. 거짓(`0`)
3. 참(`1`)

### 03-3 `getch()` / `scanf()` 키보드 입력
- `getch()`는 키보드 입력을 1글자 단위로 받거나, 화면이 바로 닫히지 않게 잠시 대기할 때 사용합니다.
- 입문 단계에서는 "프로그램이 끝나기 전에 결과를 확인"하는 용도로 많이 사용합니다.
- 환경에 따라 동작 차이가 있을 수 있으므로, 표준 입력 함수와 함께 비교해보는 것이 좋습니다.
- **Windows/MSVC 전용 안내:** 아래 `getch()`/`conio.h` 예시는 `Windows + Visual Studio(MSVC)` 기준입니다. 다른 컴파일러에서는 `getchar()` 또는 `scanf()` 예제로 대체하세요.
- `scanf()`는 정수/실수/문자열처럼 형식이 있는 데이터를 입력받아 변수에 저장할 때 사용합니다.
- `scanf()`는 보통 변수의 주소를 넘겨야 하므로 `&변수명` 형태를 자주 사용합니다. (`char[]` 문자열은 예외)
  - 쉽게 말해, `scanf`가 "값을 넣어야 할 변수의 실제 위치"를 알아야 값을 기록할 수 있기 때문입니다.
  - 변수명 앞의 `&`는 그 변수의 메모리 위치(주소)를 전달하는 기호입니다.
  - 예: `scanf("%d", &age);` (`age`의 값이 저장된 위치에 입력값을 써 넣음)
  - 문자열 배열은 배열 이름 자체가 시작 주소처럼 동작하므로 `scanf("%s", name);`처럼 `&` 없이 사용합니다.
  - 포인터/주소 개념의 자세한 원리는 `Chapter 12 포인터의 이해`에서 본격적으로 학습합니다.

`getch()` 기본 예시:
```c
#include <stdio.h>
#include <conio.h> // getch() 함수를 사용하기 위해 가져와야하는 conio.h 파일을 현재 코드에 추가

int main(void) {
    char key;

    printf("아무 키나 누르세요: ");
    key = getch(); // 키 1개 입력 받기

    printf("\n입력한 키: %c\n", key);
    return 0;
}
```

화면 닫힘 방지 예시:
```c
#include <stdio.h>
#include <conio.h>

int main(void) {
    printf("작업 완료!\n");
    printf("종료하려면 아무 키나 누르세요...");
    getch(); // 키를 누를 때까지 대기
    return 0;
}
```

`getch()` 연습문제:
1. 키 하나를 입력받아, 그 문자를 그대로 다시 출력해보세요.
2. 키 하나를 입력받아, ASCII 코드(정수값)도 함께 출력해보세요.

`getch()` 연습문제 정답 예시:
```c
#include <stdio.h>
#include <conio.h>

int main(void) {
    char ch;

    printf("문자 1개 입력: ");
    ch = getch();

    printf("\n입력 문자: %c\n", ch);
    printf("ASCII 코드: %d\n", ch);

    return 0;
}
```

`scanf()` 기본 예시:
```c
#include <stdio.h>

int main(void) {
    int age;
    float height;
    char name[20];

    printf("이름 나이 키를 입력하세요 (예: Kim 20 173.5): ");
    scanf("%19s %d %f", name, &age, &height);

    printf("입력 결과 -> 이름: %s, 나이: %d, 키: %.1f\n", name, age, height);
    return 0;
}
```

`scanf()`에서 자주 하는 실수:
- `%d`인데 `float` 변수에 받거나, `%f`인데 `int` 변수에 받는 등 형식 지정자를 잘못 맞추는 경우
- 숫자 변수 입력에서 `&`를 빼먹는 경우 (예: `scanf("%d", age);`는 잘못된 코드)
- 문자열 입력 버퍼 크기를 고려하지 않는 경우 (`%19s`처럼 길이 제한 권장)

`scanf()` 연습문제:
1. 정수 2개를 입력받아 합을 출력해보세요.
2. 이름(`char[]`)과 점수(`int`)를 입력받아 `"이름: OOO, 점수: 100"` 형태로 출력해보세요.

`scanf()` 연습문제 정답 예시:
```c
#include <stdio.h>

int main(void) {
    int a, b;
    char name[20];
    int score;

    printf("정수 2개 입력: ");
    scanf("%d %d", &a, &b);
    printf("합계: %d\n", a + b);

    printf("이름 점수 입력: ");
    scanf("%19s %d", name, &score);
    printf("이름: %s, 점수: %d\n", name, score);

    return 0;
}
```

`getch()`와 `scanf()` 한눈에 비교:
- `getch()`: 키 1개를 즉시 입력받을 때 사용 (대기/단일 키 처리)
- `scanf()`: 정수/실수/문자열 등 형식 있는 데이터를 입력받을 때 사용
