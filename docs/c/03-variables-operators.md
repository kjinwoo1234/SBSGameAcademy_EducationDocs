## 본문

### 03-1 변수란?

변수는 값을 저장하는 **이름이 붙은 상자**라고 생각하면 이해하기 쉽습니다. 값을 처음 넣는 것을 **초기화**라고 하며, 초기화 습관은 이후에 나오는 “쓰레기값” 문제를 줄이는 데 큰 도움이 됩니다. 변수 이름은 나중에 코드를 다시 읽을 때를 대비해 **의미가 드러나게** 짓는 것이 좋습니다.

#### 변수 타입(자료형) 기본

C 언어는 **자료형(type, 타입)**으로 “이 변수에 어떤 종류의 값이 들어가는지”를 미리 정합니다. 아래는 표준 C에서 정의되는 **대표적인 기본 자료형**을 입문 관점에서만 묶은 표입니다. **용량(바이트)** 열은 변수 하나가 메모리에서 차지하는 공간을 **바이트(byte)** 단위로 본 **예**이며, **일반적인 Windows + MSVC 64비트** 조합에서 자주 보이는 값입니다. 표준은 `int`·`long` 등 **정수형 하나가 몇 바이트인지**까지는 고정하지 않아, **구현(컴파일러·대상 CPU·데이터 모델)마다** 달라질 수 있습니다. **`long`만 그런 것이 아니라 `int`도 마찬가지**이고, 다만 요즘 PC용 32·64비트 환경에서는 `int`가 4 bytes인 경우가 매우 흔합니다. Linux·다른 컴파일러·32비트 빌드에서는 표에 적은 숫자와 다를 수 있으므로, 확실히 알고 싶을 때는 뒤에서 배우는 `sizeof`로 직접 확인하는 습관이 좋습니다.

| 자료형 | 역할(한 줄) | 용량(byte, MSVC x64 예) | 비고 |
|---|---|---|---|
| `char` | 문자 1개 또는 아주 작은 정수(바이트 단위) | 1 byte | `'A'`; 문자열 `char[]`는 배열 챕터에서 확장 |
| `short` | 범위가 작은 정수 | 2 bytes | `int`보다 메모리를 덜 쓰고 싶을 때 |
| `int` | 가장 흔히 쓰는 **정수(integer)** | 4 bytes (구현마다 다를 수 있으나, 이 표는 MSVC x64에서 흔한 예) | 예: `-3`, `0`, `120` |
| `long` | `int`보다 짧을 수 없는 정수(실제 용량·`int`와의 관계는 컴파일러·OS마다 다름) | 4 bytes (Windows MSVC 64비트에서는 `int`와 동일인 경우가 많음) | Linux 64비트 등에서는 8 bytes인 경우도 있음 |
| `long long` | 더 큰 범위의 정수 | 8 bytes | 큰 수·누적 카운트 등 |
| `unsigned char` 등 `unsigned` 정수형 | **부호 없음**(0과 양수만) | 대응하는 `signed`와 **같은 용량**(예: `unsigned int` 4 bytes) | 같은 비트 수로 양수 범위를 넓힘 |
| `float` | 실수(단정밀도, floating) | 4 bytes | 예: `3.14f` |
| `double` | 실수(배정밀도) | 8 bytes | 소수 리터럴은 보통 `double`로 처리 |
| `long double` | 더 높은 정밀도를 목표로 한 실수 | 8 bytes (MSVC에서는 `double`과 같게 잡히는 경우가 많음) | 다른 컴파일러에서는 16 bytes 등으로 더 클 수 있음 |
| `void` | “값이 없음”을 나타냄 | (완전한 타입의 용량 없음) | 예: 반환값 없는 함수, `void*` |
| `bool` | 참/거짓만 저장 | 1 byte | 0이 아니면 참으로 해석되는 경우가 많음 |

`int`는 정수(예: `-3`, `0`, `120`), `float`는 소수가 있는 실수(예: `3.14f`), `double`은 `float`보다 정밀도가 높은 실수, `char`는 문자 한 개(예: `'A'`), `char[]`는 문자열(예: `"Hello"`)을 다룰 때 쓰는 **문자의 배열**입니다. 배열 자체는 뒤 챕터에서 더 깊게 다룹니다. 타입이 다르면 저장 방식과 계산 결과도 달라질 수 있으므로, “무엇을 저장할지”를 먼저 정한 뒤 선언하는 습관이 중요합니다.

#### 변수 선언/초기화 방법

선언만 하려면 `int score;`처럼 쓰고, 선언과 동시에 값을 넣으면 `int score = 100;`처럼 **선언과 초기화**를 한 줄에 묶을 수 있습니다. 같은 타입을 여러 개 둘 때는 `int x = 1, y = 2, z = 3;`처럼 나열할 수 있습니다. 바뀌면 안 되는 값은 `const int MAX_HP = 100;`처럼 `const`로 고정해 두는 패턴도 자주 씁니다. 초기화하지 않은 지역 변수에는 **쓰레기값**이 들어갈 수 있으므로, 값을 넣기 전에 바로 읽거나 출력하지 않도록 주의합니다.

#### 변수 이름 짓기 규칙(필수)

예약어(언어 자체에서 이미 사용중인 키워드)인 `int`, `for`, `while`, `return` 같은 단어는 변수 이름으로 쓸 수 없습니다. 사용할 수 있는 문자는 영문 대소문자, 숫자, 밑줄(`_`)이며, 이름은 **숫자로 시작할 수 없고** 보통 영문자나 밑줄로 시작합니다. `score`, `Score`, `SCORE`는 서로 다른 이름으로 취급됩니다. 공백·하이픈·점 등은 이름 안에 넣을 수 없습니다. 표준 C에서 이름 길이 한도는 구현마다 다르지만, 너무 길게 늘리기보다 **의미가 보이는 적당한 길이**(대략 수십 자 이내)를 권장합니다.

#### 추가로 알아두면 좋은 네이밍 팁

변수명에는 값의 의미가 드러나게 쓰는 편이 좋고, `a`, `b`만 반복하기보다 `playerHp`, `totalScore`처럼 읽히는 이름이 유지보수에 유리합니다. 참/거짓 느낌이면 `isAlive`처럼 `is`, `has`, `can` 접두어를 붙이면 읽기 쉽습니다. 전부 대문자인 `MAX_BUFFER_SIZE` 같은 스타일은 보통 매크로나 상수에 쓰는 경우가 많습니다. 밑줄 두 개로 시작하는 이름은 시스템·컴파일러 규칙과 겹칠 수 있어 피하는 것이 안전합니다.

#### 자주 쓰는 네이밍 스타일

`snake_case`는 단어를 소문자와 밑줄로 이어 붙이는 방식(`player_hp`), `camelCase`는 첫 단어만 소문자로 시작하고 이후 단어의 첫 글자를 대문자로 올립니다(`playerHp`). `PascalCase`는 각 단어의 첫 글자를 대문자로 씁니다(`PlayerHp`). C 코드에서는 변수에 `snake_case`나 `camelCase`를 많이 쓰고, `PascalCase`는 구조체나 타입 이름에 자주 맞춥니다. 무엇보다 **한 프로젝트 안에서 스타일을 통일**하는 것이 가장 중요합니다.

#### 유효·무효한 변수 이름 예시

아래는 앞의 이름 규칙이 코드에서 어떻게 보이는지 짧게 모은 예입니다.

```c
int score = 100;      // 가능
int player_hp = 80;   // 가능
int _temp = 0;        // 가능(가능하지만 남용 비권장)

// int 3level = 3;    // 불가: 숫자로 시작
// int my score = 10; // 불가: 공백 포함
// int for = 1;       // 불가: 예약어 사용
```

#### `printf`로 변수 값 출력해 보기

변수에 넣어 둔 값을 화면에서 확인하려면 `printf`를 쓰면 됩니다. **첫 번째 인자(argument, 함수에 넘기는 값)**에 출력할 문장 틀을 적고, 그 안에 `%`로 시작하는 **형식 지정자(format specifier, 출력 형태를 정하는 기호)**를 끼워 넣은 뒤, 쉼표 뒤에 **같은 순서**로 변수를 적으면 각 `%…` 자리에 값이 들어갑니다. 예를 들어 `printf("나이: %d\n", age);`에서 `%d`에 `age`의 값이 출력됩니다.

아래 예제를 저장한 뒤 컴파일·실행해 보면, `int`·`float`·`char`·`char[]`가 각각 어떻게 찍히는지 한 번에 비교할 수 있습니다. 정수는 보통 `%d`, 실수는 `%f`(소수 자릿수는 `%.1f`처럼 조절), 문자 한 글자는 `%c`, 문자열(`char[]`)은 `%s`를 자주 씁니다. 형식 지정자와 변수 타입을 어긋나게 맞추면 출력이 이상해지거나 위험할 수 있으니, **타입과 기호를 짝지어** 익히는 것이 좋습니다.

```c
#include <stdio.h>

int main(void) {
    int age = 20;
    float height = 173.5f;
    char grade = 'A';
    char name[] = "Kim";

    printf("이름: %s\n", name);
    printf("나이: %d\n", age);
    printf("키: %.1f\n", height);
    printf("등급: %c\n", grade);

    return 0;
}
```

**예상 출력**

```text
이름: Kim
나이: 20
키: 173.5
등급: A
```

**코드 해석**

- `printf`마다 첫 문자열 안의 형식 지정자 개수와 뒤에 오는 변수 개수·순서를 맞춥니다.
- `%.1f`는 실수를 소수 첫째 자리까지 반올림해 출력하는 예입니다.
- `char name[] = "Kim";`은 문자 여러 개를 연속으로 둔 **문자열**을 `name`에 두는 선언이며, `%s`로 통째로 출력합니다.

### 03-2 연산자란?

연산자는 값을 계산하거나 비교할 때 쓰는 기호입니다. 산술 연산자 `+`, `-`, `*`, `/`는 계산에, 비교 연산자 `==`, `!=`, `>` 등은 조건 판단에 쓰입니다. 논리 연산자 `&&`, `||`, `!`는 여러 조건을 한데 묶을 때 사용하며, 뒤에서 배울 조건문·반복문과 자주 같이 등장합니다.

연산자는 분류별로 한 번에 보면 훨씬 정리됩니다. 아래 표는 C에서 자주 쓰는 연산자를 최대한 넓게 모은 요약입니다.

| 분류 | 연산자 | 뜻(한 줄) | 예시 (`int a=10, b=3`) | 결과/설명 |
|---|---|---|---|---|
| 산술 | `+` | 더하기 | `a + b` | `13` |
| 산술 | `-` | 빼기 | `a - b` | `7` |
| 산술 | `*` | 곱하기 | `a * b` | `30` |
| 산술 | `/` | 나누기 | `a / b` | `3`(정수 나눗셈) |
| 산술 | `%` | 나머지 | `a % b` | `1` |
| 증감 | `++` | 1 증가 | `a++` | 사용 후 `a`가 1 증가 |
| 증감 | `--` | 1 감소 | `--b` | 사용 전 `b`가 1 감소 |
| 대입 | `=` | 오른쪽 값을 대입 | `a = b` | `a`에 `b` 값 저장 |
| 복합 대입 | `+=`, `-=`, `*=`, `/=`, `%=` | 연산 후 대입 | `a += b` | `a = a + b`와 동일 |
| 복합 대입 | `<<=`, `>>=`, `&=`, `\|=`, `^=` | 비트 연산 후 대입 | `a <<= 1` | `a = a << 1`과 동일 |
| 비교 | `==`, `!=` | 같은지/다른지 비교 | `a == b` | `0` 또는 `1` |
| 비교 | `>`, `<`, `>=`, `<=` | 대소 비교 | `a >= b` | `0` 또는 `1` |
| 논리 | `&&` | 둘 다 참이면 참 | `(a > 5) && (b < 5)` | `1` |
| 논리 | `\|\|` | 하나라도 참이면 참 | `(a < 5) \|\| (b < 5)` | `1` |
| 논리 | `!` | 참/거짓 반전 | `!(a == b)` | `1` |
| 비트 | `&` | AND(같은 비트가 1일 때 1) | `a & b` | 비트 단위 계산 |
| 비트 | `\|` | OR(둘 중 하나가 1이면 1) | `a \| b` | 비트 단위 계산 |
| 비트 | `^` | XOR(서로 다르면 1) | `a ^ b` | 비트 단위 계산 |
| 비트 | `~` | NOT(비트 반전) | `~a` | 모든 비트 반전 |
| 비트 이동 | `<<`, `>>` | 비트를 좌/우로 이동 | `a << 1` | 보통 2배/2로 나눔 효과 |
| 조건(삼항) | `?:` | 조건에 따라 두 값 중 선택 | `(a>b) ? a : b` | 큰 값 선택 |
| 형 변환 | `(type)` | 타입을 명시적으로 바꿈 | `(double)a / b` | 실수 나눗셈 수행 |
| 크기 확인 | `sizeof` | 타입/값의 바이트 크기 확인 | `sizeof(int)` | 구현별 크기 반환 |
| 기타 | `,` | 왼쪽부터 순서대로 계산 | `x = (a=1, b=2, a+b)` | 마지막 식 값 사용 |

비교·논리 연산 결과는 C에서 보통 `참은 1`, `거짓은 0`으로 다룹니다. 또 `/`는 피연산자가 모두 정수일 때 소수점을 버리는 정수 나눗셈이 되고, 비트 연산은 내부 비트 패턴을 직접 다루므로 일반 산술/논리와 구분해서 써야 합니다.

주요 연산자 예시는 다음과 같습니다.

```c
#include <stdio.h>

int main(void) {
    int a = 10, b = 3;

    // 산술 연산자
    printf("a + b = %d\n", a + b); // 13
    printf("a - b = %d\n", a - b); // 7
    printf("a * b = %d\n", a * b); // 30
    printf("a / b = %d\n", a / b); // 3 (정수 나눗셈)
    printf("a %% b = %d\n", a % b); // 1 (나머지)

    // 비교 연산자 (참:1, 거짓:0)
    printf("a > b = %d\n", a > b);   // 1
    printf("a == b = %d\n", a == b); // 0

    // 논리 연산자
    printf("(a > 5) && (b < 5) = %d\n", (a > 5) && (b < 5)); // 1
    printf("(a < 5) || (b < 5) = %d\n", (a < 5) || (b < 5)); // 1
    printf("!(a == b) = %d\n", !(a == b)); // 1

    // 삼항 연산자
    printf("max(a, b) = %d\n", (a > b) ? a : b); // 10

    return 0;
}
```

### 연습문제 (연산자)

**문제 1**
- 문제: `int x = 8, y = 5;`일 때 `x + y`, `x / y`, `x % y` 결과를 각각 구하세요.
- 입력: `x = 8`, `y = 5`
- 출력: 세 연산 결과값
- 조건(힌트): 정수 나눗셈 `x / y` 결과가 실수가 아닌 정수 몫인지 확인하세요.

**문제 2**
- 문제: `int score = 75;`일 때 `score >= 60`의 결과(참/거짓)를 출력하세요.
- 입력: `score = 75`
- 출력: 참/거짓 값(`1` 또는 `0`)
- 조건(힌트): 비교 연산 결과를 `%d`로 출력해 보세요.

**문제 3**
- 문제: `int age = 17;`일 때 `(age >= 20) || (age == 17)`의 결과를 확인하세요.
- 입력: `age = 17`
- 출력: 논리식 결과(`1` 또는 `0`)
- 조건(힌트): OR(`||`)는 둘 중 하나라도 참이면 참입니다.

**문제 4**
- 문제: `int a = 10, b = 3;`일 때 삼항 연산자 `a > b ? a : b`의 결과를 출력하세요.
- 입력: `a = 10`, `b = 3`
- 출력: 더 큰 값 1개
- 조건(힌트): 조건이 참이면 `?` 뒤 값, 거짓이면 `:` 뒤 값을 선택합니다.

### 정답 (연산자)

문제 1은 각각 `13`, `1`, `3`입니다. 문제 2는 거짓(`0`), 문제 3은 참(`1`)입니다. 문제 4는 `10`입니다.

### 03-3 `getch()` / `scanf()` 키보드 입력

`getch()`는 키보드를 **한 글자** 단위로 받거나, 콘솔 창이 바로 닫히지 않게 잠시 멈춰 둘 때 입문 단계에서 자주 씁니다. 다만 환경에 따라 동작이 달라질 수 있으므로, 표준에 가까운 `getchar()`나 `scanf()`와 비교해 보는 것도 좋습니다. **Windows + Visual Studio(MSVC)**에서는 아래 `getch()` / `conio.h` 예시를 그대로 따라 할 수 있지만, 다른 컴파일러에서는 `getchar()`나 `scanf()` 예제로 바꾸는 편이 안전합니다.

`scanf()`는 정수·실수·문자열처럼 **형식이 정해진 데이터**를 읽어 변수에 넣을 때 씁니다. 숫자형에는 보통 변수의 **주소**를 넘기므로 `&변수명` 형태가 자주 나옵니다. `scanf`가 값을 기록하려면 “어디에 써야 하는지”를 알아야 하기 때문입니다. `&`는 그 변수가 있는 메모리 위치를 넘기는 기호로 이해하면 됩니다. 예를 들어 `scanf("%d", &age);`는 `age`가 저장된 위치에 입력값을 씁니다. `char[]` 문자열은 배열 이름이 시작 위치처럼 동작하는 경우가 많아 `scanf("%s", name);`처럼 `&` 없이 쓰는 패턴도 있습니다. 주소와 포인터의 자세한 원리는 **Chapter 12 포인터의 이해**에서 본격적으로 다룹니다.

`getch()` 기본 예시는 다음과 같습니다.

```c
#include <stdio.h>
#include <conio.h> // getch() 함수를 사용하기 위해 가져와야하는 conio.h 파일을 현재 코드에 추가

int main(void) {
    char key;

    printf("아무 키나 누르세요: ");
    key = getch(); // 키 1개 입력 받기

    printf("\n입력한 키: %c\n", key);
    return 0;
}
```

화면이 바로 닫히는 것을 막고 싶을 때는 아래처럼 마지막에 `getch()`로 한 번 기다릴 수 있습니다.

```c
#include <stdio.h>
#include <conio.h>

int main(void) {
    printf("작업 완료!\n");
    printf("종료하려면 아무 키나 누르세요...");
    getch(); // 키를 누를 때까지 대기
    return 0;
}
```

### 연습문제 (`getch()`)

**문제 1**
- 문제: 키 하나를 입력받아, 그 문자를 그대로 다시 출력해 보세요.
- 입력: 문자 키 1개
- 출력: 입력한 문자 1개
- 조건(힌트): `getch()`로 입력받은 문자를 `printf`로 출력하세요.

**문제 2**
- 문제: 키 하나를 입력받아, ASCII 코드(정수값)도 함께 출력해 보세요.
- 입력: 문자 키 1개
- 출력: 입력 문자와 ASCII 코드
- 조건(힌트): ASCII 코드는 `%d` 형식 지정자를 사용하세요.

### 정답 예시 (`getch()`)

```c
#include <stdio.h>
#include <conio.h>

int main(void) {
    char ch;

    printf("문자 1개 입력: ");
    ch = getch();

    printf("\n입력 문자: %c\n", ch);
    printf("ASCII 코드: %d\n", ch);

    return 0;
}
```

`scanf()` 기본 예시는 다음과 같습니다.

```c
#include <stdio.h>

int main(void) {
    int age;
    float height;
    char name[20];

    printf("이름 나이 키를 입력하세요 (예: Kim 20 173.5): ");
    scanf("%19s %d %f", name, &age, &height);

    printf("입력 결과 -> 이름: %s, 나이: %d, 키: %.1f\n", name, age, height);
    return 0;
}
```

`scanf()`에서 자주 나는 실수는 세 가지로 묶어 볼 수 있습니다. 첫째, `%d`인데 `float`에 받거나 `%f`인데 `int`에 받는 식으로 **형식 지정자와 변수 타입**을 어긋나게 맞추는 경우입니다. 둘째, 숫자 입력에서 `&`를 빼먹어 `scanf("%d", age);`처럼 쓰는 경우입니다. 셋째, 문자열을 받을 때 버퍼 크기를 생각하지 않아 넘치는 입력이 들어갈 수 있는 경우인데, `%19s`처럼 **최대 길이를 제한**하는 형태가 더 안전한 편입니다.

### 연습문제 (`scanf()`)

**문제 1**
- 문제: 정수 두 개를 입력받아 합을 출력해 보세요.
- 입력: 정수 2개
- 출력: 합계 1개
- 조건(힌트): `scanf("%d %d", &a, &b);` 형태로 입력받으세요.

**문제 2**
- 문제: 이름(`char[]`)과 점수(`int`)를 입력받아 `"이름: OOO, 점수: 100"` 형태로 출력해 보세요.
- 입력: 이름(문자열) 1개, 점수(정수) 1개
- 출력: `"이름: ..., 점수: ..."` 형식의 한 줄
- 조건(힌트): 문자열 버퍼 길이에 맞게 `%19s`처럼 입력 길이를 제한하세요.

### 정답 예시 (`scanf()`)

```c
#include <stdio.h>

int main(void) {
    int a, b;
    char name[20];
    int score;

    printf("정수 2개 입력: ");
    scanf("%d %d", &a, &b);
    printf("합계: %d\n", a + b);

    printf("이름 점수 입력: ");
    scanf("%19s %d", name, &score);
    printf("이름: %s, 점수: %d\n", name, score);

    return 0;
}
```

`getch()`는 키 한 개를 바로 받거나 대기할 때, `scanf()`는 정수·실수·문자열처럼 형식이 있는 입력을 변수에 나누어 넣을 때 쓰면 역할이 잘 맞습니다.
