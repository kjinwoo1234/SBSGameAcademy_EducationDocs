- 행과 열 기준 순회의 차이를 이해한다.

## 세부 주제

- 표 형태 데이터 저장과 탐색

## 실습 체크리스트
- 2차원 배열로 간단한 게임 맵 데이터를 표현한다.

## 본문

### 16-1 다차원 배열 활용

2차원 배열은 **행(row)**과 **열(column)**이 있는 표처럼 데이터를 놓을 때 적합합니다. 게임 맵, 좌석표, 여러 과목 점수표처럼 “가로·세로”가 동시에 의미 있을 때 자주 등장합니다.

### 16-2 3차원 배열

3차원 배열은 그 위에 **층(layer)** 개념을 하나 더 얹은 구조로 생각할 수 있습니다. `(x, y, z)` 좌표나, 같은 격자가 시간·층마다 반복되는 데이터를 표현할 때 활용할 수 있습니다.

| 차원 | 형태 | 예시 |
|---|---|---|
| 1차원 | 선형 리스트 | 점수 목록 |
| 2차원 | 행/열 테이블 | 게임 맵 |
| 3차원 | 층 + 행/열 | 3D 좌표 데이터 |

```c
#include <stdio.h>

int main(void) {
    int map[2][3] = {
        {1, 0, 1},
        {0, 1, 0}
    };

    for (int r = 0; r < 2; r++) {
        for (int c = 0; c < 3; c++) {
            printf("%d ", map[r][c]);
        }
        printf("\n");
    }
    return 0;
}
```

**예상 출력**

```text
1 0 1
0 1 0
```

### 연습문제

**문제 1**  
3×3 정수 배열의 모든 원소 합을 구하세요. 입력은 정수 아홉 개, 출력은 합계 하나입니다. 중첩 반복문을 사용합니다.

**문제 2**  
2차원 배열과 목표값 `target`이 주어질 때, `target`과 같은 값이 몇 개인지 세어 출력하세요. 행·열 범위를 벗어나지 않게 순회합니다.

### 정답 포인트

중첩 반복문으로 행과 열을 모두 돌고, 인덱스는 `0`부터 **크기 − 1**까지인지 매번 확인합니다.
