# Chapter 24 파일 입출력

## 학습 목표
- 파일 스트림 기반 입출력의 기본 흐름을 익힌다.
- 텍스트/바이너리 모드 차이를 이해한다.

## 세부 주제
### 24-1 파일/스트림/기본 입출력
- 파일 포인터와 기본 함수

### 24-2 파일 개방 모드
- 읽기/쓰기/추가 모드 구분

### 24-3 파일 입출력 함수
- `fopen`, `fclose`, `fread`, `fwrite`

### 24-4 텍스트/바이너리 입출력
- 데이터 저장 형식 차이

### 24-5 파일 위치 지시자 이동
- `fseek`, `ftell`, `rewind`

## 실습 체크리스트
- 구조체 배열을 바이너리 파일로 저장/복원한다.

## 본문

### 24-1 파일/스트림/기본 입출력
- 파일 작업은 `FILE*` 스트림을 통해 처리합니다.
- 기본 흐름은 `열기 -> 읽기/쓰기 -> 닫기`입니다.

### 24-2 파일 개방 모드
- `"r"` 읽기, `"w"` 쓰기(기존 내용 삭제), `"a"` 추가 모드를 먼저 익힙니다.
- 바이너리 모드는 `"rb"`, `"wb"`처럼 `b`를 붙입니다.

### 24-3 파일 입출력 함수
- 텍스트 중심: `fprintf`, `fscanf`, `fgets`, `fputs`
- 바이너리 중심: `fread`, `fwrite`

### 24-4 텍스트/바이너리 입출력
- 텍스트는 사람이 읽기 쉽고, 바이너리는 저장/복원이 빠르고 정확합니다.

### 24-5 파일 위치 지시자 이동
- `fseek`, `ftell`, `rewind`로 파일 포인터 위치를 제어합니다.

예시 코드:
```c
#include <stdio.h>

int main(void) {
    FILE *fp = fopen("score.txt", "w");
    if (!fp) return 1;

    fprintf(fp, "Kim 95\n");
    fclose(fp);
    return 0;
}
```

연습문제:
1. 사용자 이름/점수를 파일에 저장
2. 저장한 파일을 다시 읽어 콘솔에 출력

정답 포인트:
- `fopen` 실패 체크
- 사용 후 `fclose` 필수
