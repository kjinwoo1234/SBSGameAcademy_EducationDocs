## 본문

### 24-1 파일/스트림/기본 입출력

파일 작업은 `FILE*` 타입의 **스트림**을 통해 이루어집니다. 흐름은 대개 **열기 → 읽거나 쓰기 → 닫기** 순으로 잡으면 됩니다.

| 용어 | 의미 |
|---|---|
| 파일 스트림(stream) | 파일과 프로그램 사이에서 데이터가 이동하는 통로 |
| 파일 포인터(`FILE*`) | 현재 열려 있는 파일 스트림을 가리키는 포인터 |
| 파일 위치 지시자 | 파일 안에서 지금 읽거나 쓰는 위치를 나타내는 내부 커서 |

### 24-2 파일 개방 모드

`"r"`은 읽기, `"w"`는 쓰기(있으면 내용을 비우고 새로 쓰는 쪽에 가깝습니다), `"a"`는 파일 끝에 이어 붙이기입니다. 바이너리로 다룰 때는 `"rb"`, `"wb"`처럼 `b`를 붙입니다.

### 24-3 파일 입출력 함수

텍스트 위주로 다룰 때는 `fprintf`, `fscanf`, `fgets`, `fputs` 등을 쓰고, 바이너리 블록을 그대로 옮길 때는 `fread`와 `fwrite`가 자주 쓰입니다.

### 24-4 텍스트/바이너리 입출력

텍스트 모드는 사람이 에디터로 열어 읽기 좋고, 바이너리 모드는 구조체나 배열을 **비트 그대로** 저장했다가 복원하기에 적합한 경우가 많습니다.

### 24-5 파일 위치 지시자 이동

`fseek`, `ftell`, `rewind`로 파일 안의 읽기/쓰기 위치를 옮기거나, 현재 위치를 숫자로 알 수 있습니다.

| 모드 | 의미 |
|---|---|
| `"r"` | 읽기 전용 |
| `"w"` | 쓰기(기존 내용 삭제) |
| `"a"` | 파일 끝에 추가 |
| `"rb"`, `"wb"` | 바이너리 읽기/쓰기 |

```c
#include <stdio.h>

int main(void) {
    FILE *fp = fopen("score.txt", "w");
    if (!fp) return 1;

    fprintf(fp, "Kim 95\n");
    fclose(fp);
    return 0;
}
```

실행 후에는 `score.txt` 파일이 생기고 그 안에 `Kim 95`가 저장된 상태가 됩니다.

### 연습문제

**문제 1**  
이름과 점수를 입력받아 텍스트 파일에 저장하세요. `fopen`이 실패하면 반드시 처리합니다.

**문제 2**  
저장한 파일을 다시 열어 내용을 콘솔에 출력하세요. 사용이 끝나면 `fclose`를 호출합니다.

### 정답 포인트

`fopen`의 결과가 `NULL`인지 확인하고, 열었던 파일은 `fclose`로 닫는 것을 잊지 않으면 됩니다.
