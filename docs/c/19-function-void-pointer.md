## 본문

### 19-1 함수 포인터 / `void` 포인터

함수 포인터는 **함수가 들어 있는 메모리 위치**를 저장하는 포인터입니다. 메뉴 번호에 따라 다른 함수를 호출한다든지, 정렬할 때 비교 함수를 넘긴다든지 할 때 유용합니다. `void*`는 “어떤 타입의 주소든 담을 수 있는 **범용 포인터**”로 쓰이지만, 실제로 값을 읽거나 쓸 때는 **다시 구체적인 타입**으로 변환해야 합니다.

**콜백(callback, 콜백)**은 함수 하나를 다른 함수에 넘겨 두었다가, 필요한 시점에 그쪽에서 **대신 호출**해 주는 방식을 말합니다.

### 19-2 `main` 함수 인자 전달

`main`에 `argc`, `argv`를 받는 형태를 쓰면, 프로그램을 실행할 때 **명령줄에서 문자열 인자**를 넘길 수 있습니다. 예를 들어 `program.exe hello`를 실행하면 `argv[1]`이 `"hello"`에 해당하는 문자열을 가리키는 식으로 접근합니다.

| 항목 | 의미 |
|---|---|
| 함수 포인터 | 함수 주소를 저장하는 포인터 |
| `void*` | 타입 정보 없이 주소만 담는 범용 포인터 |
| `argc` | 인자 개수 |
| `argv` | 인자 문자열 배열 |

```c
#include <stdio.h>

int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }

int main(int argc, char *argv[]) {
    int (*op[2])(int, int) = {add, sub};
    printf("add: %d\n", op[0](7, 3));
    printf("sub: %d\n", op[1](7, 3));
    printf("argc: %d\n", argc);
    return 0;
}
```

**예상 출력(인자 없이 실행 시)**

```text
add: 10
sub: 4
argc: 1
```

### 연습문제

**문제 1**  
`add`, `sub`에 더해 `mul`, `div`를 넣어 네 가지 연산을 함수 포인터 배열로 처리하세요. 0으로 나누는 경우는 예외 처리합니다.

**문제 2**  
`argc`와 `argv`로 실행 시 넘긴 모든 인자를 순서대로 출력하세요. `for (i = 0; i < argc; i++)` 같은 루프를 쓰면 됩니다.

### 정답 포인트

함수 포인터 선언은 `반환형 (*이름)(매개변수)` 형태로 익히고, `argv`는 문자열 배열로 다루면 됩니다.
