## 본문

### 04-1 컴퓨터의 데이터 표현

컴퓨터는 모든 데이터를 0과 1의 조합으로 저장합니다. 이 한 자리를 **비트(bit)**라고 부릅니다. 8비트가 모이면 **1바이트(byte)**가 되고, 자료형마다 사용하는 바이트 수가 달라집니다. 결국 문자·숫자·이미지도 내부에서는 비트 패턴으로만 구분됩니다.

용량 단위를 한 번에 정리하면 다음과 같습니다. 1비트는 0 또는 1 한 자리이고, 8비트가 1바이트이며, 1024바이트가 1KB, 1024KB가 1MB로 이어집니다(상용 표기에서 1000 기준을 쓰는 경우도 있으나, 여기서는 학습용으로 1024 배수를 전제로 합니다).

Windows에서 MSVC를 쓸 때 자주 보는 크기의 예는 `char` 1바이트, `int` 4바이트, `float` 4바이트, `double` 8바이트 정도로 기억해 두면 됩니다. 자료형의 종류와 의미는 **[Chapter 05 상수와 기본 자료형](./05-constants-types.md)**에서 이어서 정리합니다.

### 04-2 정수/실수 표현 방식

정수는 부호와 크기를 **정해진 비트 수** 안에 맞춰 저장합니다. 실수는 소수점을 포함한 값을 다루기 위해 **부동소수점(floating-point)**이라는 별도의 비트 규칙을 씁니다. 실수 연산에서는 아주 작은 오차가 누적될 수 있으므로, 결과를 비교할 때는 `==`만으로 판단하지 않는 습관이 필요합니다.

#### 필수 이해(먼저 여기까지만)

실수는 비트로 **정확한 값이 아니라 근사값**으로 저장되는 경우가 많습니다. 그래서 `0.1 + 0.2`가 우리가 기대하는 `0.3`과 비트 단위로 완전히 같지 않을 수 있습니다. 실수를 비교할 때는 `==` 대신 **허용 오차**를 두고 가까운지를 보는 방식이 안전합니다.

#### 심화: 부동소수점이 비트에 얹히는 큰 그림(IEEE 754)

C의 `float`와 `double`은 대부분 **IEEE 754** 규격으로 저장됩니다. 비트를 크게 나누면 **부호**, **지수**, **가수** 세 부분으로 생각할 수 있습니다. 부호 비트는 `0`이면 양수, `1`이면 음수에 대응합니다. 지수 비트는 값의 크기(소수점을 어디까지 옮겼는지)를 담고, 가수 비트는 유효 숫자(정밀도)를 담습니다.

일반적인 구성은 `float`(32비트)가 부호 1비트 + 지수 8비트 + 가수 23비트, `double`(64비트)이 부호 1비트 + 지수 11비트 + 가수 52비트로 잡히는 경우가 많습니다. `float` 32비트를 한 줄로 그리면 아래처럼 생각할 수 있습니다.

```text
[부호 1칸][지수 8칸][가수 23칸]
 s eeeeeeee fffffffffffffffffffffff
```

부호 한 칸은 앞에서 말한 대로 양·음을 가릅니다. 지수 여덟 칸은 소수점을 얼마나 옮겼는지에 해당하는데, 여기에는 실제 지수를 그대로 넣기보다 **바이어스(bias)를 더한 값**을 저장하는 방식이 쓰입니다. 가수 스물세 칸은 유효 숫자를 담으며, 정규화된 수에서는 앞의 `1`을 생략해 저장하는 **숨은 비트(hidden bit)** 아이디어가 함께 따라옵니다. 아주 작은 값(비정규화 수, subnormal)은 이 규칙의 예외가 될 수 있습니다.

#### 바이어스가 필요한 이유

지수칸만으로는 음수 지수도 표현해야 하므로, 미리 정해 둔 기준값(바이어스)을 더해 “항상 저장값이 한쪽 방향으로만 보이게” 만듭니다. `float`는 지수가 8비트일 때 바이어스가 `127`인 경우가 많고, 저장할 때는 `저장지수 = 실제지수 + 127`, 읽을 때는 `실제지수 = 저장지수 - 127`처럼 되돌립니다. 예를 들어 실제 지수가 `+3`이면 저장지수는 `130`, 실제 지수가 `-3`이면 저장지수는 `124`가 됩니다.

값을 실제 숫자로 해석하는 공식은 보통 `값 = (-1)^부호 × (1.가수비트) × 2^(지수값 - 바이어스)` 형태로 정리됩니다. `float`의 바이어스는 127, `double`은 1023이 흔합니다. 정규화된 수는 앞자리가 1이 되도록 맞추기 때문에 `1.가수` 형태로 읽는 것이고, 가수 비트에는 소수점 아래 위주가 들어갑니다.

#### 예시 1) 13.25가 저장되는 흐름

먼저 10진수를 2진수로 바꾸면 `13.25 = 1101.01(2)`가 됩니다. 이를 `1.10101 × 2^3` 꼴로 정규화합니다. 양수이면 부호는 `0`, 지수는 `3 + 127 = 130`을 2진수로 옮겨 넣고, 가수는 `10101` 뒤를 0으로 채워 23비트를 맞춥니다.

#### 예시 2) 0.1에서 오차가 생기는 이유

10진수 `0.1`은 2진수로 `0.0001100110011...`처럼 **끝없이 반복**됩니다. `float`/`double`의 가수 비트는 길이가 유한하므로 중간에서 잘라 **가장 가까운 근사값**만 저장합니다. 그 근사값끼리 더하면 `0.1 + 0.2`가 `0.3`과 완전히 같지 않게 보일 수 있습니다.

실무에서는 실수 동등 비교에 `==`를 쓰지 않고, 예를 들어 `fabs(a - b) < 1e-9`처럼 **오차 허용**으로 비교합니다.

```c
#include <stdio.h>

int main(void) {
    double x = 0.1 + 0.2;
    printf("%.17f\n", x); // 0.30000000000000004 처럼 보일 수 있음
    return 0;
}
```

### 04-3 비트 연산자

비트 연산은 정수를 비트 단위로 직접 다루는 기능입니다. `&`, `|`, `^`, `~`, `<<`, `>>`로 특정 비트를 켜거나 끄거나, 패턴을 검사할 수 있습니다. 게임·임베디드·시스템 코드에서 상태 플래그를 압축해 두고 싶을 때 자주 등장합니다.

연산자의 뜻을 한 줄씩 정리하면 다음과 같습니다. `&`(AND)는 두 비트가 모두 1일 때만 1이고, `|`(OR)는 하나라도 1이면 1입니다. `^`(XOR)는 서로 다를 때 1이며, `~`(NOT)는 0과 1을 뒤집습니다. `<<`는 비트를 왼쪽으로 밀어 보통 값이 2배씩 커지는 방향으로 이해하면 되고, `>>`는 오른쪽으로 밀어 보통 2로 나누는 방향에 가깝습니다(부호 있는 타입의 오른쪽 시프트는 구현/설정에 따라 주의가 필요합니다).

비트 연산 예시는 아래와 같습니다.

```c
#include <stdio.h>

int main(void) {
    unsigned char flags = 0; // 00000000

    flags |= (1 << 0); // bit0 ON
    flags |= (1 << 2); // bit2 ON -> 00000101
    printf("flags: %u\n", flags); // 5

    flags &= ~(1 << 0); // bit0 OFF -> 00000100
    printf("flags: %u\n", flags); // 4

    printf("bit2 is on? %d\n", (flags & (1 << 2)) != 0); // 1
    return 0;
}
```

### 연습문제

**문제 1**
- 문제: `num = 12`일 때 `num << 1`, `num >> 1` 결과를 구하세요.
- 입력: 정수 `num = 12`
- 출력: 왼쪽 시프트 결과와 오른쪽 시프트 결과
- 조건(힌트): 결과를 각각 `printf`로 출력해 확인하세요.

**문제 2**
- 문제: 정수의 3번째 비트를 ON으로 만드는 코드를 작성하세요.
- 입력: 정수 `num` 1개
- 출력: 비트 ON 적용 후 정수 값
- 조건(힌트): `num |= (1 << 2);` 형태를 사용하세요.

**문제 3**
- 문제: 정수의 2번째 비트가 켜져 있는지 검사하는 코드를 작성하세요.
- 입력: 정수 `num` 1개
- 출력: 켜짐 여부(참/거짓 또는 1/0)
- 조건(힌트): `(num & (1 << 1)) != 0` 형태를 활용하세요.

### 정답

문제 1은 각각 `24`, `6`입니다. 문제 2는 `num |= (1 << 2);`, 문제 3은 `(num & (1 << 1)) != 0`처럼 검사할 수 있습니다.
