## 본문

### 04-1 컴퓨터의 데이터 표현
- 컴퓨터는 모든 데이터를 0(전기가 흐르지 않음)과 1(전기가 흐르고 있음)로 저장합니다. 이 단위를 비트라고 합니다.
- 8비트가 1바이트이며, 자료형마다 사용하는 바이트 수가 다릅니다.
- 결국 문자, 숫자, 이미지도 내부에서는 비트 조합으로 저장됩니다.

용량 단위 기초:
- 1 bit: 0 또는 1 한 자리
- 8 bit = 1 byte
- 1024 byte = 1 KB
- 1024 KB = 1 MB

자주 쓰는 자료형 용량(일반적인 Windows MSVC 기준):
- `char`: 1바이트
- `int`: 4바이트
- `float`: 4바이트
- `double`: 8바이트
- 자료형의 종류와 용량은 [Chapter 05 상수와 기본 자료형](./05-constants-types.md)에서 학습합니다.

### 04-2 정수/실수 표현 방식
- 정수는 부호와 크기를 정해진 비트 수 안에 저장합니다.
- 실수는 소수점을 포함한 값을 저장하기 위해 별도의 표현 규칙(부동소수점)을 사용합니다.
- 실수 계산에서는 작은 오차가 발생할 수 있으므로, 결과 비교 시 주의가 필요합니다.

필수 이해(먼저 여기까지만):
- 실수는 비트를 사용해 "근사값"으로 저장될 수 있습니다.
- 그래서 `0.1 + 0.2`가 정확히 `0.3`이 아닐 수 있습니다.
- 실수 비교 시 `==` 대신 오차 허용 비교를 사용합니다.

심화(지금은 넘겨도 되는 범위):
부동소수점 실제 저장 방식(IEEE 754):
- C의 `float`, `double`은 대부분 IEEE 754 규격으로 저장됩니다.
- 저장된 비트는 크게 3부분입니다.
  1. 부호 비트(Sign): `0`이면 양수, `1`이면 음수
  2. 지수 비트(Exponent): 값의 크기(소수점 이동량)
  3. 가수 비트(Fraction/Mantissa): 유효 숫자

대표 비트 구성(일반적인 환경):
- `float`(32비트): 부호 1 + 지수 8 + 가수 23
- `double`(64비트): 부호 1 + 지수 11 + 가수 52

`float` 32비트를 칸으로 보면:
```text
[부호 1칸][지수 8칸][가수 23칸]
 s eeeeeeee fffffffffffffffffffffff
```

각 칸이 하는 일:
- 부호 1칸(`s`)
  - `0`이면 양수, `1`이면 음수
- 지수 8칸(`eeeeeeee`)
  - 소수점을 얼마나 이동할지(숫자의 크기)를 저장
  - 단, 지수칸에는 실제 지수값을 그대로 저장하지 않고 "바이어스가 더해진 값"을 저장
- 가수 23칸(`ffffffff...`)
  - 숫자의 유효 자릿수(정밀도)를 저장
  - 정규화된 수에서는 앞의 `1`은 생략 저장(숨은 1, hidden bit)

바이어스(Bias)가 뭐냐?
- 바이어스는 지수칸에 음수 지수까지 저장하기 위해 미리 더해두는 기준값입니다.
- `float`는 지수칸이 8비트라 바이어스가 `127`입니다.
- 저장할 때: `저장지수 = 실제지수 + 127`
- 읽을 때: `실제지수 = 저장지수 - 127`

간단 예시:
- 실제 지수가 `+3`이면 저장지수는 `130` (`3 + 127`)
- 실제 지수가 `-3`이면 저장지수는 `124` (`-3 + 127`)
- 이렇게 하면 지수칸에는 음수도 "양수 형태"로 저장할 수 있습니다.

저장값을 실제 숫자로 해석하는 공식:
- `값 = (-1)^부호 × (1.가수비트) × 2^(지수값 - 바이어스)`
- `float` 바이어스: 127, `double` 바이어스: 1023

왜 `1.가수비트` 형태인가?
- 정규화된 부동소수점(normalized number)은 앞자리가 `1`이 되도록 맞춰 저장합니다.
- 이 앞자리 `1`은 저장하지 않고(숨은 비트, hidden bit), 가수 비트에는 소수점 아래 부분만 저장합니다.
- 단, 매우 작은 값(비정규화 수, subnormal)은 이 규칙의 예외가 될 수 있습니다.

예시 1) 13.25가 저장되는 큰 흐름:
1. 10진수 -> 2진수: `13.25 = 1101.01(2)`
2. 정규화: `1.10101 × 2^3`
3. 부호: 양수이므로 `0`
4. 지수: `3 + 127 = 130` -> 2진수 `10000010` (`float` 기준)
5. 가수: `10101` 뒤를 0으로 채워 23비트 구성

예시 2) 0.1에서 오차가 생기는 이유:
- `0.1(10)`은 `0.0001100110011...`처럼 2진수에서 무한 반복됩니다.
- `float`/`double`은 가수 비트 수가 제한되어 있어 중간에서 잘라 저장합니다.
- 따라서 정확한 0.1이 아니라 "가장 가까운 근사값"이 저장됩니다.
- 이 근사값들끼리 계산하면 `0.1 + 0.2` 결과가 `0.3`과 완전히 같지 않을 수 있습니다.

실무 주의사항:
- 실수 동등 비교는 `==` 대신 오차 허용 비교를 사용합니다.
- 예: `fabs(a - b) < 1e-9`

오차 예시:
```c
#include <stdio.h>

int main(void) {
    double x = 0.1 + 0.2;
    printf("%.17f\n", x); // 0.30000000000000004 처럼 보일 수 있음
    return 0;
}
```

### 04-3 비트 연산자
- 비트 연산은 숫자를 비트 단위로 직접 조작할 수 있는 기능입니다.
- `&`, `|`, `^`, `~`, `<<`, `>>`를 이용해 특정 비트를 켜거나 끄는 작업을 할 수 있습니다.
- 게임/임베디드/시스템 코드에서 상태 플래그를 효율적으로 관리할 때 자주 사용됩니다.

비트 연산자 기능 요약:
- `&` (AND): 두 비트가 모두 1일 때만 1
- `|` (OR): 둘 중 하나라도 1이면 1
- `^` (XOR): 두 비트가 다를 때 1
- `~` (NOT): 0과 1을 반전
- `<<` (왼쪽 시프트): 비트를 왼쪽으로 이동(보통 값이 2배씩 증가)
- `>>` (오른쪽 시프트): 비트를 오른쪽으로 이동(보통 값이 2로 나눠짐)

비트 연산 예시:
```c
#include <stdio.h>

int main(void) {
    unsigned char flags = 0; // 00000000

    flags |= (1 << 0); // bit0 ON
    flags |= (1 << 2); // bit2 ON -> 00000101
    printf("flags: %u\n", flags); // 5

    flags &= ~(1 << 0); // bit0 OFF -> 00000100
    printf("flags: %u\n", flags); // 4

    printf("bit2 is on? %d\n", (flags & (1 << 2)) != 0); // 1
    return 0;
}
```
