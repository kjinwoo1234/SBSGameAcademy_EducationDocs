# Chapter 08 반복문

## 학습 목표
- 반복문의 종류와 사용 기준을 이해한다.
- 반복 제어 변수를 올바르게 다룰 수 있다.
- 반복문 안에서 **`continue`(컨티뉴, “이번 회차만 건너뛴다”)**와 **`break`(브레이크, “반복 자체를 끝낸다”)**가 흐름에 주는 차이를 구분한다.

## 세부 주제
### 08-1 `while`
- 조건 기반 반복의 기본 패턴

### 08-2 `do-while`
- 최소 1회 실행이 필요한 경우

### 08-3 `for`
- 카운트 기반 반복과 중첩 반복

### 08-4 `break`와 `continue`
- 현재 반복을 건너뛰기와 반복문 전체 종료

## 실습 체크리스트
- `while`과 `for`로 동일 문제를 각각 구현해 비교한다.
- 같은 조건이라도 **`continue`로 건너뛸 때**와 **`break`로 끝낼 때** 출력이 어떻게 달라지는지 확인한다.

## 본문

### 08-1 `while`

`while`은 괄호 안 조건이 **참인 동안** 본문을 반복합니다. 반복 횟수가 처음부터 정해지지 않았거나, 어떤 조건이 만족될 때까지 기다려야 할 때 잘 맞습니다. 조건이 영원히 바뀌지 않으면 **무한 루프**가 되므로, 본문 안에서 조건에 영향을 주는 코드(예: 카운터 증가, 입력 읽기)를 빠뜨리지 않도록 주의합니다.

### 08-2 `do-while`

`do-while`은 조건을 검사하기 **전에** 본문을 한 번 실행합니다. 그래서 조건이 처음부터 거짓이어도 **최소 한 번**은 돌아갑니다. 메뉴를 한 번 보여 주고 입력을 받는 구조처럼 “한 번은 실행되어야 하는 루프”에 자주 씁니다. 문법 끝에 세미콜론(`while (조건);`)이 붙는 형태도 헷갈리기 쉬우니 처음엔 예제를 옆에 두고 맞춰 쓰는 것이 좋습니다.

### 08-3 `for`

`for`는 초기식, 조건식, 증감식을 한 줄에 모아 두는 반복문입니다. “1부터 100까지 합”처럼 **횟수나 범위가 분명할 때** 가장 읽기 쉬운 경우가 많습니다. `for`를 겹쳐 쓰면 별 찍기나 2차원 배열 순회처럼 “안쪽·바깥쪽 반복”이 필요한 문제도 표현하기 좋습니다.

세 가지 반복문의 차이를 한 표로 비교하면 아래와 같습니다.

| 반복문 | 실행 시점 | 적합한 상황 | 주의점 |
|---|---|---|---|
| `while` | 조건 확인 후 실행 | 종료 조건이 이벤트/입력에 따라 바뀔 때 | 조건 갱신 누락 시 무한 루프 |
| `do-while` | 본문 1회 실행 후 조건 확인 | 최소 1회 실행이 필요한 메뉴/입력 루프 | 끝에 세미콜론(`;`) 필요 |
| `for` | 초기화 -> 조건 -> 본문 -> 증감 | 반복 횟수가 명확할 때 | 인덱스 범위 실수 주의 |

아래 예시에서 `for (i = 1; i <= 5; i++)`는 `i`를 1에서 시작해 5 이하인 동안 돌리고, 매 반복 끝에 `i++`로 하나씩 올립니다. `sum += i`로 1부터 5까지 누적합을 만듭니다. 이어서 `while (i > 0)`은 조건이 참인 동안 숫자를 거꾸로 출력하는데, 여기서 `i--`가 없으면 조건이 바뀌지 않아 끝나지 않을 수 있습니다.

```c
#include <stdio.h>

int main(void) {
    int i, sum = 0;

    for (i = 1; i <= 5; i++) {
        sum += i;
    }
    printf("1~5 합: %d\n", sum);

    i = 5;
    while (i > 0) {
        printf("%d ", i);
        i--;
    }
    printf("\n");
    return 0;
}
```

**예상 출력**

```text
1~5 합: 15
5 4 3 2 1
```

`do-while`로 메뉴를 한 번 이상 보여 주는 예는 다음과 같습니다.

```c
#include <stdio.h>

int main(void) {
    int menu = 0;

    do {
        printf("메뉴: 1. 시작  2. 종료\n");
        printf("입력: ");
        scanf("%d", &menu);
    } while (menu != 2);

    printf("프로그램 종료\n");
    return 0;
}
```

### 08-4 `break`와 `continue`

반복문은 보통 조건식이 참인 한 계속 도는데, 그 안에서 “이번 회차만 건너뛰고 다음 조건 검사로” 가고 싶을 때 **`continue`**를 씁니다. `for`에서는 남아 있던 본문을 실행하지 않고 **증감식으로 건너뛴 뒤** 조건을 다시 검사합니다. **`break`**는 조건식이 아직 참이어도 **반복문 전체를 즉시 벗어납니다**. 찾던 값을 만났다거나 오류 입력이 들어와 더 순회할 이유가 없을 때처럼, 루프를 중간에 끊어야 하는 상황에 맞습니다.

두 키워드 모두 **`if`**와 함께 쓰는 경우가 많습니다. `continue`는 “특정 경우엔 처리하지 않음” 패턴으로, `break`는 “목표 조건 만족 시 조기 종료” 패턴으로 읽히면 이해하기 쉽습니다. 중첩 반복에서는 `break`가 **가장 가까운 안쪽 반복문 한 겹만** 끝낸다는 점을 기억해 두세요. **`switch` 안의 `break`**도 같은 단어지만, 선택한 분기 처리만 마치고 **`switch` 블록 밖으로 빠져나간다**는 의미입니다. 분기 종류별 차이는 **[Chapter 07 조건문과 분기](./07-condition-branch.md)**의 `07-3`에서 이미 살폈던 내용입니다.

**오류 방지 3요소 (`continue` 위치 실수)**

- **발생 조건**: 특정 입력을 건너뛰려고 `continue`를 두었는데, 그보다 아래 줄에 같은 반복 안에서 반드시 실행되어야 하는 누적·출력 코드가 남아 있을 때입니다.
- **영향**: 일부 회차에서는 중요 처리가 실행되지 않아 합계가 줄거나 출력이 빈칸처럼 보일 수 있습니다.
- **해결 원리**: `continue`보다 위에서만 검증하고 누적하고, 건너뛸 때는 아직 할 일이 없는 시점에서만 다음 반복으로 넘어가도록 순서를 잡습니다.

아래 코드는 `1`부터 `10`까지 돌며 3의 배수는 출력하지 않고(`continue`), `8` 이상이면 반복 자체를 끝냅니다(`break`). `printf` 줄에 도달하기 전에 분기되는 경우를 따라가며 출력을 적어 보면 두 키워드 차이가 분명해집니다.

```c
#include <stdio.h>

int main(void) {
    int i;
    for (i = 1; i <= 10; i++) {
        if (i % 3 == 0) {
            continue;  // 3의 배수는 이번 반복 나머지 건너뜀
        }
        if (i >= 8) {
            break;    // 반복문 전체 종료
        }
        printf("%d ", i);
    }
    printf("\n");
    return 0;
}
```

**예상 출력**

```text
1 2 4 5 7
```

### 연습문제

**문제 1**
- 문제: `for`문으로 1부터 10까지 짝수만 출력하세요.
- 입력: 해당 없음
- 출력: `2 4 6 8 10`
- 조건(힌트): 초기값/조건/증감식 중 하나를 짝수 패턴에 맞게 설계하세요.

**문제 2**
- 문제: `while`문으로 10부터 1까지 거꾸로 출력하세요.
- 입력: 해당 없음
- 출력: `10 9 8 ... 1`
- 조건(힌트): 반복마다 값을 1씩 감소시키세요.

**문제 3**
- 문제: `do-while`로 메뉴를 한 번 이상 출력하는 코드를 작성하세요.
- 입력: 메뉴 선택 정수 1개 이상
- 출력: 메뉴 문장 최소 1회 출력
- 조건(힌트): 종료 조건을 만족해도 본문이 한 번은 실행되는지 확인하세요.

**문제 4**
- 문제: 위 **08-4** 예제와 같은 조건을 유지하면서 출력 결과를 종이에 따라 적어 보세요(`1`부터 `10`까지, `3`의 배수는 건너뜀, `8` 이상이면 종료). 그 다음 직접 컴파일·실행해 예상과 일치하는지 확인하세요.
- 입력: 해당 없음
- 출력: `1 2 4 5 7` 줄바꿈
- 조건(힌트): 예제에서 `break`와 `continue` 줄을 순서만 바꿔 실행해 보았을 때 왜 결과가 깨지는지도 설명해 보세요.

**문제 5**
- 문제: 1부터 증가시키며 출력하되, 3의 배수는 건너뛰고(`continue`) 20 이상이면 종료(`break`)하는 코드를 작성하세요.
- 입력: 해당 없음
- 출력: 조건에 맞게 필터링된 숫자 목록
- 조건(힌트): **`08-4`** 예제의 흐름을 그대로 삼되, `break` 조건식의 상수값을 목표 종료값에 맞게 바꿉니다.

### 정답 예시

```c
for (int i = 2; i <= 10; i += 2) printf("%d ", i);
int n = 10;
while (n >= 1) { printf("%d ", n); n--; }

int menu = 0;
do {
    printf("메뉴 출력\n");
    scanf("%d", &menu);
} while (menu != 2);
```

문제 4는 **코드 수정 없이 예제 블록**을 따라 적고 실행해 확인하면 됩니다. `continue` 다음에 두었던 `if (i >= 8)` 검사까지 도달하지 못하게 `break`를 먼저 두면 패턴이 달라지므로, 분기 순서가 결과에 직결된다는 점만 짚어도 충분합니다.

문제 5는 예를 들어 `for (;;)`나 `while`로 카운터를 올린 뒤 `if (i >= 20) break;`, `if (i % 3 == 0) continue;`, `printf` 순으로 두면 `20`까지 가기 전까지 3의 배수만 걸러내는 패턴으로 맞출 수 있습니다. `break`와 `continue`를 배치한 순서에 따라 같은 `i`에서도 결과가 어긋날 수 있으니, 숫자를 따라가며 비교해 보세요.
